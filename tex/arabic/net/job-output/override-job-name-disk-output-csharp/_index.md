# كيفية التقاط مخرجات وحدة التحكم C# – تجاوز اسم المهمة وكتابة إلى القرص

في هذا الدرس خطوة بخطوة ستكتشف **كيفية التقاط مخرجات وحدة التحكم C#** عند استخدام Aspose.TeX لـ .NET. من خلال تجاوز اسم المهمة وتوجيه تدفق الطرفية إلى ملف، ستحصل على تسمية ملفات حتمية، تجميع سجلات سهل، وأساس قوي لأنابيب CI/CD أو سيناريوهات المعالجة الدفعية.

## إجابات سريعة
- **ما معنى “capture console output C#”?** إنه يعيد توجيه تدفق الطرفية القياسي الذي تولده Aspose.TeX إلى ملف تحدده، مع الحفاظ على كل سجل، تحذير، ورسالة حالة.  
- **لماذا يتم تجاوز اسم المهمة؟** يضمن التجاوز أسماء ملفات متوقعة، ويقضي على تصادمات الأسماء في المهام المتوازية، ويسهل السكريبتات اللاحقة.  
- **أي فئة في Aspose.TeX تكتب المخرجات؟** `OutputFileTerminal`، يتم تعيينها عبر `options.TerminalOut`. *OutputFileTerminal هي فئة تكتب رسائل الطرفية إلى ملف.*  
- **هل يمكنني تعيين مجلد إدخال TeX مخصص؟** نعم—استخدم `options.InputWorkingDirectory` للإشارة إلى ملفات `.tex` المصدرية الخاصة بك.  
- **هل هذا النهج متوافق مع .NET Core؟** بالتأكيد؛ نفس الـ API يعمل على .NET Framework و .NET Core و .NET 5+.

## ما هو “capture console output C#” في سياق Aspose.TeX؟
يعني التقاط مخرجات وحدة التحكم أخذ كل ما كان سيظهر عادةً في نافذة الطرفية (رسائل السجل، التحذيرات، تفاصيل التجميع) وكتابته إلى ملف دائم. هذا مفيد بشكل خاص لتصحيح مشاريع TeX الكبيرة أو دمج معالجة TeX في سير عمل آلي.

## لماذا يتم تجاوز اسم المهمة وكتابة مخرجات الطرفية إلى ملف؟
يتيح لك تجاوز اسم المهمة تحديد الاسم الأساسي لجميع القطع المولدة، بينما كتابة سجل الطرفية يمنحك مسار تدقيق كامل. تجعل أسماء الملفات المتوقعة سكريبتات ما بعد المعالجة بسيطة، ويسمح السجل المحفوظ بمراجعة خطوات التجميع بعد حدوثها، وهو أمر حاسم للبُنى التحتية للبناء الآلي والبيئات التنفيذية المتوازية.

## الفوائد الكمية لاستخدام Aspose.TeX
يدعم Aspose.TeX **أكثر من 30 تنسيق إدخال وإخراج**—بما في ذلك `.tex`، `.pdf`، `.xps`، `.html`، وأنواع الصور الشائعة—ويمكنه معالجة **مستندات تصل إلى 500 صفحة** دون تحميل الملف بالكامل في الذاكرة. يقلل بنية البث في المكتبة من استهلاك الذاكرة القصوى بنسبة تصل إلى **70 %** مقارنةً بالنهج القائمة على الملفات البسيطة.

- **مجلدات الإدخال والإخراج** – أنشئ مجلدين على القرص: أحدهما لملفات `.tex` المصدرية والآخر للقطع المولدة والسجلات.

## استيراد مساحات الأسماء

أضف توجيهات `using` المطلوبة في أعلى ملف C# الخاص بك حتى يتمكن المترجم من العثور على أنواع Aspose.TeX.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## الخطوة 1: إنشاء خيارات التحويل

تجمع فئة `TeXOptions` جميع إعدادات التكوين لوظيفة تحويل TeX. تتيح لك تحديد سلوك تطبيق وحدة التحكم، مجلدات الإدخال، وأجهزة الإخراج.

`ConsoleAppOptions` يضبط سلوك تطبيق وحدة التحكم لـ Aspose.TeX.  
`TeXConfig` يحدد نوع تكوين معالجة TeX.

أنشئ مثيلًا من `TeXOptions` باستخدام `ConsoleAppOptions`، مع تحديد `ObjectTeX` كـ `TeXConfig`. هذا يخبر Aspose.TeX أنك تعمل في سيناريو تطبيق وحدة تحكم.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## الخطوة 2: تحديد اسم المهمة (تجاوز الافتراضي)

تستخدم فئة `TeXJob` اسم المهمة كأساس لكل ملف مولد. تجاوزها يمنع المكتبة من إنشاء أسماء عشوائية مثل `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

قم بتعيين `options.JobName = "MyCustomJob"` (أو أي معرف تفضله) قبل إنشاء مثيل الوظيفة.

## الخطوة 3: تعيين دليل إدخال TeX

خاصية `InputWorkingDirectory` تخبر Aspose.TeX بمكان العثور على ملفات `.tex` المصدرية الخاصة بك. هذه هي خطوة **تعيين دليل إدخال tex**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

قدّم المسار المطلق أو النسبي للمجلد الذي يحتوي على مصادر TeX الخاصة بك، على سبيل المثال، `@"C:\\TeX\\Sources"`.

## الخطوة 4: تحديد دليل العمل للإخراج

حدد المجلد الذي سيتم حفظ الملفات المعالجة وسجل وحدة التحكم الملتقطة فيه. سيقوم Aspose.TeX بإنشاء مجلدات فرعية حسب الحاجة، لكن يجب أن يكون المجلد الجذر موجودًا وقابلًا للكتابة.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

عيّن `options.OutputWorkingDirectory = @"C:\\TeX\\Results"` (تأكد من أن المجلد لديه أذونات كتابة).

## الخطوة 5: كتابة مخرجات الطرفية إلى ملف

`OutputFileTerminal` هو التنفيذ الفعلي الذي يكتب كل رسالة وحدة تحكم إلى ملف. من خلال تعيينه إلى `options.TerminalOut`، تستبدل تدفق وحدة التحكم الافتراضي في الذاكرة.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

مثال: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## الخطوة 6: تشغيل المهمة

فئة `TeXJob` تنسق عملية التحويل. تقوم بتوفير اسم المهمة المتجاوز، جهاز الإخراج (`XpsDevice` في هذا المثال)، وكائن الخيارات المكوَّن بالكامل. يولد `XpsDevice` مخرجات XPS من مصدر TeX.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

بعد استدعاء `job.Run();`، يقوم Aspose.TeX بإنشاء ملف XPS ويكتب سجل الطرفية بالكامل إلى `console.log` في مجلد الإخراج.

## المشكلات الشائعة & استكشاف الأخطاء وإصلاحها

| العَرَض | السبب المحتمل | الحل |
|---------|--------------|-----|
| لم يتم إنشاء ملف الإخراج | مسار دليل الإخراج غير صحيح أو يفتقر إلى أذونات الكتابة | تحقق من أن `options.OutputWorkingDirectory` يشير إلى مجلد صالح وأن العملية لديها أذونات كتابة. |
| سجل الطرفية فارغ | `TerminalOut` غير مُعيَّن أو تم الكتابة فوقه لاحقًا | تأكد من تنفيذ `options.TerminalOut = new OutputFileTerminal(...);` قبل `job.Run();`. |
| فشل المهمة برسالة “الملف غير موجود” | دليل الإدخال غير مُحدد بشكل صحيح | تحقق مرة أخرى من المسار الممرَّر إلى `options.InputWorkingDirectory` وأن ملفات `.tex` موجودة هناك. |
| أحرف غير متوقعة في السجل | عدم توافق الترميز بين محرك TeX ونظام الملفات | قم بتعيين `options.Encoding = Encoding.UTF8` (أو الترميز المناسب) قبل تشغيل المهمة. |
