## Waarom SVG genereren vanuit LaTeX?

- **Scalability:** SVG‑grafieken schalen zonder pixelvorming, perfect voor responsieve webpagina’s en hoge‑resolutie‑afdrukken.  
- **Small File Size:** Vector‑data is vaak lichter dan raster‑afbeeldingen, waardoor bandbreedte‑ en opslagkosten dalen.  
- **Editability:** SVG kan worden gestyled met CSS of gemanipuleerd via JavaScript, waardoor je dynamische controle krijgt na het renderen.  
- **Cross‑Platform Consistency:** dezelfde SVG ziet er identiek uit op Windows, macOS, Linux en mobiele apparaten.

## Veelvoorkomende valkuilen & pro‑tips

- **Pro tip:** Stel altijd een aangepaste job‑naam in bij batch‑conversies; dit houdt je output‑mappen overzichtelijk en maakt debugging makkelijker.  
- **Pitfall:** Het vergeten te sluiten van de `TeXEngine` kan leiden tot geheugenlekken. Gebruik een try‑with‑resources‑blok of roep expliciet `engine.dispose()` aan.  
- **Pro tip:** Wanneer je terminaloutput naar een ZIP‑archief schrijft, zorg er dan voor dat de ZIP‑stream wordt geflusht voordat de engine beëindigt om corrupte logs te voorkomen.
