# 如何捕獲 C# 主控台輸出 – 覆寫工作名稱並寫入磁碟

在本步驟教學中，您將了解在使用 Aspose.TeX for .NET 時 **如何捕獲 C# 主控台輸出**。透過覆寫工作名稱並將終端串流導向檔案，您可以獲得確定性的檔案命名、輕鬆的日誌彙總，以及為 CI/CD 管線或批次處理情境奠定堅實基礎。

## 快速解答
- **「捕獲 C# 主控台輸出」是什麼意思？** 它會將 Aspose.TeX 產生的標準終端串流重新導向至您指定的檔案，保留所有日誌、警告與狀態訊息。  
- **為什麼要覆寫工作名稱？** 覆寫可確保檔名可預測，避免平行工作中的命名衝突，並簡化下游腳本。  
- **哪個 Aspose.TeX 類別負責寫入輸出？** `OutputFileTerminal`，透過 `options.TerminalOut` 指定。*OutputFileTerminal 是一個將終端訊息寫入檔案的類別。*  
- **我可以設定自訂的 TeX 輸入資料夾嗎？** 可以——使用 `options.InputWorkingDirectory` 指向您的來源 `.tex` 檔案。  
- **此方法是否相容於 .NET Core？** 完全相容；相同的 API 可在 .NET Framework、.NET Core 與 .NET 5+ 上運作。

## 在 Aspose.TeX 中，「捕獲 C# 主控台輸出」是什麼意思？
捕獲主控台輸出是指將原本會顯示在終端視窗的所有內容（日誌訊息、警告、編譯細節）寫入永久檔案。這對於除錯大型 TeX 專案或將 TeX 處理整合至自動化工作流程特別有用。

## 為什麼要覆寫工作名稱並將終端輸出寫入檔案？
覆寫工作名稱讓您自行決定所有產出檔案的基礎名稱，而寫入終端日誌則提供完整的稽核軌跡。可預測的檔名使後續處理腳本變得簡單，且已保存的日誌讓您能在事後檢視編譯步驟，這對於自動化建置與平行執行環境至關重要。

## 使用 Aspose.TeX 的量化效益
Aspose.TeX 支援 **30 多種輸入與輸出格式**——包括 `.tex`、`.pdf`、`.xps`、`.html` 以及常見影像類型，且可處理 **最多 500 頁** 的文件而無需將整個檔案載入記憶體。此函式庫的串流架構相較於傳統檔案方式，可將峰值記憶體使用量降低至 **70 %**。

## 前置條件

- **Aspose.TeX for .NET** – 從 [Aspose.TeX 官方網站](https://releases.aspose.com/tex/net/) 下載最新版本。  
- **.NET 開發環境** – Visual Studio 2022、Rider，或任何支援 .NET 6+ 的 IDE。  
- **基本 C# 知識** – 您應熟悉命名空間、物件與檔案系統路徑。  
- **輸入與輸出目錄** – 在磁碟上建立兩個資料夾：一個放置來源 `.tex` 檔案，另一個存放產出檔案與日誌。

## 匯入命名空間

在 C# 檔案的頂部加入必要的 `using` 指示，以讓編譯器能找到 Aspose.TeX 類型。

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## 步驟 1：建立轉換選項

`TeXOptions` 類別封裝了 TeX 轉換工作所有的設定。它讓您指定主控台應用程式行為、輸入目錄與輸出裝置。

`ConsoleAppOptions` 設定 Aspose.TeX 的主控台應用程式行為。  
`TeXConfig` 定義 TeX 處理的設定類型。

建立一個 `TeXOptions` 實例，使用 `ConsoleAppOptions`，並將 `ObjectTeX` 指定為 `TeXConfig`。這告訴 Aspose.TeX 您正於主控台應用程式情境下執行。

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## 步驟 2：指定工作名稱（覆寫預設）

`TeXJob` 類別使用工作名稱作為所有產生檔案的基礎。覆寫它可避免函式庫產生類似 `job_12345` 的隨機名稱。

```csharp
options.JobName = "overridden-job-name";
```

在建立工作實例之前，設定 `options.JobName = "MyCustomJob"`（或任何您偏好的識別字）。

## 步驟 3：設定 TeX 輸入目錄

`InputWorkingDirectory` 屬性告訴 Aspose.TeX 您的來源 `.tex` 檔案所在位置。這就是 **設定 TeX 輸入目錄** 的步驟。

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

提供包含 TeX 原始檔的資料夾之絕對或相對路徑，例如 `@"C:\TeX\Sources"`。

## 步驟 4：指定輸出工作目錄

定義處理後檔案與捕獲的主控台日誌要儲存的資料夾。Aspose.TeX 會依需求建立子資料夾，但根目錄必須已存在且具寫入權限。

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

設定 `options.OutputWorkingDirectory = @"C:\TeX\Results"`（確保該目錄具寫入權限）。

## 步驟 5：將終端輸出寫入檔案

`OutputFileTerminal` 是將每筆主控台訊息寫入檔案的具體實作。將它指派給 `options.TerminalOut` 後，即可取代預設的記憶體主控台串流。

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

範例：`options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## 步驟 6：執行工作

`TeXJob` 類別負責協調轉換。您提供已覆寫的工作名稱、輸出裝置（本例為 `XpsDevice`）以及完整設定的 options 物件。`XpsDevice` 會從 TeX 原始檔產生 XPS 輸出。

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

呼叫 `job.Run();` 後，Aspose.TeX 會產生 XPS 檔案，並將完整的終端日誌寫入輸出資料夾中的 `console.log`。

## 常見問題與疑難排解

| 症狀 | 可能原因 | 解決方式 |
|------|----------|----------|
| 未產生輸出檔案 | 輸出目錄路徑不正確或缺少寫入權限 | 確認 `options.OutputWorkingDirectory` 指向有效的資料夾且程式具有寫入權限。 |
| 終端日誌為空 | `TerminalOut` 未設定或稍後被覆寫 | 確保在 `job.Run();` 之前執行 `options.TerminalOut = new OutputFileTerminal(...);`。 |
| 工作失敗，顯示「找不到檔案」 | 輸入目錄設定不正確 | 再次確認傳遞給 `options.InputWorkingDirectory` 的路徑，以及該目錄中是否存在 `.tex` 檔案。 |
| 日誌中出現非預期字元 | TeX 引擎與檔案系統之間的編碼不匹配 | 在執行工作前設定 `options.Encoding = Encoding.UTF8`（或相應的編碼）。 |
