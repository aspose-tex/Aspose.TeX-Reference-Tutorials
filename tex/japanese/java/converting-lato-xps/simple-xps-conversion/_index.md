# ステップバイステップ変換 – Javaで印刷可能な請求書を作成: LaTeXからXPS形式へ

## はじめに

Javaアプリケーション内でLaTeXソースから**java create printable invoices**（印刷可能な請求書）を作成する必要がある場合、正しいチュートリアルにたどり着きました。**Aspose.TeX for Java** を使用すれば、LaTeXファイルをレンダリングし、複雑な数式を処理し、期待通りに印刷できる高品質なXPSファイルを出力できます。数分で全体のワークフローを解説し、この手法が請求書生成に最適な理由を説明し、独自のレポートパイプライン向けに変換をカスタマイズする方法を示します。このガイドは、任意のJavaプロジェクトにコピー＆ペーストできる完全な**step by step conversion**（ステップバイステップ変換）を示しています。

## クイック回答
- **どのライブラリを使用すべきですか？** Aspose.TeX for Java – 50以上のフォーマットに対応した純粋なJavaエンジンです。  
- **実装にどれくらい時間がかかりますか？** 基本的な請求書パイプラインでおおよそ10〜15分です。  
- **前提条件は何ですか？** JDK 8以上、Aspose.TeX JAR、そしてIDE（Eclipse/IntelliJ）。  
- **複雑な数式をレンダリングできますか？** はい – エンジンはAMS数学パッケージを完全にサポートしています。  
- **本番環境でライセンスは必要ですか？** はい、トライアル以外の展開には商用ライセンスが必須です。  

## LaTeXからXPSへの変換でJavaで印刷可能な請求書を作成する方法

LaTeXソースを読み込み、レンダリングオプションを設定し、XPSデバイスを呼び出すだけで、5つの簡潔なステップで変換が完了します。Aspose.TeX はフォント埋め込み、数式のラスタライズ、グラフィック変換を自動的に処理するため、典型的な2ページ文書でも1秒未満で印刷可能なXPS請求書が得られます。

## ステップバイステップ変換とは何ですか？

ステップバイステップ変換は、大規模なLaTeXからXPSへのタスクを小さく繰り返し可能なアクションに分割し、各段階を個別に検証できるようにします。ファイルの読み込み、オプション設定、デバイス初期化、最終実行を分離することで、エラーを迅速に特定し、パフォーマンス設定を調整し、数十件の請求書のバッチ処理に同じパイプラインを再利用できます。

## なぜJavaのLaTeXレンダリングにAspose.TeXを使用するのか？

Aspose.TeX は外部のLaTeXバイナリを必要としない純粋なJavaレンダリングエンジンを提供し、デプロイの複雑さを削減し、Windows、macOS、Linux間で一貫した出力を保証します。**50以上**の入力・出力フォーマットに対応し、数百ページの文書でも全体をメモリに読み込まずに処理でき、フォントのサブセット化やラスタライズに対する細かな制御を提供します。

## 前提条件

- Java Development Kit (JDK) 8 以上がインストールされていること。  
- Aspose.TeX for Java ライブラリ（公式の [Aspose.TeX Java ダウンロードページ](https://releases.aspose.com/tex/java/) からダウンロード）。  
- Java IDE またはビルドツール（Maven/Gradle）で、JAR をプロジェクトのクラスパスに追加できる環境。  

## パッケージのインポート

`import` ブロックは必須クラスをスコープに持ち込みます。このリストを正確に保つことで、コードが変更なしでコンパイルできることが保証されます。

```java
package com.aspose.tex.LaTeXXpsConversionSimplest;

import java.io.ByteArrayInputStream;
import java.io.IOException;
import java.util.Calendar;
import java.util.GregorianCalendar;

import com.aspose.tex.InputFileSystemDirectory;
import com.aspose.tex.Interaction;
import com.aspose.tex.OutputFileSystemDirectory;
import com.aspose.tex.TeXConfig;
import com.aspose.tex.TeXJob;
import com.aspose.tex.TeXOptions;
import com.aspose.tex.rendering.XpsDevice;
import com.aspose.tex.rendering.XpsSaveOptions;

import util.Utils;
```

それでは、変換手順を順に見ていき、各コード断片の目的を説明します。

## 手順 1: 入力および出力ディレクトリの設定

Aspose.TeX に、ソースの `.ltx` ファイルの場所と、生成された XPS ファイルを書き込む場所を指示する必要があります。

```java
options.setInputWorkingDirectory(new InputFileSystemDirectory("Your Input Directory"));
options.setOutputWorkingDirectory(new OutputFileSystemDirectory("Your Output Directory"));
```

*Tip:* `"Your Input Directory"` と `"Your Output Directory"` を、マシン上に存在する絶対パスまたは相対パスに置き換えてください。

## 手順 2: TeX オプションの設定

これらのオプションは、変換中の LaTeX エンジンの動作を制御します。ドキュメントの要件に合わせて調整してください。  
TeXOptions はこれらの設定を保持するクラスです。

```java
options.setInteraction(Interaction.NonstopMode);
options.setDateTime(new GregorianCalendar(2022, Calendar.DECEMBER, 18).getTime());
options.ignoreMissingPackages(true);
options.noLigatures(true);
options.repeat(true);
```

- **Interaction mode** – `NonstopMode` はエラーが発生しても処理を継続します。  
- **DateTime** – ドキュメントのタイトルページに表示される日付を設定します。  
- **ignoreMissingPackages** – パッケージが見つからなくてもジョブが失敗しないようにします。  
- **noLigatures** – タイポグラフィ的リガチャを無効にし、プレーン文字を使用します。  
- **repeat** – クロスリファレンスのためにエンジンを再実行できるようにします。  

## 手順 3: XPS 保存オプションの初期化

XpsSaveOptions は、圧縮やページサイズなどの XPS 出力パラメータを設定します。

```java
options.setSaveOptions(new XpsSaveOptions());
```

## 手順 4: XPS 保存オプションのカスタマイズ

XPS 出力が数式、グラフィック、フォントをどのように扱うかを細かく調整します。  
`rasterizeFormulas` は数式を画像としてレンダリングするかどうかを決定します。

```java
options.getSaveOptions().rasterizeFormulas(true);
options.getSaveOptions().rasterizeIncludedGraphics(true);
options.getSaveOptions().subsetFonts(true);
```

- **rasterizeFormulas** – 数学式を画像に変換し、すべての XPS ビューアで正しく表示できるようにします。  
- **rasterizeIncludedGraphics** – 埋め込みグラフィックをラスタライズさせ、互換性を向上させます。  
- **subsetFonts** – 文書で使用されているグリフだけを埋め込み、典型的な請求書で最大 40 % のファイルサイズ削減を実現します。  

## 手順 5: LaTeX から XPS への変換を実行

XpsDevice はレンダリングされたコンテンツを XPS ファイルに書き込みます。  
`TeXJob` は入力ファイル、出力デバイス（`XpsDevice`）および設定したすべてのオプションを結び付けます。

```java
new TeXJob("Your Input Directory" + "sample.ltx", new XpsDevice(), options).run();
```

`run()` 呼び出しが完了すると、先ほど指定した出力ディレクトリに `sample.xps` が作成されます。この XPS ファイルは直接プリンターに送信するか、PDF 請求書バンドルに埋め込むことができます。

## 追加例

### InputStream の使用（LaTeX 文字列を直接変換）

ファイルではなくメモリ上の LaTeX ソースを供給したい場合は、ソースを `ByteArrayInputStream` でラップします。

```java
new TeXJob(new ByteArrayInputStream(
    "\\documentclass{article} \\begin{document} Hello, World! \\end{document}".getBytes("ASCII")),
    new XpsDevice(), options).run();
```

### メイン入力ターミナルの使用（Aspose.TeX にファイルを自動検出させる）

ファイルパスを明示的に指定する必要がない場合は、デフォルトの入力ターミナルに任せることができます。

```java
new TeXJob(new XpsDevice(), options).run();
```
