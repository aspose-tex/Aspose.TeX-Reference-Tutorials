# コンソール出力のキャプチャ方法 C# – ジョブ名のオーバーライドとディスクへの書き込み

## クイック回答
- **“capture console output C#” とは何ですか？** Aspose.TeX が生成する標準ターミナルストリームを指定したファイルにリダイレクトし、すべてのログ、警告、ステータスメッセージを保持します。  
- **なぜジョブ名をオーバーライドするのですか？** オーバーライドにより予測可能なファイル名が保証され、並列ジョブでの名前衝突が防止され、下流スクリプトが簡素化されます。  
- **どの Aspose.TeX クラスが出力を書き込みますか？** `OutputFileTerminal` は `options.TerminalOut` を介して割り当てられます。*OutputFileTerminal はターミナルメッセージをファイルに書き込むクラスです。*  
- **カスタムの TeX 入力フォルダーを設定できますか？** はい — `options.InputWorkingDirectory` を使用してソース `.tex` ファイルの場所を指定します。  
- **このアプローチは .NET Core と互換性がありますか？** 完全に対応しています。 同じ API が .NET Framework、.NET Core、.NET 5+ で動作します。

## Aspose.TeX のコンテキストで “capture console output C#” とは何か
コンソール出力のキャプチャとは、通常ターミナルウィンドウに表示されるすべて（ログメッセージ、警告、コンパイル詳細）を永続的なファイルに書き込むことです。 大規模な TeX プロジェクトのデバッグや、TeX 処理を自動化ワークフローに組み込む際に特に有用です。

## なぜジョブ名をオーバーライドし、ターミナル出力をファイルに書き込むのか
ジョブ名をオーバーライドすると、生成されるすべてのアーティファクトのベース名を自分で決められます。 ターミナルログをファイルに保存すれば、完全な監査トレイルが得られます。 予測可能なファイル名によりポストプロセススクリプトが簡素化され、保存されたログによりビルド後にコンパイル手順を確認でき、 自動ビルドや並列実行環境で重要です。

## Aspose.TeX を使用する定量的なメリット
Aspose.TeX は **30 以上の入力および出力フォーマット**（`.tex`、`.pdf`、`.xps`、`.html`、一般的な画像形式など）をサポートし、 **最大 500 ページ** のドキュメントをメモリ全体にロードせずに処理できます。 ライブラリのストリーミングアーキテクチャにより、従来のファイルベースアプローチと比較してピークメモリ使用量が **最大 70 %** 削減されます。

## 前提条件

- **Aspose.TeX for .NET** – 最新リリースは [Aspose.TeX のウェブサイト](https://releases.aspose.com/tex/net/) からダウンロードしてください。  
- **.NET 開発環境** – Visual Studio 2022、Rider、または .NET 6+ をサポートする任意の IDE。  
- **基本的な C# の知識** – 名前空間、オブジェクト、ファイルシステムパスに慣れていること。  
- **入力および出力ディレクトリ** – ディスク上に 2 つのフォルダーを作成します。1 つはソース `.tex` ファイル用、もう 1 つは生成アーティファクトとログ用です。

## 名前空間のインポート

C# ファイルの先頭に必要な `using` ディレクティブを追加し、コンパイラが Aspose.TeX の型を見つけられるようにします。

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## 手順 1: 変換オプションの作成

`TeXOptions` クラスは TeX 変換ジョブのすべての設定をカプセル化します。 コンソールアプリの動作、入力ディレクトリ、出力デバイスを指定できます。

`ConsoleAppOptions` は Aspose.TeX のコンソールアプリケーション動作を構成します。  
`TeXConfig` は TeX 処理構成の種類を定義します。

`ConsoleAppOptions` と `ObjectTeX`（`TeXConfig`）を使用して `TeXOptions` インスタンスを作成します。 これにより、コンソールアプリケーションシナリオであることを Aspose.TeX に通知します。

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## 手順 2: ジョブ名の指定（デフォルトのオーバーライド）

`TeXJob` クラスはジョブ名をすべての生成ファイルのベースとして使用します。 オーバーライドしない場合、ライブラリは `job_12345` のようなランダム名を生成します。

```csharp
options.JobName = "overridden-job-name";
```

`options.JobName = "MyCustomJob"`（または任意の識別子）をジョブインスタンスを作成する前に設定してください。

## 手順 3: TeX 入力ディレクトリの設定

`InputWorkingDirectory` プロパティは Aspose.TeX にソース `.tex` ファイルの場所を指示します。 これが **TeX 入力ディレクトリの設定** 手順です。

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

TeX ソースが格納されているフォルダーへの絶対パスまたは相対パスを指定します。例: `@"C:\TeX\Sources"`。

## 手順 4: 出力作業ディレクトリの指定

処理されたファイルとキャプチャされたコンソールログを保存するフォルダーを定義します。 Aspose.TeX は必要に応じてサブフォルダーを作成しますが、ルートフォルダーは事前に存在し、書き込み可能である必要があります。

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

`options.OutputWorkingDirectory = @"C:\TeX\Results"` を割り当てます（ディレクトリに書き込み権限があることを確認してください）。

## 手順 5: ターミナル出力をファイルに書き込む

`OutputFileTerminal` はすべてのコンソールメッセージをファイルに書き込む具体的な実装です。 これを `options.TerminalOut` に割り当てることで、デフォルトのインメモリコンソールストリームを置き換えます。

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

例: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## 手順 6: ジョブの実行

`TeXJob` クラスが変換を統括します。 オーバーライドしたジョブ名、出力デバイス（この例では `XpsDevice`）、完全に構成されたオプションオブジェクトを提供します。 `XpsDevice` は TeX ソースから XPS 出力を生成します。

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

`job.Run();` を呼び出すと、Aspose.TeX は XPS ファイルを生成し、出力フォルダー内の `console.log` にターミナル全体のログを書き込みます。

## よくある問題とトラブルシューティング

| 症状 | 考えられる原因 | 対処法 |
|------|----------------|--------|
| 出力ファイルが作成されない | 出力ディレクトリのパスが間違っている、または書き込み権限がない | `options.OutputWorkingDirectory` が有効なフォルダーを指し、プロセスに書き込み権限があることを確認してください。 |
| ターミナルログが空 | `TerminalOut` が設定されていない、または後で上書きされている | `job.Run();` の前に `options.TerminalOut = new OutputFileTerminal(...);` が実行されていることを確認してください。 |
| “file not found” エラーでジョブが失敗 | 入力ディレクトリが正しく設定されていない | `options.InputWorkingDirectory` に渡したパスと、そこに `.tex` ファイルが存在することを再確認してください。 |
| ログに予期しない文字が含まれる | TeX エンジンとファイルシステム間のエンコーディング不一致 | ジョブ実行前に `options.Encoding = Encoding.UTF8`（または適切なエンコーディング）を設定してください。 |
