# How to Capture Console Output C# – Override Job Name & Write to Disk

In this step‑by‑step tutorial you’ll discover **how to capture console output C#** when using Aspose.TeX for .NET. By overriding the job name and routing the terminal stream to a file, you gain deterministic file naming, easy log aggregation, and a solid foundation for CI/CD pipelines or batch‑processing scenarios.

## Quick Answers
- **What does “capture console output C#” mean?** It redirects the standard terminal stream generated by Aspose.TeX into a file you specify, preserving every log, warning, and status message.  
- **Why override the job name?** Overriding guarantees predictable file names, eliminates naming collisions in parallel jobs, and simplifies downstream scripts.  
- **Which Aspose.TeX class writes the output?** `OutputFileTerminal`, assigned through `options.TerminalOut`. *OutputFileTerminal is a class that writes terminal messages to a file.*  
- **Can I set a custom TeX input folder?** Yes—use `options.InputWorkingDirectory` to point to your source `.tex` files.  
- **Is this approach compatible with .NET Core?** Absolutely; the same API works on .NET Framework, .NET Core, and .NET 5+.

## What is “capture console output C#” in the context of Aspose.TeX?
Capturing console output means taking everything that would normally appear in the terminal window (log messages, warnings, compilation details) and writing it to a persistent file. This is especially useful for debugging large TeX projects or integrating TeX processing into automated workflows.

## Why override the job name and write terminal output to a file?
Overriding the job name lets you decide the base name for all generated artifacts, while writing the terminal log gives you a complete audit trail. Predictable filenames make post‑processing scripts trivial, and a saved log enables you to review compilation steps after the fact, which is critical for automated builds and parallel execution environments.

## Quantified Benefits of Using Aspose.TeX
Aspose.TeX supports **30+ input and output formats**—including `.tex`, `.pdf`, `.xps`, `.html`, and common image types—and can process **documents up to 500 pages** without loading the entire file into memory. The library’s streaming architecture reduces peak memory usage by up to **70 %** compared with naïve file‑based approaches.

## Import Namespaces

Add the required `using` directives at the top of your C# file so the compiler can locate Aspose.TeX types.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Step 1: Create Conversion Options

The `TeXOptions` class encapsulates all configuration settings for a TeX conversion job. It lets you specify console‑app behavior, input directories, and output devices.

`ConsoleAppOptions` configures console‑application behavior for Aspose.TeX.  
`TeXConfig` defines the type of TeX processing configuration.

Create a `TeXOptions` instance with `ConsoleAppOptions`, specifying `ObjectTeX` as the `TeXConfig`. This tells Aspose.TeX that you are running in a console‑application scenario.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Step 2: Specify Job Name (Override Default)

The `TeXJob` class uses the job name as the base for every generated file. Overriding it prevents the library from generating random names like `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

Set `options.JobName = "MyCustomJob"` (or any identifier you prefer) before creating the job instance.

## Step 3: Set TeX Input Directory

The `InputWorkingDirectory` property tells Aspose.TeX where to locate your source `.tex` files. This is the **set tex input directory** step.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Provide the absolute or relative path to the folder that contains your TeX sources, e.g., `@"C:\TeX\Sources"`.

## Step 4: Specify Output Working Directory

Define the folder where processed files and the captured console log will be saved. Aspose.TeX will create sub‑folders as needed, but the root folder must exist and be writable.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Assign `options.OutputWorkingDirectory = @"C:\TeX\Results"` (ensure the directory has write permissions).

## Step 5: Write Terminal Output to File

`OutputFileTerminal` is the concrete implementation that writes every console message to a file. By assigning it to `options.TerminalOut`, you replace the default in‑memory console stream.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Example: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Step 6: Run the Job

The `TeXJob` class orchestrates the conversion. You provide the overridden job name, the output device (`XpsDevice` in this example), and the fully configured options object. `XpsDevice` generates XPS output from the TeX source.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

After calling `job.Run();`, Aspose.TeX generates the XPS file and writes the entire terminal log to `console.log` in the output folder.

## Common Issues & Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|--------------|-----|
| No output file created | Output directory path is incorrect or missing write permissions | Verify `options.OutputWorkingDirectory` points to a valid folder and the process has write access. |
| Terminal log is empty | `TerminalOut` not set or overwritten later | Ensure `options.TerminalOut = new OutputFileTerminal(...);` is executed before `job.Run();`. |
| Job fails with “file not found” | Input directory not set correctly | Double‑check the path passed to `options.InputWorkingDirectory` and that the `.tex` files exist there. |
| Unexpected characters in log | Encoding mismatch between TeX engine and file system | Set `options.Encoding = Encoding.UTF8` (or the appropriate encoding) before running the job. |
