# Wie man Konsolenausgabe in C# erfasst – Jobnamen überschreiben & auf Festplatte schreiben

In diesem Schritt‑für‑Schritt‑Tutorial erfahren Sie **wie man Konsolenausgabe in C# erfasst** bei der Verwendung von Aspose.TeX für .NET. Durch das Überschreiben des Jobnamens und das Weiterleiten des Terminal‑Streams in eine Datei erhalten Sie deterministische Dateinamen, einfache Log‑Aggregation und eine solide Grundlage für CI/CD‑Pipelines oder Batch‑Verarbeitungsszenarien.

## Schnelle Antworten
- **Was bedeutet „capture console output C#“?** Es leitet den von Aspose.TeX erzeugten Standard‑Terminal‑Stream in eine von Ihnen angegebene Datei um und bewahrt jedes Log, jede Warnung und jede Statusmeldung.  
- **Warum den Jobnamen überschreiben?** Das Überschreiben garantiert vorhersehbare Dateinamen, verhindert Namenskollisionen bei parallelen Jobs und vereinfacht nachgelagerte Skripte.  
- **Welche Aspose.TeX‑Klasse schreibt die Ausgabe?** `OutputFileTerminal`, zugewiesen über `options.TerminalOut`. *OutputFileTerminal ist eine Klasse, die Terminalnachrichten in eine Datei schreibt.*  
- **Kann ich einen benutzerdefinierten TeX‑Eingabeordner festlegen?** Ja – verwenden Sie `options.InputWorkingDirectory`, um auf Ihre Quell‑`.tex`‑Dateien zu verweisen.  
- **Ist dieser Ansatz mit .NET Core kompatibel?** Absolut; dieselbe API funktioniert unter .NET Framework, .NET Core und .NET 5+.

## Was bedeutet „capture console output C#“ im Kontext von Aspose.TeX?
Das Erfassen der Konsolenausgabe bedeutet, alles, was normalerweise im Terminalfenster erscheinen würde (Log‑Nachrichten, Warnungen, Kompilierungsdetails), in eine dauerhafte Datei zu schreiben. Dies ist besonders nützlich zum Debuggen großer TeX‑Projekte oder zur Integration der TeX‑Verarbeitung in automatisierte Workflows.

## Warum den Jobnamen überschreiben und die Terminalausgabe in eine Datei schreiben?
Das Überschreiben des Jobnamens ermöglicht es Ihnen, den Basisnamen für alle erzeugten Artefakte festzulegen, während das Schreiben des Terminal‑Logs Ihnen einen vollständigen Prüfpfad bietet. Vorhersehbare Dateinamen machen Nachbearbeitungsskripte trivial, und ein gespeichertes Log ermöglicht es Ihnen, die Kompilierungsschritte nachträglich zu überprüfen, was für automatisierte Builds und Parallel‑Ausführungsumgebungen entscheidend ist.

## Quantifizierte Vorteile der Verwendung von Aspose.TeX
Aspose.TeX unterstützt **30+ Eingabe‑ und Ausgabeformate** – einschließlich `.tex`, `.pdf`, `.xps`, `.html` und gängiger Bildtypen – und kann **Dokumente bis zu 500 Seiten** verarbeiten, ohne die gesamte Datei in den Speicher zu laden. Die Streaming‑Architektur der Bibliothek reduziert die Spitzen‑Speicherauslastung um bis zu **70 %** im Vergleich zu naiven dateibasierten Ansätzen.

## Namespaces importieren

Fügen Sie die erforderlichen `using`‑Direktiven am Anfang Ihrer C#‑Datei hinzu, damit der Compiler die Aspose.TeX‑Typen finden kann.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Schritt 1: Konvertierungsoptionen erstellen

Die Klasse `TeXOptions` fasst alle Konfigurationseinstellungen für einen TeX‑Konvertierungsjob zusammen. Sie ermöglicht die Angabe des Verhaltens der Konsolen‑App, Eingabeverzeichnisse und Ausgabegeräte.

`ConsoleAppOptions` konfiguriert das Verhalten der Konsolen‑Anwendung für Aspose.TeX.  
`TeXConfig` definiert den Typ der TeX‑Verarbeitungskonfiguration.

Erstellen Sie eine `TeXOptions`‑Instanz mit `ConsoleAppOptions` und geben Sie `ObjectTeX` als `TeXConfig` an. Dies teilt Aspose.TeX mit, dass Sie in einem Konsolen‑Anwendungsszenario laufen.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Schritt 2: Jobnamen festlegen (Standard überschreiben)

Die Klasse `TeXJob` verwendet den Jobnamen als Basis für jede erzeugte Datei. Das Überschreiben verhindert, dass die Bibliothek zufällige Namen wie `job_12345` erzeugt.

```csharp
options.JobName = "overridden-job-name";
```

Setzen Sie `options.JobName = "MyCustomJob"` (oder einen beliebigen Bezeichner Ihrer Wahl), bevor Sie die Job‑Instanz erstellen.

## Schritt 3: TeX‑Eingabeverzeichnis festlegen

Die Eigenschaft `InputWorkingDirectory` gibt Aspose.TeX an, wo Ihre Quell‑`.tex`‑Dateien zu finden sind. Dies ist der Schritt **TeX‑Eingabeverzeichnis festlegen**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Geben Sie den absoluten oder relativen Pfad zu dem Ordner an, der Ihre TeX‑Quellen enthält, z. B. `@"C:\\TeX\\Sources"`.

## Schritt 4: Ausgabearbeitsverzeichnis festlegen

Definieren Sie den Ordner, in dem verarbeitete Dateien und das erfasste Konsolen‑Log gespeichert werden. Aspose.TeX erstellt bei Bedarf Unterordner, aber der Stammordner muss existieren und beschreibbar sein.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Setzen Sie `options.OutputWorkingDirectory = @"C:\\TeX\\Results"` (stellen Sie sicher, dass das Verzeichnis Schreibrechte hat).

## Schritt 5: Terminalausgabe in Datei schreiben

`OutputFileTerminal` ist die konkrete Implementierung, die jede Konsolennachricht in eine Datei schreibt. Durch die Zuweisung zu `options.TerminalOut` ersetzen Sie den standardmäßigen In‑Memory‑Konsolen‑Stream.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Beispiel: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Schritt 6: Job ausführen

Die Klasse `TeXJob` steuert die Konvertierung. Sie geben den überschriebenen Jobnamen, das Ausgabegerät (`XpsDevice` in diesem Beispiel) und das vollständig konfigurierte Options‑Objekt an. `XpsDevice` erzeugt XPS‑Ausgabe aus der TeX‑Quelle.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Nach dem Aufruf von `job.Run();` erzeugt Aspose.TeX die XPS‑Datei und schreibt das gesamte Terminal‑Log in `console.log` im Ausgabeverzeichnis.

## Häufige Probleme & Fehlersuche

| Symptom | Wahrscheinliche Ursache | Lösung |
|---------|--------------------------|--------|
| Keine Ausgabedatei erstellt | Pfad des Ausgabeverzeichnisses ist falsch oder Schreibrechte fehlen | Überprüfen Sie, dass `options.OutputWorkingDirectory` auf einen gültigen Ordner zeigt und der Prozess Schreibzugriff hat. |
| Terminal‑Log ist leer | `TerminalOut` ist nicht gesetzt oder später überschrieben | Stellen Sie sicher, dass `options.TerminalOut = new OutputFileTerminal(...);` vor `job.Run();` ausgeführt wird. |
| Job schlägt mit „Datei nicht gefunden“ fehl | Eingabeverzeichnis nicht korrekt gesetzt | Überprüfen Sie den an `options.InputWorkingDirectory` übergebenen Pfad und dass die `.tex`‑Dateien dort vorhanden sind. |
| Unerwartete Zeichen im Log | Kodierungsinkompatibilität zwischen TeX‑Engine und Dateisystem | Setzen Sie `options.Encoding = Encoding.UTF8` (oder die passende Kodierung) vor dem Ausführen des Jobs. |
