# Schritt‑für‑Schritt‑Konvertierung – Java druckbare Rechnungen erstellen: LaTeX zu XPS‑Format

## Einführung

Wenn Sie **java create printable invoices** aus LaTeX‑Quellen innerhalb einer Java‑Anwendung erstellen müssen, sind Sie hier genau richtig. Mit **Aspose.TeX for Java** können Sie LaTeX‑Dateien rendern, komplexe Formeln verarbeiten und hochwertige XPS‑Dateien ausgeben, die exakt so drucken, wie Sie es erwarten. In den nächsten Minuten führen wir Sie durch den gesamten Workflow, erklären, warum diese Methode ideal für die Rechnungserstellung ist, und zeigen Ihnen, wie Sie die Konvertierung für Ihre eigene Reporting‑Pipeline anpassen können. Dieser Leitfaden demonstriert eine vollständige **step by step conversion**, die Sie in jedes Java‑Projekt kopieren‑und‑einfügen können.

## Schnelle Antworten
- **Welche Bibliothek sollte ich verwenden?** Aspose.TeX for Java – eine reine Java‑Engine mit Unterstützung für mehr als 50 Formate.  
- **Wie lange dauert die Implementierung?** Ungefähr 10‑15 Minuten für eine grundlegende Rechnungspipeline.  
- **Was sind die Voraussetzungen?** JDK 8+, Aspose.TeX JAR und eine IDE (Eclipse/IntelliJ).  
- **Kann ich komplexe Formeln rendern?** Ja – die Engine unterstützt vollständig AMS‑Mathe‑Pakete.  
- **Ist für die Produktion eine Lizenz erforderlich?** Ja, eine kommerzielle Lizenz ist für den Einsatz außerhalb der Testversion zwingend erforderlich.  

## Wie man java create printable invoices mit LaTeX‑zu‑XPS‑Konvertierung erstellt?

Laden Sie Ihre LaTeX‑Quelle, konfigurieren Sie die Rendering‑Optionen und rufen Sie das XPS‑Gerät auf – das ist die gesamte Konvertierung in fünf knappen Schritten. Aspose.TeX übernimmt das Einbetten von Schriftarten, die Rasterisierung von Formeln und die Grafik‑Konvertierung automatisch, sodass Sie innerhalb einer Sekunde eine druckfertige XPS‑Rechnung für typische 2‑seitige Dokumente erhalten.

## Was ist Schritt‑für‑Schritt‑Konvertierung?

Eine Schritt‑für‑Schritt‑Konvertierung zerlegt eine große LaTeX‑zu‑XPS‑Aufgabe in kleine, wiederholbare Aktionen, sodass Sie jede Phase unabhängig validieren können. Durch die Trennung von Dateiladen, Optionskonfiguration, Geräteinitialisierung und dem abschließenden Durchlauf können Sie Fehler schnell finden, Leistungseinstellungen anpassen und dieselbe Pipeline für die Stapelverarbeitung von Dutzenden Rechnungen wiederverwenden.

## Warum Aspose.TeX for Java LaTeX‑Rendering verwenden?

Aspose.TeX bietet eine reine Java‑Rendering‑Engine, die die Notwendigkeit externer LaTeX‑Binärdateien eliminiert, die Bereitstellungskomplexität reduziert und konsistente Ausgaben unter Windows, macOS und Linux garantiert. Es unterstützt **50+** Eingabe‑ und Ausgabeformate, verarbeitet Dokumente mit mehreren hundert Seiten, ohne die gesamte Datei in den Speicher zu laden, und bietet eine feinkörnige Kontrolle über Schriftart‑Subset‑Erstellung und Rasterisierung.

## Pakete importieren

Der `import`‑Block bringt die wesentlichen Klassen in den Gültigkeitsbereich. Das Beibehalten dieser genauen Liste stellt sicher, dass der Code ohne Änderungen kompiliert.

```java
package com.aspose.tex.LaTeXXpsConversionSimplest;

import util.Utils;
```

Jetzt gehen wir die Konvertierungsschritte durch und erklären den Zweck jedes Code‑Fragments.

## Schritt 1: Eingabe‑ und Ausgabeverzeichnisse festlegen

Sie müssen Aspose.TeX mitteilen, wo die Quell‑`.ltx`‑Datei zu finden ist und wohin die resultierende XPS‑Datei geschrieben werden soll.

```java
options.setInputWorkingDirectory(new InputFileSystemDirectory("Your Input Directory"));
options.setOutputWorkingDirectory(new OutputFileSystemDirectory("Your Output Directory"));
```

*Hinweis:* Ersetzen Sie `"Your Input Directory"` und `"Your Output Directory"` durch absolute oder relative Pfade, die auf Ihrem Rechner existieren.

## Schritt 2: TeX‑Optionen konfigurieren

Diese Optionen steuern das Verhalten der LaTeX‑Engine während der Konvertierung. Passen Sie sie an die Anforderungen Ihres Dokuments an.  
TeXOptions ist die Klasse, die diese Konfigurationseinstellungen enthält.

```java
options.setInteraction(Interaction.NonstopMode);
options.setDateTime(new GregorianCalendar(2022, Calendar.DECEMBER, 18).getTime());
options.ignoreMissingPackages(true);
options.noLigatures(true);
options.repeat(true);
```

- **Interaction mode** – `NonstopMode` fährt mit der Verarbeitung fort, selbst wenn Fehler auftreten.  
- **DateTime** – Legt das Datum fest, das auf der Titelseite des Dokuments erscheint.  
- **ignoreMissingPackages** – Verhindert, dass der Vorgang fehlschlägt, wenn ein Paket nicht gefunden wird.  
- **noLigatures** – Deaktiviert typografische Ligaturen, falls Sie reine Zeichen bevorzugen.  
- **repeat** – Ermöglicht der Engine, für Querverweise erneut zu laufen.

## Schritt 3: XPS‑Speicheroptionen initialisieren

XpsSaveOptions konfiguriert XPS‑Ausgabeparameter wie Kompression und Seitengröße.

```java
options.setSaveOptions(new XpsSaveOptions());
```

## Schritt 4: XPS‑Speicheroptionen anpassen

Feinabstimmung, wie die XPS‑Ausgabe Formeln, Grafiken und Schriftarten behandelt.  
`rasterizeFormulas` bestimmt, ob Formeln als Bilder gerendert werden.

```java
options.getSaveOptions().rasterizeFormulas(true);
options.getSaveOptions().rasterizeIncludedGraphics(true);
options.getSaveOptions().subsetFonts(true);
```

- **rasterizeFormulas** – Konvertiert mathematische Ausdrücke in Bilder, sodass sie in jedem XPS‑Betrachter korrekt angezeigt werden.  
- **rasterizeIncludedGraphics** – Erzwingt die Rasterisierung eingebetteter Grafiken, was die Kompatibilität verbessern kann.  
- **subsetFonts** – Bettet nur die im Dokument verwendeten Glyphen ein und reduziert die Dateigröße um bis zu 40 % für typische Rechnungen.

## Schritt 5: LaTeX‑zu‑XPS‑Konvertierung ausführen

XpsDevice schreibt den gerenderten Inhalt in eine XPS‑Datei.  
`TeXJob` verbindet die Eingabedatei, das Ausgabegerät (`XpsDevice`) und alle von Ihnen konfigurierten Optionen.

```java
new TeXJob("Your Input Directory" + "sample.ltx", new XpsDevice(), options).run();
```

Nachdem der Aufruf `run()` abgeschlossen ist, finden Sie `sample.xps` im zuvor angegebenen Ausgabeverzeichnis. Diese XPS‑Datei kann direkt an einen Drucker gesendet oder in ein PDF‑Rechnungs‑Bundle eingebettet werden.

## Zusätzliche Beispiele

### InputStream verwenden (LaTeX‑String direkt konvertieren)

Wenn Sie LaTeX‑Quellcode lieber aus dem Speicher statt aus einer Datei zuführen möchten, verpacken Sie die Quelle in einen `ByteArrayInputStream`.

```java
new TeXJob(new ByteArrayInputStream(
    "\\documentclass{article} \\begin{document} Hello, World! \\end{document}".getBytes("ASCII")),
    new XpsDevice(), options).run();
```

### Haupt‑Eingabeterminal verwenden (Aspose.TeX die Datei automatisch finden lassen)

Wenn Sie den Dateipfad nicht explizit angeben müssen, können Sie das Standard‑Eingabeterminal nutzen.

```java
new TeXJob(new XpsDevice(), options).run();
```
