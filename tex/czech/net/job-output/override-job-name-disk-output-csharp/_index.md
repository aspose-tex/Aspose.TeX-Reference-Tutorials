# Jak zachytit výstup konzole v C# – Přepsat název úlohy a zapisovat na disk

V tomto krok‑za‑krokem tutoriálu objevíte **jak zachytit výstup konzole v C#** při použití Aspose.TeX pro .NET. Přepsáním názvu úlohy a směrováním výstupního proudu terminálu do souboru získáte deterministické pojmenování souborů, snadnou agregaci logů a solidní základ pro CI/CD pipeline nebo scénáře dávkového zpracování.

## Rychlé odpovědi
- **Co znamená “capture console output C#”?** Přesměruje standardní výstupní proud generovaný Aspose.TeX do souboru, který určíte, a zachová každý log, varování i stavovou zprávu.  
- **Proč přepsat název úlohy?** Přepsání zaručuje předvídatelná jména souborů, eliminuje kolize názvů při paralelních úlohách a zjednodušuje následné skripty.  
- **Která třída Aspose.TeX zapisuje výstup?** `OutputFileTerminal`, přiřazená přes `options.TerminalOut`. *OutputFileTerminal je třída, která zapisuje zprávy terminálu do souboru.*  
- **Mohu nastavit vlastní vstupní složku TeX?** Ano — použijte `options.InputWorkingDirectory` a nasměrujte na své zdrojové soubory `.tex`.  
- **Je tento přístup kompatibilní s .NET Core?** Rozhodně; stejné API funguje na .NET Framework, .NET Core i .NET 5+.

## Co znamená “capture console output C#” v kontextu Aspose.TeX?
Zachycení výstupu konzole znamená převzít vše, co by se normálně zobrazilo v okně terminálu (logy, varování, podrobnosti kompilace) a zapsat to do trvalého souboru. To je zvláště užitečné při ladění rozsáhlých TeX projektů nebo integraci zpracování TeX do automatizovaných pracovních postupů.

## Proč přepsat název úlohy a zapisovat výstup terminálu do souboru?
Přepsání názvu úlohy vám umožní určit základní jméno pro všechny generované artefakty, zatímco zápis logu terminálu vám poskytne kompletní auditní stopu. Předvídatelná jména souborů usnadňují post‑processing skripty a uložený log vám umožní po dokončení revizi kroků kompilace, což je klíčové pro automatizované sestavení a paralelní prostředí.

## Kvantifikované výhody používání Aspose.TeX
Aspose.TeX podporuje **30+ vstupních a výstupních formátů** — včetně `.tex`, `.pdf`, `.xps`, `.html` a běžných typů obrázků — a dokáže zpracovat **dokumenty až do 500 stran** bez načítání celého souboru do paměti. Architektura knihovny založená na streamování snižuje špičkovou spotřebu paměti až o **70 %** ve srovnání s naivními souborovými přístupy.

- **Vstupní a výstupní adresáře** – vytvořte na disku dvě složky: jednu pro zdrojové soubory `.tex` a druhou pro generované artefakty a logy.

## Import Namespaces

Přidejte požadované `using` direktivy na začátek svého C# souboru, aby kompilátor mohl najít typy Aspose.TeX.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Krok 1: Vytvořit možnosti konverze

Třída `TeXOptions` zapouzdřuje všechna konfigurační nastavení pro úlohu konverze TeX. Umožňuje specifikovat chování konzolové aplikace, vstupní adresáře a výstupní zařízení.

`ConsoleAppOptions` konfiguruje chování konzolové aplikace pro Aspose.TeX.  
`TeXConfig` definuje typ konfigurace zpracování TeX.

Vytvořte instanci `TeXOptions` s `ConsoleAppOptions`, přičemž jako `TeXConfig` použijte `ObjectTeX`. Tím řeknete Aspose.TeX, že běžíte v scénáři konzolové aplikace.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Krok 2: Specifikovat název úlohy (přepsat výchozí)

Třída `TeXJob` používá název úlohy jako základ pro každý generovaný soubor. Přepsání zabraňuje knihovně generovat náhodná jména jako `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

Nastavte `options.JobName = "MyCustomJob"` (nebo libovolný identifikátor, který preferujete) před vytvořením instance úlohy.

## Krok 3: Nastavit vstupní adresář TeX

Vlastnost `InputWorkingDirectory` říká Aspose.TeX, kde má hledat vaše zdrojové soubory `.tex`. Toto je krok **nastavení vstupního adresáře TeX**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Uveďte absolutní nebo relativní cestu ke složce, která obsahuje vaše TeX zdroje, např. `@"C:\TeX\Sources"`.

## Krok 4: Specifikovat výstupní pracovní adresář

Definujte složku, kam budou uloženy zpracované soubory a zachycený log konzole. Aspose.TeX vytvoří podadresáře podle potřeby, ale kořenová složka musí existovat a být zapisovatelná.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Přiřaďte `options.OutputWorkingDirectory = @"C:\TeX\Results"` (ujistěte se, že adresář má oprávnění k zápisu).

## Krok 5: Zapsat výstup terminálu do souboru

`OutputFileTerminal` je konkrétní implementace, která zapisuje každou zprávu konzole do souboru. Přiřazením k `options.TerminalOut` nahradíte výchozí paměťový stream konzole.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Příklad: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Krok 6: Spustit úlohu

Třída `TeXJob` orchestruje konverzi. Poskytnete přepsaný název úlohy, výstupní zařízení (`XpsDevice` v tomto příkladu) a plně nakonfigurovaný objekt možností. `XpsDevice` generuje XPS výstup ze zdrojového TeX.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Po volání `job.Run();` Aspose.TeX vygeneruje XPS soubor a zapíše celý log terminálu do `console.log` ve výstupní složce.

## Časté problémy a řešení

| Příznak | Pravděpodobná příčina | Řešení |
|---------|-----------------------|--------|
| Nebyl vytvořen žádný výstupní soubor | Cesta k výstupnímu adresáři je nesprávná nebo chybí oprávnění k zápisu | Ověřte, že `options.OutputWorkingDirectory` ukazuje na platnou složku a proces má právo zapisovat. |
| Log terminálu je prázdný | `TerminalOut` není nastaven nebo je později přepsán | Ujistěte se, že `options.TerminalOut = new OutputFileTerminal(...);` je vykonáno před `job.Run();`. |
| Úloha selže s chybou “file not found” | Vstupní adresář není nastaven správně | Zkontrolujte cestu předanou do `options.InputWorkingDirectory` a ověřte, že v ní existují soubory `.tex`. |
| Neočekávané znaky v logu | Nesoulad kódování mezi TeX enginem a souborovým systémem | Nastavte `options.Encoding = Encoding.UTF8` (nebo vhodné kódování) před spuštěním úlohy. |
