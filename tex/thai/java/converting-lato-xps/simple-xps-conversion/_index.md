# การแปลงขั้นตอนต่อขั้นตอน – Java สร้างใบแจ้งหนี้ที่พิมพ์ได้: จาก LaTeX ไปยังรูปแบบ XPS

## บทนำ

หากคุณต้องการ **java create printable invoices** จากแหล่งที่มาของ LaTeX ภายในแอปพลิเคชัน Java คุณมาถูกที่แล้ว ด้วย **Aspose.TeX for Java** คุณสามารถเรนเดอร์ไฟล์ LaTeX จัดการสูตรซับซ้อน และส่งออกไฟล์ XPS คุณภาพสูงที่พิมพ์ได้ตรงตามที่คุณคาดหวัง ในไม่กี่นาทีต่อไป เราจะพาคุณผ่านกระบวนการทำงานทั้งหมด อธิบายว่าทำไมวิธีนี้จึงเหมาะสมสำหรับการสร้างใบแจ้งหนี้ และแสดงวิธีปรับแต่งการแปลงสำหรับสายงานรายงานของคุณ คู่มือนี้แสดง **step by step conversion** ที่สมบูรณ์ซึ่งคุณสามารถคัดลอกและวางลงในโครงการ Java ใดก็ได้

## คำตอบอย่างรวดเร็ว
- **ควรใช้ไลบรารีอะไร?** Aspose.TeX for Java – เครื่องยนต์ pure‑Java ที่รองรับรูปแบบกว่า 50 แบบ.  
- **ใช้เวลานานเท่าไหร่ในการทำงาน?** ประมาณ 10‑15 นาทีสำหรับ pipeline ใบแจ้งหนี้พื้นฐาน.  
- **ข้อกำหนดเบื้องต้นคืออะไร?** JDK 8+, Aspose.TeX JAR, และ IDE (Eclipse/IntelliJ).  
- **ฉันสามารถเรนเดอร์สูตรซับซ้อนได้หรือไม่?** ใช่ – เครื่องยนต์รองรับแพคเกจคณิตศาสตร์ AMS อย่างเต็มที่.  
- **ต้องการใบอนุญาตสำหรับการผลิตหรือไม่?** ใช่, ใบอนุญาตเชิงพาณิชย์เป็นสิ่งจำเป็นสำหรับการใช้งานที่ไม่ใช่รุ่นทดลอง.  

## วิธีการ java create printable invoices ด้วยการแปลง LaTeX เป็น XPS

โหลดแหล่งที่มาของ LaTeX ของคุณ ตั้งค่าตัวเลือกการเรนเดอร์ และเรียกอุปกรณ์ XPS – นั่นคือการแปลงทั้งหมดในห้าขั้นตอนสั้น ๆ Aspose.TeX จัดการการฝังฟอนต์ การเรสเตอร์สูตร และการแปลงกราฟิกโดยอัตโนมัติ ดังนั้นคุณจะได้ใบแจ้งหนี้ XPS ที่พร้อมพิมพ์ภายในไม่กี่วินาทีสำหรับเอกสารทั่วไป 2 หน้า.

## การแปลงขั้นตอนต่อขั้นตอนคืออะไร?

การแปลงขั้นตอนต่อขั้นตอนจะแบ่งงานแปลง LaTeX‑to‑XPS ขนาดใหญ่เป็นการกระทำเล็ก ๆ ที่ทำซ้ำได้ ทำให้คุณสามารถตรวจสอบแต่ละขั้นตอนได้อย่างอิสระ โดยการแยกการโหลดไฟล์ การกำหนดค่าตัวเลือก การเริ่มต้นอุปกรณ์ และการรันขั้นสุดท้าย คุณสามารถค้นหาข้อผิดพลาดได้อย่างรวดเร็ว ปรับการตั้งค่าประสิทธิภาพ และใช้ pipeline เดียวกันสำหรับการประมวลผลเป็นชุดของใบแจ้งหนี้หลายสิบฉบับ.

## ทำไมต้องใช้ Aspose.TeX for Java ในการเรนเดอร์ LaTeX?

Aspose.TeX ให้เครื่องยนต์เรนเดอร์ pure‑Java ที่ขจัดความจำเป็นในการใช้ไบนารี LaTeX ภายนอก ลดความซับซ้อนของการปรับใช้ และรับประกันผลลัพธ์ที่สม่ำเสมอบน Windows, macOS, และ Linux. รองรับ **50+** รูปแบบอินพุตและเอาต์พุต ประมวลผลเอกสารหลายร้อยหน้าโดยไม่ต้องโหลดไฟล์ทั้งหมดเข้าสู่หน่วยความจำ และให้การควบคุมละเอียดเหนือการย่อยฟอนต์และการเรสเตอร์.

## ข้อกำหนดเบื้องต้น

- Java Development Kit (JDK) 8 หรือใหม่กว่า ติดตั้งแล้ว.  
- ไลบรารี Aspose.TeX for Java (ดาวน์โหลดจากหน้า [Aspose.TeX Java download page](https://releases.aspose.com/tex/java/)).  
- IDE หรือเครื่องมือสร้าง (Maven/Gradle) เพื่อเพิ่ม JAR ไปยัง classpath ของโครงการของคุณ.  

## นำเข้าแพ็กเกจ

บล็อก `import` นำคลาสที่จำเป็นเข้าสู่ขอบเขต การรักษารายการนี้ให้ตรงตามเดิมจะทำให้โค้ดคอมไพล์ได้โดยไม่มีการแก้ไข.

```java
package com.aspose.tex.LaTeXXpsConversionSimplest;

import java.io.ByteArrayInputStream;
import java.io.IOException;
import java.util.Calendar;
import java.util.GregorianCalendar;

import com.aspose.tex.InputFileSystemDirectory;
import com.aspose.tex.Interaction;
import com.aspose.tex.OutputFileSystemDirectory;
import com.aspose.tex.TeXConfig;
import com.aspose.tex.TeXJob;
import com.aspose.tex.TeXOptions;
import com.aspose.tex.rendering.XpsDevice;
import com.aspose.tex.rendering.XpsSaveOptions;

import util.Utils;
```

ตอนนี้เราจะเดินผ่านขั้นตอนการแปลงโดยอธิบายวัตถุประสงค์ของแต่ละส่วนของโค้ด.

## ขั้นตอนที่ 1: ตั้งค่าไดเรกทอรีอินพุตและเอาต์พุต

คุณต้องบอก Aspose.TeX ว่าจะหาไฟล์ต้นฉบับ `.ltx` ที่ไหนและจะเขียนไฟล์ XPS ที่ได้ลงที่ไหน.

```java
options.setInputWorkingDirectory(new InputFileSystemDirectory("Your Input Directory"));
options.setOutputWorkingDirectory(new OutputFileSystemDirectory("Your Output Directory"));
```

*Tip:* แทนที่ `"Your Input Directory"` และ `"Your Output Directory"` ด้วยเส้นทางแบบ absolute หรือ relative ที่มีอยู่บนเครื่องของคุณ.

## ขั้นตอนที่ 2: กำหนดค่า TeX Options

ตัวเลือกเหล่านี้ควบคุมพฤติกรรมของเอนจิน LaTeX ระหว่างการแปลง ปรับให้ตรงกับความต้องการของเอกสารของคุณ.  
`TeXOptions` คือคลาสที่เก็บการตั้งค่าเหล่านี้.

```java
options.setInteraction(Interaction.NonstopMode);
options.setDateTime(new GregorianCalendar(2022, Calendar.DECEMBER, 18).getTime());
options.ignoreMissingPackages(true);
options.noLigatures(true);
options.repeat(true);
```

- **Interaction mode** – `NonstopMode` ทำการประมวลผลต่อไปแม้จะพบข้อผิดพลาด.  
- **DateTime** – ตั้งค่าวันที่ที่ปรากฏในหน้าชื่อเอกสาร.  
- **ignoreMissingPackages** – ป้องกันไม่ให้งานล้มเหลวเมื่อไม่พบแพคเกจ.  
- **noLigatures** – ปิดการใช้ลิการเจอร์แบบพิมพ์ถ้าคุณต้องการอักขระธรรมดา.  
- **repeat** – อนุญาตให้เครื่องยนต์ทำงานซ้ำเพื่ออ้างอิงข้าม.

## ขั้นตอนที่ 3: เริ่มต้น XpsSaveOptions

`XpsSaveOptions` กำหนดพารามิเตอร์การส่งออก XPS เช่น การบีบอัดและขนาดหน้า.

```java
options.setSaveOptions(new XpsSaveOptions());
```

## ขั้นตอนที่ 4: ปรับแต่ง XpsSaveOptions

ปรับจูนวิธีที่เอาต์พุต XPS จัดการสูตร, กราฟิก, และฟอนต์.  
`rasterizeFormulas` กำหนดว่าจะแปลงสูตรเป็นภาพหรือไม่.

```java
options.getSaveOptions().rasterizeFormulas(true);
options.getSaveOptions().rasterizeIncludedGraphics(true);
options.getSaveOptions().subsetFonts(true);
```

- **rasterizeFormulas** – แปลงนิพจน์คณิตศาสตร์เป็นภาพ เพื่อให้แสดงผลได้อย่างถูกต้องบน XPS viewer ใดก็ได้.  
- **rasterizeIncludedGraphics** – บังคับให้กราฟิกที่ฝังอยู่ถูกเรสเตอร์ ซึ่งอาจช่วยเพิ่มความเข้ากันได้.  
- **subsetFonts** – ฝังเฉพาะ glyph ที่ใช้ในเอกสาร ลดขนาดไฟล์ได้ถึง 40 % สำหรับใบแจ้งหนี้ทั่วไป.

## ขั้นตอนที่ 5: เรียกใช้การแปลง LaTeX เป็น XPS

`XpsDevice` เขียนเนื้อหาที่เรนเดอร์ลงไฟล์ XPS.  
`TeXJob` เชื่อมไฟล์อินพุต, อุปกรณ์เอาต์พุต (`XpsDevice`), และตัวเลือกทั้งหมดที่คุณกำหนดไว้.

```java
new TeXJob("Your Input Directory" + "sample.ltx", new XpsDevice(), options).run();
```

หลังจากเรียก `run()` เสร็จสิ้น คุณจะพบ `sample.xps` ในไดเรกทอรีเอาต์พุตที่ระบุไว้ก่อนหน้านี้ ไฟล์ XPS นี้สามารถส่งตรงไปยังเครื่องพิมพ์หรือฝังในชุดใบแจ้งหนี้ PDF ได้.

## ตัวอย่างเพิ่มเติม

### ใช้ InputStream (แปลงสตริง LaTeX โดยตรง)

หากคุณต้องการป้อนแหล่งที่มาของ LaTeX จากหน่วยความจำแทนไฟล์ ให้ห่อแหล่งที่มาด้วย `ByteArrayInputStream`.

```java
new TeXJob(new ByteArrayInputStream(
    "\\documentclass{article} \\begin{document} Hello, World! \\end{document}".getBytes("ASCII")),
    new XpsDevice(), options).run();
```

### ใช้ Main Input Terminal (ให้ Aspose.TeX ค้นหาไฟล์โดยอัตโนมัติ)

เมื่อคุณไม่จำเป็นต้องระบุเส้นทางไฟล์อย่างชัดเจน คุณสามารถพึ่งพา terminal อินพุตเริ่มต้นได้.

```java
new TeXJob(new XpsDevice(), options).run();
```
