# Capture Console Output C# – Override Job Name and Write Terminal Output to Disk (C#)

## บทนำ

ในคู่มือแบบขั้นตอนนี้ คุณจะได้เรียนรู้ **วิธีบันทึกผลลัพธ์คอนโซล C#** เมื่อทำงานกับ Aspose.TeX for .NET โดยการแทนที่ชื่องานและกำหนดให้ผลลัพธ์เทอร์มินัลเขียนลงไฟล์ คุณจะได้ควบคุมการประมวลผล TeX อย่างเต็มที่ — เหมาะสำหรับการสร้างอัตโนมัติ, สถานการณ์ CI/CD หรือสถานการณ์ใด ๆ ที่คุณต้องการบันทึกสตรีมคอนโซลเพื่อการวิเคราะห์ในภายหลัง.

## คำตอบสั้น

- **“capture console output C#” หมายถึงอะไร?** มันทำการเปลี่ยนเส้นทางสตรีมเทอร์มินัลมาตรฐานที่สร้างโดย Aspose.TeX ไปยังไฟล์ที่คุณระบุ.  
- **ทำไมต้องแทนที่ชื่องาน?** การแทนที่ทำให้ได้ชื่อไฟล์ที่คาดเดาได้และหลีกเลี่ยงการชนกันเมื่อรันหลายงานพร้อมกัน.  
- **คลาส Aspose.TeX ใดที่เขียนผลลัพธ์?** `OutputFileTerminal` (ใช้ผ่าน `options.TerminalOut`).  
- **ฉันสามารถตั้งค่าโฟลเดอร์อินพุต TeX แบบกำหนดเองได้หรือไม่?** ใช่ — ใช้ `options.InputWorkingDirectory` เพื่อ **ตั้งค่าไดเรกทอรีอินพุต TeX**.  
- **วิธีนี้เข้ากันได้กับ .NET Core หรือไม่?** แน่นอน; API เดียวกันทำงานบน .NET Framework และ .NET Core.

## “capture console output C#” คืออะไรในบริบทของ Aspose.TeX?

การบันทึกผลลัพธ์คอนโซลหมายถึงการนำทุกอย่างที่ปกติจะแสดงในหน้าต่างเทอร์มินัล (ข้อความบันทึก, คำเตือน, รายละเอียดการคอมไพล์) แล้วเขียนลงไฟล์ที่คงอยู่ นี่มีประโยชน์อย่างยิ่งสำหรับการดีบักโครงการ TeX ขนาดใหญ่หรือการรวมการประมวลผล TeX เข้าในกระบวนการทำงานอัตโนมัติ.

## ทำไมต้องแทนที่ชื่องานและเขียนผลลัพธ์เทอร์มินัลลงไฟล์?

- **ชื่อไฟล์ที่คาดเดาได้:** การแทนที่ชื่องานทำให้คุณกำหนดชื่อฐานสำหรับไฟล์ที่สร้างขึ้น ทำให้สคริปต์หลังการประมวลผลเขียนง่ายขึ้น.  
- **ความสามารถในการตรวจสอบ:** การเก็บบันทึกเทอร์มินัลให้คุณมีเส้นทางการตรวจสอบที่สมบูรณ์ของกระบวนการคอมไพล์ TeX.  
- **การทำงานแบบขนาน:** เมื่อรันหลายงานพร้อมกัน ชื่องานที่ไม่ซ้ำกันจะป้องกันการชนกันของไฟล์.

## ข้อกำหนดเบื้องต้น

ก่อนที่เราจะเริ่ม โปรดตรวจสอบว่าคุณมีสิ่งต่อไปนี้:

- Aspose.TeX for .NET Library: ตรวจสอบว่าคุณได้ติดตั้งไลบรารี Aspose.TeX for .NET แล้ว คุณสามารถดาวน์โหลดได้จาก [Aspose.TeX website](https://releases.aspose.com/tex/net/).
- .NET Development Environment: มีสภาพแวดล้อมการพัฒนา .NET ที่ทำงานได้ รวมถึง IDE เช่น Visual Studio.
- Basic C# Knowledge: มีความคุ้นเคยกับพื้นฐานของภาษาโปรแกรม C#.
- Input and Output Directories: เตรียมไดเรกทอรีอินพุตและเอาต์พุตสำหรับไฟล์ TeX ของคุณ.

## นำเข้า Namespaces

ในโค้ด C# ของคุณ ให้แน่ใจว่าได้รวม Namespaces ที่จำเป็นเพื่อเข้าถึงฟังก์ชันของ Aspose.TeX:

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## ขั้นตอนที่ 1: สร้าง Conversion Options

แรก เราจะสร้างอินสแตนซ์ `TeXOptions` ที่บอก Aspose.TeX ว่าเรากำลังทำงานในสถานการณ์แอปพลิเคชันคอนโซล.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

สร้าง `TeXOptions` ด้วย `ConsoleAppOptions` โดยระบุ `ObjectTeX` เป็น `TeXConfig`.

## ขั้นตอนที่ 2: ระบุชื่องาน (แทนที่ค่าเริ่มต้น)

การแทนที่ชื่องานทำให้เราควบคุมชื่อฐานของทุก artefact ที่สร้างขึ้น.

```csharp
options.JobName = "overridden-job-name";
```

ระบุชื่องานเพื่อแทนที่ชื่อเริ่มต้น.

## ขั้นตอนที่ 3: ตั้งค่าไดเรกทอรีอินพุต TeX

บอก Aspose.TeX ว่าไฟล์ต้นฉบับ `.tex` ของคุณอยู่ที่ไหน ขั้นตอนนี้คือ **ตั้งค่าไดเรกทอรีอินพุต tex**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

ตั้งค่าไดเรกทอรีทำงานอินพุตสำหรับไฟล์ TeX ของคุณ.

## ขั้นตอนที่ 4: ระบุไดเรกทอรีทำงานเอาต์พุต

กำหนดตำแหน่งที่ไฟล์ที่ประมวลผลและบันทึกคอนโซลที่บันทึกไว้จะถูกบันทึก.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

กำหนดไดเรกทอรีทำงานเอาต์พุตเพื่อบันทึกไฟล์ TeX ที่ประมวลผลแล้ว.

## ขั้นตอนที่ 5: เขียนผลลัพธ์เทอร์มินัลลงไฟล์

ตอนนี้เราจะกำหนดสตรีมคอนโซลให้เขียนลงไฟล์จริงในไดเรกทอรีเอาต์พุต นี่เป็นการทำตามความต้องการ **เขียนผลลัพธ์เทอร์มินัลลงไฟล์**.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

กำหนดให้ผลลัพธ์เทอร์มินัลเขียนลงไฟล์ในไดเรกทอรีเอาต์พุต.

## ขั้นตอนที่ 6: รันงาน

สุดท้าย เราจะสร้าง `TeXJob` ด้วยชื่องานที่แทนที่, อุปกรณ์เอาต์พุต XPS, และตัวเลือกที่เราตั้งค่า การรันงานจะสร้างไฟล์ XPS และบันทึกบันทึกคอนโซล.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

สร้าง `TeXJob` โดยระบุชื่องาน, อุปกรณ์เอาต์พุต (`XpsDevice`), และตัวเลือก สุดท้าย รันงาน.

## ปัญหาทั่วไปและการแก้ไขปัญหา

| อาการ | สาเหตุที่เป็นไปได้ | วิธีแก้ |
|---------|--------------|-----|
| ไม่มีไฟล์เอาต์พุตสร้างขึ้น | เส้นทางไดเรกทอรีเอาต์พุตไม่ถูกต้องหรือไม่มีสิทธิ์การเขียน | ตรวจสอบว่า `options.OutputWorkingDirectory` ชี้ไปยังโฟลเดอร์ที่ถูกต้องและกระบวนการมีสิทธิ์การเขียน |
| บันทึกเทอร์มินัลว่างเปล่า | `TerminalOut` ไม่ได้ตั้งค่า หรือถูกเขียนทับภายหลัง | ตรวจสอบว่า `options.TerminalOut = new OutputFileTerminal(...);` ถูกเรียกก่อน `job.Run();` |
| งานล้มเหลวด้วยข้อความ “file not found” | ไดเรกทอรีอินพุตไม่ได้ตั้งค่าอย่างถูกต้อง | ตรวจสอบเส้นทางที่ส่งให้ `InputFileSystemDirectory` อีกครั้งและตรวจสอบว่าไฟล์ `.tex` มีอยู่ในนั้น |
