# Πώς να καταγράψετε την έξοδο της κονσόλας C# – Παράκαμψη ονόματος εργασίας & εγγραφή στο δίσκο

Σε αυτό το βήμα‑βήμα tutorial θα ανακαλύψετε **πώς να καταγράψετε την έξοδο της κονσόλας C#** όταν χρησιμοποιείτε το Aspose.TeX για .NET. Με την παράκαμψη του ονόματος εργασίας και τη δρομολόγηση της ροής τερματικού σε αρχείο, αποκτάτε καθοριστική ονομασία αρχείων, εύκολη συγκέντρωση καταγραφών και μια σταθερή βάση για CI/CD pipelines ή σενάρια επεξεργασίας παρτίδας.

## Σύντομες Απαντήσεις
- **Τι σημαίνει “capture console output C#”;** Ανακατευθύνει τη στάνταρ ροή τερματικού που δημιουργείται από το Aspose.TeX σε ένα αρχείο που καθορίζετε, διατηρώντας κάθε καταγραφή, προειδοποίηση και μήνυμα κατάστασης.  
- **Γιατί να παρακάμψετε το όνομα εργασίας;** Η παράκαμψη εγγυάται προβλέψιμα ονόματα αρχείων, εξαλείφει συγκρούσεις ονομάτων σε παράλληλες εργασίες και απλοποιεί τα scripts που ακολουθούν.  
- **Ποια κλάση του Aspose.TeX γράφει την έξοδο;** `OutputFileTerminal`, που ανατίθεται μέσω `options.TerminalOut`. *OutputFileTerminal είναι μια κλάση που γράφει μηνύματα τερματικού σε αρχείο.*  
- **Μπορώ να ορίσω προσαρμοσμένο φάκελο εισόδου TeX;** Ναι—χρησιμοποιήστε `options.InputWorkingDirectory` για να δείξετε στα πηγαία `.tex` αρχεία σας.  
- **Είναι αυτή η προσέγγιση συμβατή με .NET Core;** Απόλυτα· το ίδιο API λειτουργεί σε .NET Framework, .NET Core και .NET 5+.

## Τι είναι το “capture console output C#” στο πλαίσιο του Aspose.TeX;
Η καταγραφή της εξόδου της κονσόλας σημαίνει τη λήψη όλων όσων θα εμφανίζονταν κανονικά στο παράθυρο τερματικού (μηνύματα καταγραφής, προειδοποιήσεις, λεπτομέρειες μεταγλώττισης) και η εγγραφή τους σε ένα μόνιμο αρχείο. Αυτό είναι ιδιαίτερα χρήσιμο για εντοπισμό σφαλμάτων σε μεγάλα έργα TeX ή για ενσωμάτωση της επεξεργασίας TeX σε αυτοματοποιημένες ροές εργασίας.

## Γιατί να παρακάμψετε το όνομα εργασίας και να γράψετε την έξοδο τερματικού σε αρχείο;
Η παράκαμψη του ονόματος εργασίας σας επιτρέπει να αποφασίσετε το βασικό όνομα για όλα τα παραγόμενα τεχνητά, ενώ η εγγραφή της καταγραφής τερματικού σας παρέχει πλήρη ίχνος ελέγχου. Προβλέψιμα ονόματα αρχείων κάνουν τα scripts μετά-επεξεργασίας τετριμμένα, και μια αποθηκευμένη καταγραφή σας επιτρέπει να εξετάσετε τα βήματα μεταγλώττισης μετά το γεγονός, κάτι που είναι κρίσιμο για αυτοματοποιημένες κατασκευές και περιβάλλοντα παράλληλης εκτέλεσης.

Η αρχιτεκτονική ροής της βιβλιοθήκης μειώνει τη μέγιστη χρήση μνήμης έως **70 %** σε σύγκριση με απλές προσεγγίσεις βασισμένες σε αρχεία.

## Προαπαιτούμενα

- **Aspose.TeX for .NET** – κατεβάστε την τελευταία έκδοση από το [Aspose.TeX website](https://releases.aspose.com/tex/net/).  
- **.NET development environment** – Visual Studio 2022, Rider ή οποιοδήποτε IDE που υποστηρίζει .NET 6+.  
- **Basic C# knowledge** – θα πρέπει να είστε άνετοι με namespaces, objects και διαδρομές συστήματος αρχείων.  
- **Input and output directories** – δημιουργήστε δύο φακέλους στο δίσκο: έναν για τα πηγαία `.tex` αρχεία και έναν για τα παραγόμενα τεχνητά και καταγραφές.

## Εισαγωγή Namespaces

Προσθέστε τις απαιτούμενες οδηγίες `using` στην αρχή του αρχείου C# ώστε ο μεταγλωττιστής να μπορεί να εντοπίσει τους τύπους του Aspose.TeX.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Βήμα 1: Δημιουργία Επιλογών Μετατροπής

Η κλάση `TeXOptions` περιλαμβάνει όλες τις ρυθμίσεις διαμόρφωσης για μια εργασία μετατροπής TeX. Σας επιτρέπει να καθορίσετε τη συμπεριφορά της console‑app, τους φακέλους εισόδου και τις συσκευές εξόδου.

`ConsoleAppOptions` διαμορφώνει τη συμπεριφορά της console‑application για το Aspose.TeX.  
`TeXConfig` ορίζει τον τύπο της διαμόρφωσης επεξεργασίας TeX.

Δημιουργήστε μια παρουσία `TeXOptions` με `ConsoleAppOptions`, καθορίζοντας το `ObjectTeX` ως το `TeXConfig`. Αυτό ενημερώνει το Aspose.TeX ότι εκτελείτε σε σενάριο console‑application.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Βήμα 2: Καθορισμός Ονόματος Εργασίας (Παράκαμψη Προεπιλογής)

Η κλάση `TeXJob` χρησιμοποιεί το όνομα εργασίας ως βάση για κάθε παραγόμενο αρχείο. Η παράκαμψη του αποτρέπει τη βιβλιοθήκη από το να δημιουργεί τυχαία ονόματα όπως `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

Ορίστε `options.JobName = "MyCustomJob"` (ή οποιοδήποτε αναγνωριστικό προτιμάτε) πριν δημιουργήσετε την παρουσία της εργασίας.

## Βήμα 3: Ορισμός Φακέλου Εισόδου TeX

Η ιδιότητα `InputWorkingDirectory` ενημερώνει το Aspose.TeX πού να εντοπίσει τα πηγαία `.tex` αρχεία σας. Αυτό είναι το βήμα **ορισμού φακέλου εισόδου tex**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Παρέχετε την απόλυτη ή σχετική διαδρομή προς το φάκελο που περιέχει τις πηγές TeX, π.χ., `@"C:\\TeX\\Sources"`.

## Βήμα 4: Ορισμός Φακέλου Εξόδου

Ορίστε το φάκελο όπου θα αποθηκευτούν τα επεξεργασμένα αρχεία και η καταγεγραμμένη καταγραφή κονσόλας. Το Aspose.TeX θα δημιουργήσει υπο‑φακέλους όπως απαιτείται, αλλά ο ριζικός φάκελος πρέπει να υπάρχει και να είναι εγγράψιμος.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Αντιστοιχίστε `options.OutputWorkingDirectory = @"C:\TeX\Results"` (βεβαιωθείτε ότι ο φάκελος έχει δικαιώματα εγγραφής).

## Βήμα 5: Εγγραφή Εξόδου Τερματικού σε Αρχείο

`OutputFileTerminal` είναι η συγκεκριμένη υλοποίηση που γράφει κάθε μήνυμα κονσόλας σε αρχείο. Αναθέτοντάς το στο `options.TerminalOut`, αντικαθιστάτε τη προεπιλεγμένη ροή κονσόλας στη μνήμη.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Παράδειγμα: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Βήμα 6: Εκτέλεση της Εργασίας

Η κλάση `TeXJob` οργανώνει τη μετατροπή. Παρέχετε το παρακαμφθέν όνομα εργασίας, τη συσκευή εξόδου (`XpsDevice` σε αυτό το παράδειγμα) και το πλήρως διαμορφωμένο αντικείμενο επιλογών. Το `XpsDevice` δημιουργεί έξοδο XPS από την πηγή TeX.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Μετά την κλήση του `job.Run();`, το Aspose.TeX δημιουργεί το αρχείο XPS και γράφει ολόκληρη την καταγραφή τερματικού στο `console.log` στον φάκελο εξόδου.

## Συχνά Προβλήματα & Επίλυση

| Σύμπτωμα | Πιθανή Αιτία | Διόρθωση |
|----------|--------------|----------|
| Δεν δημιουργήθηκε αρχείο εξόδου | Η διαδρομή του φακέλου εξόδου είναι λανθασμένη ή λείπουν δικαιώματα εγγραφής | Επαληθεύστε ότι το `options.OutputWorkingDirectory` δείχνει σε έγκυρο φάκελο και ότι η διαδικασία έχει δικαιώματα εγγραφής. |
| Η καταγραφή τερματικού είναι κενή | `TerminalOut` δεν έχει οριστεί ή έχει αντικατασταθεί αργότερα | Βεβαιωθείτε ότι το `options.TerminalOut = new OutputFileTerminal(...);` εκτελείται πριν το `job.Run();`. |
| Η εργασία αποτυγχάνει με “file not found” | Ο φάκελος εισόδου δεν έχει οριστεί σωστά | Ελέγξτε ξανά τη διαδρομή που περνιέται στο `options.InputWorkingDirectory` και ότι τα αρχεία `.tex` υπάρχουν εκεί. |
| Απρόσμενοι χαρακτήρες στην καταγραφή | Ασυμφωνία κωδικοποίησης μεταξύ του μηχανήματος TeX και του συστήματος αρχείων | Ορίστε `options.Encoding = Encoding.UTF8` (ή την κατάλληλη κωδικοποίηση) πριν την εκτέλεση της εργασίας. |
