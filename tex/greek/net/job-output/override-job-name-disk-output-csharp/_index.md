# Καταγραφή Εξόδου Κονσόλας C# – Παράκαμψη Ονόματος Εργασίας και Εγγραφή Εξόδου Τερματικού σε Αρχείο (C#)

## Εισαγωγή

Σε αυτόν τον οδηγό βήμα‑βήμα θα μάθετε **πώς να καταγράψετε την έξοδο κονσόλας C#** όταν εργάζεστε με το Aspose.TeX για .NET. Με την παράκαμψη του ονόματος εργασίας και την κατεύθυνση της εξόδου τερματικού σε αρχείο, αποκτάτε πλήρη έλεγχο πάνω στις διαδικασίες επεξεργασίας TeX—ιδανικό για αυτοματοποιημένες κατασκευές, σενάρια CI/CD ή οποιαδήποτε κατάσταση όπου χρειάζεται να καταγράψετε τη ροή της κονσόλας για μεταγενέστερη ανάλυση.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει “capture console output C#”;** Ανακατευθύνει τη στάνταρ ροή τερματικού που παράγεται από το Aspose.TeX σε ένα αρχείο που καθορίζετε.  
- **Γιατί να παρακάμψω το όνομα εργασίας;** Η παράκαμψη εξασφαλίζει προβλέψιμα ονόματα αρχείων και αποτρέπει συγκρούσεις όταν εκτελούνται πολλαπλές εργασίες.  
- **Ποια κλάση του Aspose.TeX γράφει την έξοδο;** `OutputFileTerminal` (χρησιμοποιείται μέσω `options.TerminalOut`).  
- **Μπορώ να ορίσω προσαρμοσμένο φάκελο εισόδου TeX;** Ναι—χρησιμοποιήστε `options.InputWorkingDirectory` για **ορισμό φακέλου εισόδου TeX**.  
- **Είναι αυτή η προσέγγιση συμβατή με .NET Core;** Απόλυτα· το ίδιο API λειτουργεί σε .NET Framework και .NET Core.

## Τι είναι το “capture console output C#” στο πλαίσιο του Aspose.TeX;
Η καταγραφή της εξόδου κονσόλας σημαίνει τη λήψη όλων όσων θα εμφανίζονταν κανονικά στο παράθυρο τερματικού (μηνύματα καταγραφής, προειδοποιήσεις, λεπτομέρειες μεταγλώττισης) και η εγγραφή τους σε ένα μόνιμο αρχείο. Αυτό είναι ιδιαίτερα χρήσιμο για εντοπισμό σφαλμάτων σε μεγάλα έργα TeX ή για ενσωμάτωση της επεξεργασίας TeX σε αυτοματοποιημένες ροές εργασίας.

## Γιατί να παρακάμψετε το όνομα εργασίας και να γράψετε την έξοδο τερματικού σε αρχείο;
- **Προβλέψιμα ονόματα αρχείων:** Η παράκαμψη του ονόματος εργασίας σας επιτρέπει να καθορίσετε τη βασική ονομασία για τα παραγόμενα αρχεία, κάνοντας τα μεταγενέστερα σενάρια επεξεργασίας πιο εύκολα.  
- **Αξιοπιστία ελέγχου:** Η αποθήκευση του αρχείου καταγραφής τερματικού παρέχει πλήρη ιχνηλασιμότητα της διαδικασίας μεταγλώττισης TeX.  
- **Παράλληλη εκτέλεση:** Όταν εκτελούνται πολλές εργασίες ταυτόχρονα, μοναδικά ονόματα εργασίας αποτρέπουν συγκρούσεις αρχείων.

## Εισαγωγή Namespaces

Στον κώδικα C#, βεβαιωθείτε ότι έχετε συμπεριλάβει τα απαραίτητα namespaces για πρόσβαση στις λειτουργίες του Aspose.TeX:

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Βήμα 1: Δημιουργία Επιλογών Μετατροπής

Πρώτα, δημιουργούμε μια παρουσία `TeXOptions` που ενημερώνει το Aspose.TeX ότι τρέχουμε σε σενάριο εφαρμογής κονσόλας.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

Δημιουργήστε `TeXOptions` με `ConsoleAppOptions`, καθορίζοντας `ObjectTeX` ως το `TeXConfig`.

## Βήμα 2: Καθορισμός Ονόματος Εργασίας (Παράκαμψη Προεπιλογής)

Η παράκαμψη του ονόματος εργασίας μας δίνει έλεγχο πάνω στη βασική ονομασία όλων των παραγόμενων τεχνουργημάτων.

```csharp
options.JobName = "overridden-job-name";
```

Καθορίστε το όνομα εργασίας για να παρακάμψετε το προεπιλεγμένο όνομα.

## Βήμα 3: Ορισμός Φακέλου Εισόδου TeX

Ενημερώστε το Aspose.TeX πού να βρει τα αρχεία `.tex` πηγής σας. Αυτό είναι το βήμα **ορισμού φακέλου εισόδου tex**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Ορίστε τον φάκελο εργασίας εισόδου για τα αρχεία TeX σας.

## Βήμα 4: Καθορισμός Φακέλου Εργασίας Εξόδου

Ορίστε πού θα αποθηκευτούν τα επεξεργασμένα αρχεία και η καταγεγραμμένη έξοδος κονσόλας.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Ορίστε τον φάκελο εργασίας εξόδου για την αποθήκευση των επεξεργασμένων αρχείων TeX.

## Βήμα 5: Εγγραφή Εξόδου Τερματικού σε Αρχείο

Τώρα κατευθύνουμε τη ροή κονσόλας σε ένα φυσικό αρχείο στον φάκελο εξόδου. Αυτό υλοποιεί την απαίτηση **εγγραφής εξόδου τερματικού σε αρχείο**.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Διαμορφώστε την έξοδο τερματικού ώστε να γράφεται σε αρχείο στον φάκελο εξόδου.

## Βήμα 6: Εκτέλεση της Εργασίας

Τέλος, δημιουργούμε ένα `TeXJob` με το παρακάμψιμο όνομα εργασίας, τη συσκευή εξόδου XPS και τις επιλογές που διαμορφώσαμε. Η εκτέλεση της εργασίας θα δημιουργήσει το αρχείο XPS και θα καταγράψει το αρχείο καταγραφής κονσόλας.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Δημιουργήστε ένα `TeXJob`, καθορίζοντας όνομα εργασίας, συσκευή εξόδου (`XpsDevice`) και επιλογές. Τέλος, εκτελέστε την εργασία.

## Συνηθισμένα Προβλήματα & Επίλυση

| Συμπτωμα | Πιθανή Αιτία | Διόρθωση |
|----------|--------------|----------|
| Δεν δημιουργείται αρχείο εξόδου | Η διαδρομή του φακέλου εξόδου είναι λανθασμένη ή λείπουν δικαιώματα εγγραφής | Επαληθεύστε ότι το `options.OutputWorkingDirectory` δείχνει σε έγκυρο φάκελο και ότι η διεργασία έχει δικαιώματα εγγραφής. |
| Το αρχείο καταγραφής τερματικού είναι κενό | Δεν έχει οριστεί ή έχει αντικατασταθεί το `TerminalOut` | Βεβαιωθείτε ότι το `options.TerminalOut = new OutputFileTerminal(...);` εκτελείται πριν από το `job.Run();`. |
| Η εργασία αποτυγχάνει με “file not found” | Ο φάκελος εισόδου δεν έχει οριστεί σωστά | Ελέγξτε ξανά τη διαδρομή που περνάτε στο `InputFileSystemDirectory` και βεβαιωθείτε ότι τα αρχεία `.tex` υπάρχουν εκεί. |
