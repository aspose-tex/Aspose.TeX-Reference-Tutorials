## Συνηθισμένα Προβλήματα & Επίλυση
- **Missing fonts** – Βεβαιωθείτε ότι οι απαιτούμενες γραμματοσειρές LaTeX είναι διαθέσιμες στο φάκελο γραμματοσειρών του Aspose.TeX ή ενσωματώστε τες χειροκίνητα.  
- **Large documents cause OutOfMemoryError** – Μεταβείτε σε επεξεργασία βασισμένη σε ροή και αυξήστε το μέγεθος της heap του JVM εάν χρειάζεται.  
- **Incorrect image resolution** – Επαληθεύστε τη ρύθμιση DPI στο αντικείμενο `RenderOptions`; η προεπιλογή είναι 96 dpi.  

## Συχνές Ερωτήσεις

**Q: Μπορώ να δημιουργήσω διανυσματικές εικόνες (SVG) αντί για μορφές raster;**  
A: Ναι, ορίστε τη μορφή εξόδου σε `Svg` στις επιλογές απόδοσης για να λάβετε κλιμακώσιμα διανυσματικά γραφικά.

**Q: Πώς να διαχειριστώ αρχεία TeX που περιλαμβάνουν εξωτερικά πακέτα;**  
A: Τοποθετήστε τα απαιτούμενα αρχεία `.sty` στον ίδιο φάκελο εισόδου ή προσθέστε τις διαδρομές τους στο `PackageSearchPath` του `TeXProcessor`.

**Q: Είναι δυνατόν να επεξεργαστώ TeX από ένα `InputStream` χωρίς εγγραφή στο δίσκο;**  
A: Απολύτως – το Aspose.TeX υποστηρίζει πλήρως εισόδους βασισμένες σε ροή, κάτι που είναι ιδανικό για web services και micro‑services.

**Q: Ποιο μοντέλο αδειοδότησης χρησιμοποιεί το Aspose.TeX;**  
A: Προσφέρει μια δια βίου άδεια με προαιρετικές ανανεώσεις υποστήριξης· διατίθεται επίσης δωρεάν άδεια αξιολόγησης.

**Q: Υποστηρίζει η βιβλιοθήκη χαρακτήρες Unicode σε κώδικα TeX;**  
A: Ναι, το Aspose.TeX διαχειρίζεται αρχεία TeX κωδικοποιημένα σε UTF‑8 αμέσως.

**Q: Μπορώ επίσης **convert latex to pdf** με την ίδια βιβλιοθήκη;**  
A: Ναι – ορίζοντας τη μορφή εξόδου σε `Pdf` μπορείτε να μετατρέψετε απευθείας έγγραφα LaTeX σε PDF.

**Q: Πώς να **render latex as svg** για κλιμάκωση φιλική προς το web;**  
A: Χρησιμοποιήστε τη μορφή εξόδου `Svg` και προαιρετικά προσαρμόστε τις ρυθμίσεις `DpiX`/`DpiY` για πιο λεπτομερή απεικόνιση.

**Q: Ποιος είναι ο καλύτερος τρόπος για **generate png from latex** για ανάγκες υψηλής ανάλυσης;**  
A: Αυξήστε την ιδιότητα `Resolution` στο `RenderOptions` (π.χ., 300 dpi) πριν καλέσετε το `process`.
