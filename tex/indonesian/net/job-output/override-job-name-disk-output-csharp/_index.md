# Cara Menangkap Output Konsol C# – Ganti Nama Pekerjaan & Tulis ke Disk

Dalam tutorial langkah‑demi‑langkah ini Anda akan menemukan **cara menangkap output konsol C#** saat menggunakan Aspose.TeX untuk .NET. Dengan mengganti nama pekerjaan dan mengarahkan aliran terminal ke sebuah file, Anda mendapatkan penamaan file yang deterministik, agregasi log yang mudah, dan fondasi yang kuat untuk pipeline CI/CD atau skenario pemrosesan batch.

## Jawaban Cepat
- **Apa arti “capture console output C#”?** Ini mengalihkan aliran terminal standar yang dihasilkan oleh Aspose.TeX ke file yang Anda tentukan, menyimpan setiap log, peringatan, dan pesan status.  
- **Mengapa mengganti nama pekerjaan?** Penggantian menjamin nama file yang dapat diprediksi, menghilangkan bentrok penamaan dalam pekerjaan paralel, dan menyederhanakan skrip hilir.  
- **Kelas Aspose.TeX mana yang menulis output?** `OutputFileTerminal`, ditetapkan melalui `options.TerminalOut`. *OutputFileTerminal adalah kelas yang menulis pesan terminal ke sebuah file.*  
- **Bisakah saya mengatur folder input TeX khusus?** Ya—gunakan `options.InputWorkingDirectory` untuk menunjuk ke file `.tex` sumber Anda.  
- **Apakah pendekatan ini kompatibel dengan .NET Core?** Tentu; API yang sama berfungsi pada .NET Framework, .NET Core, dan .NET 5+.

## Apa itu “capture console output C#” dalam konteks Aspose.TeX?
Menangkap output konsol berarti mengambil semua yang biasanya muncul di jendela terminal (pesan log, peringatan, detail kompilasi) dan menuliskannya ke file yang persisten. Ini sangat berguna untuk men-debug proyek TeX besar atau mengintegrasikan pemrosesan TeX ke dalam alur kerja otomatis.

## Mengapa mengganti nama pekerjaan dan menulis output terminal ke file?
Mengganti nama pekerjaan memungkinkan Anda menentukan nama dasar untuk semua artefak yang dihasilkan, sementara menulis log terminal memberi Anda jejak audit lengkap. Nama file yang dapat diprediksi membuat skrip pasca‑pemrosesan menjadi sederhana, dan log yang disimpan memungkinkan Anda meninjau langkah kompilasi setelahnya, yang penting untuk build otomatis dan lingkungan eksekusi paralel.

## Manfaat Terukur Menggunakan Aspose.TeX
Aspose.TeX mendukung **lebih dari 30 format input dan output**—termasuk `.tex`, `.pdf`, `.xps`, `.html`, dan tipe gambar umum—dan dapat memproses **dokumen hingga 500 halaman** tanpa memuat seluruh file ke memori. Arsitektur streaming perpustakaan mengurangi penggunaan memori puncak hingga **70 %** dibandingkan dengan pendekatan berbasis file yang naïf.

- **Pengetahuan dasar C#** – Anda harus nyaman dengan namespace, objek, dan jalur sistem file.  
- **Direktori input dan output** – buat dua folder di disk: satu untuk file `.tex` sumber dan satu lagi untuk artefak dan log yang dihasilkan.

## Impor Namespace

Tambahkan direktif `using` yang diperlukan di bagian atas file C# Anda agar kompiler dapat menemukan tipe Aspose.TeX.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Langkah 1: Buat Opsi Konversi

Kelas `TeXOptions` menyatukan semua pengaturan konfigurasi untuk pekerjaan konversi TeX. Ini memungkinkan Anda menentukan perilaku aplikasi konsol, direktori input, dan perangkat output.

`ConsoleAppOptions` mengonfigurasi perilaku aplikasi konsol untuk Aspose.TeX.  
`TeXConfig` mendefinisikan jenis konfigurasi pemrosesan TeX.

Buat instance `TeXOptions` dengan `ConsoleAppOptions`, menentukan `ObjectTeX` sebagai `TeXConfig`. Ini memberi tahu Aspose.TeX bahwa Anda menjalankan dalam skenario aplikasi konsol.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Langkah 2: Tentukan Nama Pekerjaan (Ganti Default)

Kelas `TeXJob` menggunakan nama pekerjaan sebagai dasar untuk setiap file yang dihasilkan. Mengganti nama tersebut mencegah perpustakaan menghasilkan nama acak seperti `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

Setel `options.JobName = "MyCustomJob"` (atau identifier apa pun yang Anda inginkan) sebelum membuat instance pekerjaan.

## Langkah 3: Atur Direktori Input TeX

Properti `InputWorkingDirectory` memberi tahu Aspose.TeX di mana menemukan file `.tex` sumber Anda. Ini adalah langkah **menetapkan direktori input tex**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Berikan jalur absolut atau relatif ke folder yang berisi sumber TeX Anda, misalnya, `@"C:\TeX\Sources"`.

## Langkah 4: Tentukan Direktori Kerja Output

Tentukan folder tempat file yang diproses dan log konsol yang ditangkap akan disimpan. Aspose.TeX akan membuat sub‑folder sesuai kebutuhan, tetapi folder akar harus ada dan dapat ditulisi.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Tetapkan `options.OutputWorkingDirectory = @"C:\TeX\Results"` (pastikan direktori memiliki izin menulis).

## Langkah 5: Tulis Output Terminal ke File

`OutputFileTerminal` adalah implementasi konkret yang menulis setiap pesan konsol ke sebuah file. Dengan menetapkannya ke `options.TerminalOut`, Anda menggantikan aliran konsol default dalam memori.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Contoh: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Langkah 6: Jalankan Pekerjaan

Kelas `TeXJob` mengatur konversi. Anda menyediakan nama pekerjaan yang telah diganti, perangkat output (`XpsDevice` dalam contoh ini), dan objek opsi yang telah dikonfigurasi sepenuhnya. `XpsDevice` menghasilkan output XPS dari sumber TeX.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Setelah memanggil `job.Run();`, Aspose.TeX menghasilkan file XPS dan menulis seluruh log terminal ke `console.log` di folder output.

## Masalah Umum & Pemecahan Masalah

| Gejala | Penyebab Kemungkinan | Perbaikan |
|---------|----------------------|-----------|
| Tidak ada file output yang dibuat | Jalur direktori output tidak benar atau tidak memiliki izin menulis | Verifikasi bahwa `options.OutputWorkingDirectory` mengarah ke folder yang valid dan proses memiliki akses menulis. |
| Log terminal kosong | `TerminalOut` tidak diset atau ditimpa kemudian | Pastikan `options.TerminalOut = new OutputFileTerminal(...);` dijalankan sebelum `job.Run();`. |
| Pekerjaan gagal dengan “file tidak ditemukan” | Direktori input tidak diatur dengan benar | Periksa kembali jalur yang diberikan ke `options.InputWorkingDirectory` dan pastikan file `.tex` ada di sana. |
| Karakter tak terduga dalam log | Ketidaksesuaian encoding antara mesin TeX dan sistem file | Setel `options.Encoding = Encoding.UTF8` (atau encoding yang sesuai) sebelum menjalankan pekerjaan. |
