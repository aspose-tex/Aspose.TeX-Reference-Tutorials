# Cómo capturar la salida de consola C# – Sobrescribir el nombre del trabajo y escribir en disco

En este tutorial paso a paso descubrirá **cómo capturar la salida de consola C#** al usar Aspose.TeX para .NET. Al sobrescribir el nombre del trabajo y dirigir el flujo del terminal a un archivo, obtendrá nombres de archivo determinísticos, una fácil agregación de registros y una base sólida para canalizaciones CI/CD o escenarios de procesamiento por lotes.

## Respuestas rápidas
- **¿Qué significa “capture console output C#”?** Redirige el flujo estándar del terminal generado por Aspose.TeX a un archivo que usted especifique, preservando cada registro, advertencia y mensaje de estado.  
- **¿Por qué sobrescribir el nombre del trabajo?** Sobrescribir garantiza nombres de archivo predecibles, elimina colisiones de nombres en trabajos paralelos y simplifica los scripts posteriores.  
- **¿Qué clase de Aspose.TeX escribe la salida?** `OutputFileTerminal`, asignada a través de `options.TerminalOut`. *OutputFileTerminal es una clase que escribe los mensajes del terminal a un archivo.*  
- **¿Puedo establecer una carpeta de entrada TeX personalizada?** Sí—use `options.InputWorkingDirectory` para apuntar a sus archivos fuente `.tex`.  
- **¿Es este enfoque compatible con .NET Core?** Absolutamente; la misma API funciona en .NET Framework, .NET Core y .NET 5+.

## Qué es “capture console output C#” en el contexto de Aspose.TeX?
Capturar la salida de consola significa tomar todo lo que normalmente aparecería en la ventana del terminal (mensajes de registro, advertencias, detalles de compilación) y escribirlo en un archivo persistente. Esto es especialmente útil para depurar proyectos TeX grandes o integrar el procesamiento TeX en flujos de trabajo automatizados.

## Por qué sobrescribir el nombre del trabajo y escribir la salida del terminal a un archivo?
Sobrescribir el nombre del trabajo le permite decidir el nombre base para todos los artefactos generados, mientras que escribir el registro del terminal le brinda una pista de auditoría completa. Los nombres de archivo predecibles hacen que los scripts de posprocesamiento sean triviales, y un registro guardado le permite revisar los pasos de compilación después del hecho, lo cual es crítico para compilaciones automatizadas y entornos de ejecución paralela.

## Beneficios cuantificados de usar Aspose.TeX
Aspose.TeX admite **más de 30 formatos de entrada y salida**—incluyendo `.tex`, `.pdf`, `.xps`, `.html` y tipos de imagen comunes—y puede procesar **documentos de hasta 500 páginas** sin cargar todo el archivo en memoria. La arquitectura de streaming de la biblioteca reduce el uso máximo de memoria hasta en **un 70 %** en comparación con enfoques ingenuos basados en archivos.

## Requisitos previos

- **Aspose.TeX for .NET** – descargue la última versión desde el [sitio web de Aspose.TeX](https://releases.aspose.com/tex/net/).  
- **Entorno de desarrollo .NET** – Visual Studio 2022, Rider, o cualquier IDE que soporte .NET 6+.  
- **Conocimientos básicos de C#** – debe estar cómodo con namespaces, objetos y rutas del sistema de archivos.  
- **Directorios de entrada y salida** – cree dos carpetas en el disco: una para los archivos fuente `.tex` y otra para los artefactos generados y los registros.

## Importar espacios de nombres

Agregue las directivas `using` requeridas al inicio de su archivo C# para que el compilador pueda localizar los tipos de Aspose.TeX.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Paso 1: Crear opciones de conversión

La clase `TeXOptions` encapsula todas las configuraciones para un trabajo de conversión TeX. Le permite especificar el comportamiento de la aplicación de consola, los directorios de entrada y los dispositivos de salida.

`ConsoleAppOptions` configura el comportamiento de la aplicación de consola para Aspose.TeX.  
`TeXConfig` define el tipo de configuración de procesamiento TeX.

Cree una instancia de `TeXOptions` con `ConsoleAppOptions`, especificando `ObjectTeX` como el `TeXConfig`. Esto indica a Aspose.TeX que está ejecutándose en un escenario de aplicación de consola.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Paso 2: Especificar el nombre del trabajo (sobrescribir el predeterminado)

La clase `TeXJob` usa el nombre del trabajo como base para cada archivo generado. Sobrescribirlo evita que la biblioteca genere nombres aleatorios como `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

Establezca `options.JobName = "MyCustomJob"` (o cualquier identificador que prefiera) antes de crear la instancia del trabajo.

## Paso 3: Establecer el directorio de entrada TeX

La propiedad `InputWorkingDirectory` indica a Aspose.TeX dónde localizar sus archivos fuente `.tex`. Este es el paso de **establecer el directorio de entrada tex**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Proporcione la ruta absoluta o relativa a la carpeta que contiene sus fuentes TeX, por ejemplo, `@"C:\\TeX\\Sources"`.

## Paso 4: Especificar el directorio de trabajo de salida

Defina la carpeta donde se guardarán los archivos procesados y el registro de consola capturado. Aspose.TeX creará subcarpetas según sea necesario, pero la carpeta raíz debe existir y ser escribible.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Asigne `options.OutputWorkingDirectory = @"C:\\TeX\\Results"` (asegúrese de que la carpeta tenga permisos de escritura).

## Paso 5: Escribir la salida del terminal a un archivo

`OutputFileTerminal` es la implementación concreta que escribe cada mensaje de consola a un archivo. Al asignarla a `options.TerminalOut`, reemplaza el flujo de consola en memoria predeterminado.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Ejemplo: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Paso 6: Ejecutar el trabajo

La clase `TeXJob` orquesta la conversión. Usted proporciona el nombre de trabajo sobrescrito, el dispositivo de salida (`XpsDevice` en este ejemplo) y el objeto de opciones completamente configurado. `XpsDevice` genera salida XPS a partir de la fuente TeX.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Después de llamar a `job.Run();`, Aspose.TeX genera el archivo XPS y escribe todo el registro del terminal en `console.log` en la carpeta de salida.

## Problemas comunes y solución de problemas

| Síntoma | Causa probable | Solución |
|---------|----------------|----------|
| No se creó el archivo de salida | La ruta del directorio de salida es incorrecta o faltan permisos de escritura | Verifique que `options.OutputWorkingDirectory` apunte a una carpeta válida y que el proceso tenga acceso de escritura. |
| El registro del terminal está vacío | `TerminalOut` no está configurado o se sobrescribe más tarde | Asegúrese de que `options.TerminalOut = new OutputFileTerminal(...);` se ejecute antes de `job.Run();`. |
| El trabajo falla con “file not found” | El directorio de entrada no está configurado correctamente | Verifique nuevamente la ruta pasada a `options.InputWorkingDirectory` y que los archivos `.tex` existan allí. |
| Caracteres inesperados en el registro | Desajuste de codificación entre el motor TeX y el sistema de archivos | Establezca `options.Encoding = Encoding.UTF8` (o la codificación apropiada) antes de ejecutar el trabajo. |
