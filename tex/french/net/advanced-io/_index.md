# Entrée et sortie avancées d'Aspose.TeX

## Introduction

Aspose.TeX for .NET est une révolution dans l'intégration transparente de TeX, offrant aux développeurs une bibliothèque robuste pour améliorer le traitement des documents. Dans cet article, nous explorons des tutoriels avancés axés sur la spécification des répertoires d'entrée et la maîtrise des flux, des images et de l'entrée terminale en C#. **Si vous cherchez comment définir l'entrée** pour vos projets TeX, vous êtes au bon endroit.

## Réponses rapides
- **À quoi fait référence « how to set input » ?**  
  Cela signifie configurer la bibliothèque pour localiser correctement les fichiers source TeX, les images et les données de flux.
- **Quelle classe API gère les répertoires d'entrée ?**  
  `TeXInputOptions` vous permet de définir le dossier de base et les chemins de recherche supplémentaires.
- **Puis-je ajouter des images directement depuis un flux ?**  
  Oui, en utilisant la méthode `AddImage` sur les options d'entrée (voir « how to add images » ci‑dessous).
- **L'entrée terminale est‑elle prise en charge ?**  
  Absolument – vous pouvez fournir du code LaTeX via un `MemoryStream` ou l'entrée standard.
- **Ai‑je besoin d’une licence pour une utilisation en production ?**  
  Une licence Aspose.TeX valide est requise pour les déploiements non‑évaluation.

## Comment définir l'entrée dans Aspose.TeX pour .NET
Configurer l'environnement d'entrée est la base de tout flux de travail Aspose.TeX. Vous trouverez ci‑dessous les trois scénarios les plus courants :

### Comment ajouter des images avec Aspose.TeX
Les images sont souvent référencées dans les fichiers TeX à l'aide de chemins relatifs. En configurant les options d'entrée, vous pouvez indiquer au moteur un dossier contenant toutes les graphiques requises, ou fournir directement un flux d'image. Cela élimine le besoin de copier des fichiers dans votre projet.

### Comment traiter les flux dans Aspose.TeX
Lorsque vous travaillez avec du contenu LaTeX généré dynamiquement (par exemple, créer un rapport à la volée), vous souhaiterez fournir la source sous forme de flux plutôt que de fichier physique. Aspose.TeX accepte tout objet `Stream`, vous permettant de vous intégrer aux services web, bases de données ou générateurs en mémoire.

### Comment définir le répertoire d'entrée
1. **Créer une instance de `TeXInputOptions`.**  
   Cet objet contient tous les paramètres liés aux chemins.  
2. **Spécifier le répertoire de base** où se trouve votre fichier `.tex` principal.  
3. **Ajouter des chemins de recherche supplémentaires** pour les sous‑dossiers contenant des images ou des fichiers auxiliaires.  
4. **Passer les options configurées** au `TeXProcessor` avant le rendu.  

Ces étapes garantissent que la bibliothèque peut localiser chaque ressource sans copie manuelle de fichiers, rendant votre processus de construction plus propre et plus maintenable.

## Questions fréquentes

**Q : Puis‑je changer le répertoire d'entrée à l'exécution ?**  
R : Oui, vous pouvez instancier un nouvel objet `TeXInputOptions` et le passer au processeur chaque fois que vous devez reconfigurer le chemin.

**Q : Comment ajouter des images stockées dans une base de données ?**  
R : Récupérez l'image sous forme de `byte[]`, encapsulez‑la dans un `MemoryStream`, et utilisez la méthode `AddImage` sur les options d'entrée (voir « how to add images »).

**Q : Est‑il possible de traiter du code LaTeX reçu d'une API web sans enregistrer de fichier ?**  
R : Absolument. Alimentez la chaîne LaTeX brute dans un `MemoryStream` et définissez‑la comme flux source pour le processeur (voir « how to process streams »).

**Q : Dois‑je appeler des méthodes de nettoyage après le traitement ?**  
R : Libérez tous les flux que vous créez et, si vous traitez de gros documents, envisagez d'appeler `TeXProcessor.Cleanup()` pour libérer les ressources.
