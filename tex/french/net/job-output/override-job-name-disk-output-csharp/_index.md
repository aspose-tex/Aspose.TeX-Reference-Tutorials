# Comment capturer la sortie console C# – Remplacer le nom du travail et écrire sur le disque

## Réponses rapides
- **Que signifie « capture console output C# » ?** Elle redirige le flux terminal standard généré par Aspose.TeX vers un fichier que vous spécifiez, en conservant chaque journal, avertissement et message d’état.  
- **Pourquoi remplacer le nom du travail ?** Le remplacement garantit des noms de fichiers prévisibles, élimine les collisions de noms dans les travaux parallèles et simplifie les scripts en aval.  
- **Quelle classe Aspose.TeX écrit la sortie ?** `OutputFileTerminal`, assignée via `options.TerminalOut`. *OutputFileTerminal est une classe qui écrit les messages du terminal dans un fichier.*  
- **Puis‑je définir un dossier d’entrée TeX personnalisé ?** Oui — utilisez `options.InputWorkingDirectory` pour pointer vers vos fichiers source `.tex`.  
- **Cette approche est‑elle compatible avec .NET Core ?** Absolument ; la même API fonctionne sur .NET Framework, .NET Core et .NET 5+.

## Qu'est‑ce que « capture console output C# » dans le contexte d'Aspose.TeX ?
Capturer la sortie console signifie prendre tout ce qui apparaîtrait normalement dans la fenêtre du terminal (messages de journal, avertissements, détails de compilation) et l’écrire dans un fichier persistant. Cela est particulièrement utile pour déboguer de grands projets TeX ou intégrer le traitement TeX dans des flux de travail automatisés.

## Pourquoi remplacer le nom du travail et écrire la sortie du terminal dans un fichier ?
Remplacer le nom du travail vous permet de choisir le nom de base pour tous les artefacts générés, tandis que l’écriture du journal du terminal vous fournit une piste d’audit complète. Des noms de fichiers prévisibles simplifient les scripts de post‑traitement, et un journal sauvegardé vous permet de revoir les étapes de compilation après coup, ce qui est crucial pour les builds automatisés et les environnements d’exécution parallèles.

## Avantages quantifiés de l’utilisation d'Aspose.TeX
Aspose.TeX prend en charge **plus de 30 formats d’entrée et de sortie** — y compris `.tex`, `.pdf`, `.xps`, `.html` et les types d’image courants — et peut traiter **des documents jusqu’à 500 pages** sans charger le fichier complet en mémoire. L’architecture de streaming de la bibliothèque réduit l’utilisation maximale de mémoire jusqu’à **70 %** comparée aux approches naïves basées sur des fichiers.

- **Répertoires d’entrée et de sortie** – créez deux dossiers sur le disque : un pour les fichiers source `.tex` et un autre pour les artefacts générés et les journaux.

## Importer les espaces de noms
Ajoutez les directives `using` requises en haut de votre fichier C# afin que le compilateur puisse localiser les types Aspose.TeX.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Étape 1 : créer les options de conversion
La classe `TeXOptions` encapsule tous les paramètres de configuration d’un travail de conversion TeX. Elle vous permet de spécifier le comportement de l’application console, les répertoires d’entrée et les périphériques de sortie.

`ConsoleAppOptions` configure le comportement de l’application console pour Aspose.TeX.  
`TeXConfig` définit le type de configuration du traitement TeX.

Créez une instance `TeXOptions` avec `ConsoleAppOptions`, en spécifiant `ObjectTeX` comme `TeXConfig`. Cela indique à Aspose.TeX que vous exécutez dans un scénario d’application console.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Étape 2 : spécifier le nom du travail (remplacer la valeur par défaut)
La classe `TeXJob` utilise le nom du travail comme base pour chaque fichier généré. Le remplacer empêche la bibliothèque de générer des noms aléatoires comme `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

Définissez `options.JobName = "MyCustomJob"` (ou tout identifiant de votre choix) avant de créer l’instance du travail.

## Étape 3 : définir le répertoire d'entrée TeX
La propriété `InputWorkingDirectory` indique à Aspose.TeX où localiser vos fichiers source `.tex`. Il s’agit de l’étape **définir le répertoire d’entrée tex**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Fournissez le chemin absolu ou relatif du dossier contenant vos sources TeX, par ex., `@"C:\TeX\Sources"`.

## Étape 4 : spécifier le répertoire de travail de sortie
Définissez le dossier où les fichiers traités et le journal console capturé seront enregistrés. Aspose.TeX créera les sous‑dossiers nécessaires, mais le dossier racine doit exister et être accessible en écriture.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Attribuez `options.OutputWorkingDirectory = @"C:\TeX\Results"` (assurez‑vous que le répertoire possède les permissions d’écriture).

## Étape 5 : écrire la sortie du terminal dans un fichier
`OutputFileTerminal` est l’implémentation concrète qui écrit chaque message console dans un fichier. En l’assignant à `options.TerminalOut`, vous remplacez le flux console en mémoire par défaut.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Exemple : `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Étape 6 : exécuter le travail
La classe `TeXJob` orchestre la conversion. Vous fournissez le nom du travail remplacé, le périphérique de sortie (`XpsDevice` dans cet exemple) et l’objet d’options entièrement configuré. `XpsDevice` génère une sortie XPS à partir de la source TeX.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Après l’appel à `job.Run();`, Aspose.TeX génère le fichier XPS et écrit l’intégralité du journal du terminal dans `console.log` dans le dossier de sortie.

## Problèmes courants et dépannage
| Symptôme | Cause probable | Solution |
|----------|----------------|----------|
| Aucun fichier de sortie créé | Le chemin du répertoire de sortie est incorrect ou les permissions d'écriture sont manquantes | Vérifiez que `options.OutputWorkingDirectory` pointe vers un dossier valide et que le processus dispose des droits d’écriture. |
| Le journal du terminal est vide | `TerminalOut` non défini ou écrasé plus tard | Assurez‑vous que `options.TerminalOut = new OutputFileTerminal(...);` est exécuté avant `job.Run();`. |
| Le travail échoue avec « fichier non trouvé » | Répertoire d'entrée mal configuré | Revérifiez le chemin passé à `options.InputWorkingDirectory` et que les fichiers `.tex` existent à cet emplacement. |
| Caractères inattendus dans le journal | Mauvaise correspondance d'encodage entre le moteur TeX et le système de fichiers | Définissez `options.Encoding = Encoding.UTF8` (ou l’encodage approprié) avant d’exécuter le travail. |
