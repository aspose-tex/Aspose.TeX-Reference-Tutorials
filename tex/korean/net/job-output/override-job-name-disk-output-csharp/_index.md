# 콘솔 출력 캡처 방법 C# – 작업 이름 재정의 및 디스크에 기록

이 단계별 튜토리얼에서는 Aspose.TeX for .NET을 사용할 때 **콘솔 출력 캡처 방법 C#**을 알아봅니다. 작업 이름을 재정의하고 터미널 스트림을 파일로 라우팅하면 결정적인 파일 명명, 쉬운 로그 집계, CI/CD 파이프라인 또는 배치 처리 시나리오를 위한 견고한 기반을 얻을 수 있습니다.

## 빠른 답변
- **“콘솔 출력 캡처 C#”가 의미하는 바는?** Aspose.TeX에서 생성되는 표준 터미널 스트림을 지정한 파일로 리디렉션하여 모든 로그, 경고 및 상태 메시지를 보존합니다.  
- **왜 작업 이름을 재정의하나요?** 재정의하면 예측 가능한 파일 이름을 보장하고, 병렬 작업에서 이름 충돌을 방지하며, 다운스트림 스크립트를 단순화합니다.  
- **어떤 Aspose.TeX 클래스가 출력을 기록합니까?** `OutputFileTerminal`, `options.TerminalOut`을 통해 할당됩니다. *OutputFileTerminal는 터미널 메시지를 파일에 기록하는 클래스입니다.*  
- **사용자 지정 TeX 입력 폴더를 설정할 수 있나요?** 예—`options.InputWorkingDirectory`를 사용하여 소스 `.tex` 파일을 지정합니다.  
- **이 방법이 .NET Core와 호환되나요?** 물론입니다; 동일한 API가 .NET Framework, .NET Core 및 .NET 5+에서 작동합니다.

## Aspose.TeX 컨텍스트에서 “콘솔 출력 캡처 C#”란 무엇인가요?
콘솔 출력을 캡처한다는 것은 터미널 창에 일반적으로 표시되는 모든 내용(로그 메시지, 경고, 컴파일 세부 정보)을 가져와 지속적인 파일에 기록하는 것을 의미합니다. 이는 대규모 TeX 프로젝트 디버깅이나 TeX 처리를 자동화된 워크플로에 통합할 때 특히 유용합니다.

## 작업 이름을 재정의하고 터미널 출력을 파일에 기록하는 이유는?
작업 이름을 재정의하면 생성된 모든 아티팩트의 기본 이름을 결정할 수 있고, 터미널 로그를 기록하면 완전한 감사 추적을 제공하게 됩니다. 예측 가능한 파일 이름은 후처리 스크립트를 간단하게 만들며, 저장된 로그를 통해 컴파일 단계를 사후에 검토할 수 있어 자동 빌드 및 병렬 실행 환경에 필수적입니다.

## Aspose.TeX 사용의 정량적 이점
Aspose.TeX는 **30개 이상의 입력 및 출력 형식**을 지원합니다—`.tex`, `.pdf`, `.xps`, `.html` 및 일반 이미지 형식을 포함—그리고 전체 파일을 메모리에 로드하지 않고 **최대 500페이지** 문서를 처리할 수 있습니다. 라이브러리의 스트리밍 아키텍처는 기존 파일 기반 접근 방식에 비해 피크 메모리 사용량을 **70 %**까지 감소시킵니다.

## 네임스페이스 가져오기

컴파일러가 Aspose.TeX 타입을 찾을 수 있도록 C# 파일 상단에 필요한 `using` 지시문을 추가하십시오.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## 단계 1: 변환 옵션 생성

`TeXOptions` 클래스는 TeX 변환 작업에 대한 모든 구성 설정을 캡슐화합니다. 콘솔 앱 동작, 입력 디렉터리 및 출력 장치를 지정할 수 있습니다.

`ConsoleAppOptions`는 Aspose.TeX의 콘솔 애플리케이션 동작을 구성합니다.  
`TeXConfig`는 TeX 처리 구성 유형을 정의합니다.

`ConsoleAppOptions`와 함께 `TeXOptions` 인스턴스를 생성하고, `TeXConfig`로 `ObjectTeX`를 지정합니다. 이는 Aspose.TeX에 콘솔 애플리케이션 시나리오에서 실행 중임을 알립니다.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## 단계 2: 작업 이름 지정 (기본값 재정의)

`TeXJob` 클래스는 작업 이름을 모든 생성 파일의 기본으로 사용합니다. 이를 재정의하면 라이브러리가 `job_12345`와 같은 무작위 이름을 생성하는 것을 방지합니다.

```csharp
options.JobName = "overridden-job-name";
```

작업 인스턴스를 만들기 전에 `options.JobName = "MyCustomJob"`(또는 원하는 식별자)로 설정하십시오.

## 단계 3: TeX 입력 디렉터리 설정

`InputWorkingDirectory` 속성은 Aspose.TeX에 소스 `.tex` 파일을 찾을 위치를 알려줍니다. 이것이 **TeX 입력 디렉터리 설정** 단계입니다.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

TeX 소스가 들어 있는 폴더의 절대 경로나 상대 경로를 제공하십시오. 예: `@"C:\\TeX\\Sources"`.

## 단계 4: 출력 작업 디렉터리 지정

처리된 파일과 캡처된 콘솔 로그가 저장될 폴더를 정의합니다. Aspose.TeX는 필요에 따라 하위 폴더를 생성하지만, 루트 폴더는 존재하고 쓰기 가능해야 합니다.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

`options.OutputWorkingDirectory = @"C:\TeX\Results"`(디렉터리에 쓰기 권한이 있는지 확인)으로 할당하십시오.

## 단계 5: 터미널 출력을 파일에 기록

`OutputFileTerminal`는 모든 콘솔 메시지를 파일에 기록하는 구체적인 구현입니다. 이를 `options.TerminalOut`에 할당하면 기본 메모리 내 콘솔 스트림을 대체합니다.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

예시: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## 단계 6: 작업 실행

`TeXJob` 클래스는 변환을 조정합니다. 재정의된 작업 이름, 출력 장치(이 예에서는 `XpsDevice`), 완전히 구성된 옵션 객체를 제공합니다. `XpsDevice`는 TeX 소스로부터 XPS 출력을 생성합니다.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

`job.Run();`을 호출한 후, Aspose.TeX는 XPS 파일을 생성하고 출력 폴더의 `console.log`에 전체 터미널 로그를 기록합니다.

## 일반적인 문제 및 해결 방법

| 증상 | 가능한 원인 | 해결 방법 |
|---------|--------------|-----|
| 출력 파일이 생성되지 않음 | 출력 디렉터리 경로가 잘못되었거나 쓰기 권한이 없습니다 | `options.OutputWorkingDirectory`가 유효한 폴더를 가리키고 프로세스에 쓰기 권한이 있는지 확인하십시오. |
| 터미널 로그가 비어 있음 | `TerminalOut`이 설정되지 않았거나 나중에 덮어쓰기되었습니다 | `options.TerminalOut = new OutputFileTerminal(...);`가 `job.Run();` 전에 실행되는지 확인하십시오. |
| 작업이 “파일을 찾을 수 없음” 오류로 실패 | 입력 디렉터리가 올바르게 설정되지 않음 | `options.InputWorkingDirectory`에 전달된 경로와 해당 폴더에 `.tex` 파일이 존재하는지 다시 확인하십시오. |
| 로그에 예상치 못한 문자 | TeX 엔진과 파일 시스템 간 인코딩 불일치 | 작업 실행 전에 `options.Encoding = Encoding.UTF8`(또는 적절한 인코딩)으로 설정하십시오. |
