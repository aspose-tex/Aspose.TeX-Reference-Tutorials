# 단계별 변환 – Java 인쇄 가능한 청구서 생성: LaTeX에서 XPS 형식으로

## 소개

LaTeX 소스에서 Java 애플리케이션 내부에서 **java create printable invoices**가 필요하다면, 올바른 튜토리얼에 오신 것입니다. **Aspose.TeX for Java**를 사용하면 LaTeX 파일을 렌더링하고 복잡한 수식을 처리하며, 기대한 대로 정확히 인쇄되는 고품질 XPS 파일을 출력할 수 있습니다. 다음 몇 분 동안 전체 워크플로를 단계별로 살펴보고, 이 방법이 청구서 생성에 왜 이상적인지 설명하며, 자체 보고 파이프라인에 맞게 변환을 사용자 정의하는 방법을 보여드립니다. 이 가이드는 어떤 Java 프로젝트에도 복사‑붙여넣기 할 수 있는 완전한 **step by step conversion**을 보여줍니다.

## 빠른 답변
- **어떤 라이브러리를 사용해야 하나요?** Aspose.TeX for Java – 50개 이상의 형식을 지원하는 순수 Java 엔진.  
- **구현에 얼마나 걸리나요?** 기본 청구서 파이프라인의 경우 대략 10‑15분 정도 소요됩니다.  
- **전제 조건은 무엇인가요?** JDK 8+, Aspose.TeX JAR, 그리고 IDE(Eclipse/IntelliJ).  
- **복잡한 수식을 렌더링할 수 있나요?** 예 – 엔진은 AMS 수학 패키지를 완전히 지원합니다.

## LaTeX를 XPS 변환으로 java create printable invoices 하는 방법?

LaTeX 소스를 로드하고, 렌더링 옵션을 구성한 뒤 XPS 디바이스를 호출하면 – 이것이 다섯 단계의 간결한 전체 변환 과정입니다. Aspose.TeX는 글꼴 포함, 수식 래스터화 및 그래픽 변환을 자동으로 처리하므로 일반적인 2페이지 문서의 경우 1초 미만에 인쇄 준비가 된 XPS 청구서를 얻을 수 있습니다.

## 단계별 변환이란 무엇인가요?

단계별 변환은 대규모 LaTeX‑to‑XPS 작업을 작고 반복 가능한 작업으로 나누어 각 단계를 독립적으로 검증할 수 있게 합니다. 파일 로드, 옵션 구성, 디바이스 초기화 및 최종 실행을 분리함으로써 오류를 빠르게 찾고, 성능 설정을 조정하며, 수십 개의 청구서를 배치 처리하기 위해 동일한 파이프라인을 재사용할 수 있습니다.

## 왜 Aspose.TeX for Java LaTeX 렌더링을 사용하나요?

## 패키지 가져오기

`import` 블록은 필수 클래스를 범위에 가져옵니다. 이 정확한 목록을 유지하면 코드를 수정 없이 컴파일할 수 있습니다.

```java
package com.aspose.tex.LaTeXXpsConversionSimplest;

import java.io.ByteArrayInputStream;
import java.io.IOException;
import java.util.Calendar;
import java.util.GregorianCalendar;

import com.aspose.tex.InputFileSystemDirectory;
import com.aspose.tex.Interaction;
import com.aspose.tex.OutputFileSystemDirectory;
import com.aspose.tex.TeXConfig;
import com.aspose.tex.TeXJob;
import com.aspose.tex.TeXOptions;
import com.aspose.tex.rendering.XpsDevice;
import com.aspose.tex.rendering.XpsSaveOptions;

import util.Utils;
```

이제 변환 단계를 살펴보면서 각 코드 조각의 목적을 설명하겠습니다.

## 단계 1: 입력 및 출력 디렉터리 설정

Aspose.TeX에 소스 `.ltx` 파일을 찾을 위치와 결과 XPS 파일을 쓸 위치를 알려야 합니다.

```java
options.setInputWorkingDirectory(new InputFileSystemDirectory("Your Input Directory"));
options.setOutputWorkingDirectory(new OutputFileSystemDirectory("Your Output Directory"));
```

*Tip:* `"Your Input Directory"`와 `"Your Output Directory"`를 머신에 존재하는 절대 경로나 상대 경로로 교체하십시오.

## 단계 2: TeX 옵션 구성

이 옵션들은 변환 중 LaTeX 엔진의 동작을 제어합니다. 문서 요구사항에 맞게 조정하십시오.  
TeXOptions는 이러한 구성 설정을 보유하는 클래스입니다.

```java
options.setInteraction(Interaction.NonstopMode);
options.setDateTime(new GregorianCalendar(2022, Calendar.DECEMBER, 18).getTime());
options.ignoreMissingPackages(true);
options.noLigatures(true);
options.repeat(true);
```

- **Interaction mode** – `NonstopMode`는 오류가 발생해도 처리를 계속합니다.  
- **DateTime** – 문서 표지에 표시될 날짜를 설정합니다.  
- **ignoreMissingPackages** – 패키지를 찾지 못해도 작업이 실패하지 않도록 합니다.  
- **noLigatures** – 일반 문자만 원한다면 활자 합자를 비활성화합니다.  
- **repeat** – 교차 참조를 위해 엔진이 재실행하도록 허용합니다.  

## 단계 3: XPS 저장 옵션 초기화

XpsSaveOptions는 압축 및 페이지 크기와 같은 XPS 출력 매개변수를 구성합니다.

```java
options.setSaveOptions(new XpsSaveOptions());
```

## 단계 4: XPS 저장 옵션 사용자 정의

XPS 출력이 수식, 그래픽 및 글꼴을 처리하는 방식을 미세 조정합니다.  
`rasterizeFormulas`는 수식을 이미지로 렌더링할지 여부를 결정합니다.

```java
options.getSaveOptions().rasterizeFormulas(true);
options.getSaveOptions().rasterizeIncludedGraphics(true);
options.getSaveOptions().subsetFonts(true);
```

- **rasterizeFormulas** – 수학 표현식을 이미지로 변환하여 모든 XPS 뷰어에서 올바르게 표시되도록 합니다.  
- **rasterizeIncludedGraphics** – 포함된 그래픽을 강제로 래스터화하여 호환성을 향상시킬 수 있습니다.  
- **subsetFonts** – 문서에 사용된 글리프만 포함시켜 일반 청구서의 경우 파일 크기를 최대 40 %까지 줄입니다.  

## 단계 5: LaTeX를 XPS 변환 실행

XpsDevice는 렌더링된 내용을 XPS 파일에 기록합니다.  
`TeXJob`은 입력 파일, 출력 디바이스(`XpsDevice`) 및 구성한 모든 옵션을 연결합니다.

```java
new TeXJob("Your Input Directory" + "sample.ltx", new XpsDevice(), options).run();
```

`run()` 호출이 완료되면, 앞서 지정한 출력 디렉터리에서 `sample.xps`를 찾을 수 있습니다. 이 XPS 파일은 프린터로 직접 전송하거나 PDF 청구서 번들에 포함시킬 수 있습니다.

## 추가 예제

### InputStream 사용 (LaTeX 문자열을 직접 변환)

파일 대신 메모리에서 LaTeX 소스를 제공하려면, 소스를 `ByteArrayInputStream`으로 감싸면 됩니다.

```java
new TeXJob(new ByteArrayInputStream(
    "\\documentclass{article} \\begin{document} Hello, World! \\end{document}".getBytes("ASCII")),
    new XpsDevice(), options).run();
```

### 메인 입력 터미널 사용 (Aspose.TeX가 파일을 자동으로 찾게 함)

파일 경로를 명시적으로 지정할 필요가 없을 때는 기본 입력 터미널에 의존할 수 있습니다.

```java
new TeXJob(new XpsDevice(), options).run();
```
