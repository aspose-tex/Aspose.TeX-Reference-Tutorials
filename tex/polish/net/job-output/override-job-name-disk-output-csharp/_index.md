# Jak przechwycić wyjście konsoli w C# – nadpisanie nazwy zadania i zapis na dysk

W tym samouczku krok po kroku odkryjesz **jak przechwycić wyjście konsoli w C#** przy użyciu Aspose.TeX dla .NET. Nadpisując nazwę zadania i kierując strumień terminala do pliku, uzyskasz deterministyczne nazewnictwo plików, łatwą agregację logów oraz solidną podstawę dla potoków CI/CD lub scenariuszy przetwarzania wsadowego.

## Szybkie odpowiedzi
- **Co oznacza „capture console output C#”?** Przekierowuje standardowy strumień terminala generowany przez Aspose.TeX do określonego przez Ciebie pliku, zachowując każdy log, ostrzeżenie i komunikat statusu.  
- **Dlaczego nadpisywać nazwę zadania?** Nadpisywanie zapewnia przewidywalne nazwy plików, eliminuje kolizje nazw w równoległych zadaniach i upraszcza skrypty downstream.  
- **Która klasa Aspose.TeX zapisuje wyjście?** `OutputFileTerminal`, przypisana przez `options.TerminalOut`. *OutputFileTerminal jest klasą, która zapisuje komunikaty terminala do pliku.*  
- **Czy mogę ustawić własny folder wejściowy TeX?** Tak — użyj `options.InputWorkingDirectory`, aby wskazać folder z Twoimi plikami źródłowymi `.tex`.  
- **Czy to podejście jest kompatybilne z .NET Core?** Absolutnie; to samo API działa na .NET Framework, .NET Core i .NET 5+.

## Co oznacza „capture console output C#” w kontekście Aspose.TeX?
Przechwytywanie wyjścia konsoli oznacza zebranie wszystkiego, co normalnie pojawia się w oknie terminala (wiadomości logów, ostrzeżenia, szczegóły kompilacji) i zapisanie tego do trwałego pliku. Jest to szczególnie przydatne przy debugowaniu dużych projektów TeX lub integrowaniu przetwarzania TeX w zautomatyzowanych przepływach pracy.

## Dlaczego nadpisywać nazwę zadania i zapisywać wyjście terminala do pliku?
Nadpisywanie nazwy zadania pozwala określić podstawową nazwę dla wszystkich generowanych artefaktów, a zapis logu terminala zapewnia pełny ślad audytu. Przewidywalne nazwy plików upraszczają skrypty post‑processingowe, a zapisany log umożliwia przeglądanie kroków kompilacji po ich wykonaniu, co jest kluczowe dla automatycznych buildów i środowisk równoległego wykonywania.

## Zmierzony korzyści z używania Aspose.TeX
Aspose.TeX obsługuje **ponad 30 formatów wejściowych i wyjściowych** — w tym `.tex`, `.pdf`, `.xps`, `.html` oraz popularne typy obrazów — i może przetwarzać **dokumenty do 500 stron** bez ładowania całego pliku do pamięci. Architektura strumieniowa biblioteki zmniejsza maksymalne zużycie pamięci o nawet **70 %** w porównaniu z naiwnymi podejściami opartymi na plikach.

## Importowanie przestrzeni nazw

Dodaj wymagane dyrektywy `using` na początku pliku C#, aby kompilator mógł odnaleźć typy Aspose.TeX.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Krok 1: Utwórz opcje konwersji

Klasa `TeXOptions` kapsułkuje wszystkie ustawienia konfiguracyjne dla zadania konwersji TeX. Pozwala określić zachowanie aplikacji konsolowej, katalogi wejściowe i urządzenia wyjściowe.

`ConsoleAppOptions` konfiguruje zachowanie aplikacji konsolowej dla Aspose.TeX.  
`TeXConfig` definiuje typ konfiguracji przetwarzania TeX.

Utwórz instancję `TeXOptions` z `ConsoleAppOptions`, określając `ObjectTeX` jako `TeXConfig`. To informuje Aspose.TeX, że uruchamiasz scenariusz aplikacji konsolowej.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Krok 2: Określ nazwę zadania (nadpisz domyślną)

Klasa `TeXJob` używa nazwy zadania jako podstawy dla każdego generowanego pliku. Nadpisanie jej zapobiega tworzeniu losowych nazw, takich jak `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

Ustaw `options.JobName = "MyCustomJob"` (lub dowolny identyfikator, który preferujesz) przed utworzeniem instancji zadania.

## Krok 3: Ustaw katalog wejściowy TeX

Właściwość `InputWorkingDirectory` informuje Aspose.TeX, gdzie znajduje się katalog z Twoimi plikami źródłowymi `.tex`. To jest krok **ustawiania katalogu wejściowego tex**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Podaj absolutną lub względną ścieżkę do folderu zawierającego źródła TeX, np. `@"C:\TeX\Sources"`.

## Krok 4: Określ katalog wyjściowy

Zdefiniuj folder, w którym zostaną zapisane przetworzone pliki i przechwycony log konsoli. Aspose.TeX utworzy podfoldery w razie potrzeby, ale folder główny musi istnieć i mieć prawa zapisu.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Ustaw `options.OutputWorkingDirectory = @"C:\TeX\Results"` (upewnij się, że katalog ma uprawnienia do zapisu).

## Krok 5: Zapisz wyjście terminala do pliku

`OutputFileTerminal` jest konkretną implementacją, która zapisuje każdą wiadomość konsoli do pliku. Przypisując ją do `options.TerminalOut`, zastępujesz domyślny strumień konsoli w pamięci.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Przykład: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Krok 6: Uruchom zadanie

Klasa `TeXJob` koordynuje konwersję. Dostarczasz nadpisaną nazwę zadania, urządzenie wyjściowe (`XpsDevice` w tym przykładzie) oraz w pełni skonfigurowany obiekt opcji. `XpsDevice` generuje wyjście XPS z źródła TeX.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Po wywołaniu `job.Run();`, Aspose.TeX generuje plik XPS i zapisuje cały log terminala do `console.log` w folderze wyjściowym.

## Typowe problemy i rozwiązywanie

| Objaw | Prawdopodobna przyczyna | Rozwiązanie |
|-------|--------------------------|-------------|
| Nie utworzono pliku wyjściowego | Ścieżka katalogu wyjściowego jest niepoprawna lub brakuje uprawnień do zapisu | Zweryfikuj, że `options.OutputWorkingDirectory` wskazuje na prawidłowy folder i proces ma dostęp do zapisu. |
| Log terminala jest pusty | `TerminalOut` nie jest ustawiony lub został nadpisany później | Upewnij się, że `options.TerminalOut = new OutputFileTerminal(...);` jest wykonane przed `job.Run();`. |
| Zadanie nie powiodło się z komunikatem „file not found” | Katalog wejściowy nie został poprawnie ustawiony | Sprawdź dokładnie ścieżkę przekazaną do `options.InputWorkingDirectory` oraz czy pliki `.tex` znajdują się w tym miejscu. |
| Nieoczekiwane znaki w logu | Niepasujące kodowanie między silnikiem TeX a systemem plików | Ustaw `options.Encoding = Encoding.UTF8` (lub odpowiednie kodowanie) przed uruchomieniem zadania. |
