Добавьте зависимость Maven/Gradle (или скачайте JAR) и импортируйте необходимые пространства имён.

### Шаг 4: Вызовите конверсию

Вызовите метод конверсии — Aspose.TeX читает входной TeX и записывает PDF напрямую в ваш поток. Процесс быстрый, потокобезопасный и полностью настраиваемый.

## Часто задаваемые вопросы

**Q: Можно ли использовать этот подход для генерации PDF из TeX на безсерверной платформе?**  
A: Да. Поскольку Aspose.TeX работает только с потоками, он идеально подходит для AWS Lambda, Azure Functions или Google Cloud Run, где запись на диск ограничена.

**Q: Поддерживает ли Aspose.TeX соответствие PDF/A для архивирования?**  
A: Абсолютно. Вы можете включить вывод PDF/A через класс `PdfSaveOptions`, продолжая использовать внешние потоки.

**Q: Как встроить пользовательские шрифты, которые не установлены на хост‑машине?**  
A: Поместите файлы шрифтов в ресурсы вашего приложения и укажите их с помощью `\setmainfont{MyFont}` после загрузки шрифта через `FontFactory.register()`.

**Q: Есть ли способ конвертировать только часть большого TeX‑документа?**  
A: Вы можете разбить источник на отдельные секции `InputStream` и конвертировать каждую независимо, затем при необходимости объединить полученные PDF.

**Q: Какие версии Java поддерживаются?**  
A: Aspose.TeX for Java поддерживает Java 8 до Java 21, включая все LTS‑версии.
