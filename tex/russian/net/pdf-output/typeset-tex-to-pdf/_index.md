**Прямой ответ:** Преобразование TeX в PDF означает взятие исходного файла TeX (или строки) и создание PDF‑документа высокого качества, точно воспроизводящего оригинальную разметку, уравнения и типографику. Aspose.TeX выполняет весь конвейер компиляции TeX внутри, поэтому внешняя LaTeX‑дистрибуция не требуется.

Процесс преобразования разбирает разметку TeX, разрешает макросы, формирует документ и, наконец, записывает отрисованные страницы в PDF‑файл, который можно открыть на любой платформе.

## Импорт пространств имён

Необходимые пространства имён предоставляют доступ к основным типам Aspose.TeX для наборки и вывода PDF.

```text
using Aspose.TeX;
using Aspose.TeX.Saving;
using System.IO;
using System.IO.Compression;
```

Эти инструкции `using` предоставляют вам `TeXProcessor`, `PdfSaveOptions` и утилиты ZIP, необходимые для рабочего процесса.

## Шаг 1: Настройка входных и выходных каталогов

**Прямой ответ:** Создайте два временных ZIP‑архива — один для исходных файлов TeX и ресурсов (ввод), а второй для сгенерированного PDF (вывод). Такой подход изолирует задачу, упрощает потоковую передачу данных и стабильно работает на всех платформах.

Мы используем `System.IO.Compression.ZipArchive` для построения архивов в памяти, избегая побочных эффектов файловой системы.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Pdf;
using System.IO;
```

## Шаг 2: Определение параметров преобразования

**Прямой ответ:** Создайте объект `TeXConversionOptions`, задайте имя задания, соответствующее вашему файлу `.tex` (без расширения), и укажите входной и выходной ZIP‑архивы. Этот объект сообщает Aspose.TeX, откуда читать и куда записывать полученный PDF.

`TeXConversionOptions` инкапсулирует конфигурацию задания, включая входные и выходные ZIP‑архивы и имя TeX‑файла для обработки.

`PdfDevice` — это цель рендеринга, которая записывает байты PDF в выходной поток.

```csharp
// Set up input and output ZIP archives
using (Stream inZipStream = File.Open(Path.Combine("Your Input Directory", "zip-in.zip"), FileMode.Open))
using (Stream outZipStream = File.Open(Path.Combine("Your Output Directory", "typeset-pdf-to-external-stream.zip"), FileMode.Create))
{
    // Additional setup goes here
}
```

## Шаг 3: Настройка параметров сохранения (save pdf with options)

**Прямой ответ:** Создайте экземпляр `PdfSaveOptions` для управления сжатием, встраиванием шрифтов и метаданными перед записью PDF. Здесь же можно задать размер страницы, отступы и шифрование.

`PdfSaveOptions` контролирует, как генерируемый PDF будет записан, включая уровень сжатия, встраивание шрифтов и метаданные.

```csharp
// Define TeX conversion options
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
options.JobName = "typeset-pdf-to-external-stream";
options.InputWorkingDirectory = new InputZipDirectory(inZipStream, "in");
options.OutputWorkingDirectory = new OutputZipDirectory(outZipStream);
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

## Шаг 4: Наборка TeX в PDF

**Прямой ответ:** Откройте поток для записи (например, `FileStream` или `MemoryStream`) для выходного PDF, затем вызовите `TeXProcessor.Typeset(options, device)`. Процессор читает исходный TeX, компилирует его и передаёт готовый PDF в указанный девайс.

`TeXProcessor` — ядро, которое читает исходный TeX, выполняет компиляцию и создаёт окончательный вывод с использованием заданного устройства.

Этот шаг выполняет реальное **преобразование tex в pdf** и создаёт готовый к использованию PDF‑файл.

```csharp
// Define saving options
options.SaveOptions = new PdfSaveOptions();
```

## Шаг 5: Завершение вывода

**Прямой ответ:** Закройте выходной ZIP‑архив, чтобы завершить пакет, затем извлеките PDF‑файл из архива или передайте его напрямую клиенту. Закрытие архива гарантирует, что все записи сброшены и структура ZIP‑файла валидна.

```csharp
// Typeset TeX to PDF
using (Stream stream = File.Open(Path.Combine("Your Output Directory", "file-name.pdf"), FileMode.Create))
    new TeXJob("hello-world", new PdfDevice(stream), options).Run();
```

## Распространённые проблемы и решения

| Проблема | Почему происходит | Как исправить |
|----------|-------------------|---------------|
| **Отсутствуют шрифты** | В исходном TeX‑файле указаны шрифты, не включённые в библиотеку. | Добавьте необходимые шрифты в входной ZIP или настройте `PdfSaveOptions` для их встраивания. |
| **Большие проекты TeX завершаются по таймауту** | Значение таймаута по умолчанию мало для больших документов. | Увеличьте таймаут через `options.ExecutionTimeout`. |
| **Выходной PDF пустой** | В входном ZIP отсутствует файл `.tex` или имя задания не совпадает. | Убедитесь, что `options.JobName` соответствует имени TeX‑файла без расширения. |

### Совет:
При пакетной обработке множества файлов переиспользуйте один экземпляр `TeXProcessor` и меняйте только `TeXConversionOptions` для каждого задания. Это снижает накладные расходы и повышает пропускную способность до **30 %**.
