# Hur man fångar konsolutdata C# – Åsidosätt jobbnamn och skriv till disk

## Snabba svar
- **Vad betyder “capture console output C#”?** Det omdirigerar standardterminalströmmen som genereras av Aspose.TeX till en fil du anger, och bevarar varje logg, varning och statusmeddelande.  
- **Varför åsidosätta jobbnamnet?** Åsidosättning garanterar förutsägbara filnamn, eliminerar namnkonflikter i parallella jobb och förenklar efterföljande skript.  
- **Vilken Aspose.TeX-klass skriver utdata?** `OutputFileTerminal`, tilldelad via `options.TerminalOut`. *OutputFileTerminal är en klass som skriver terminalmeddelanden till en fil.*  
- **Kan jag ange en anpassad TeX‑indatamapp?** Ja—använd `options.InputWorkingDirectory` för att peka på dina käll `.tex`‑filer.  
- **Är detta tillvägagångssätt kompatibelt med .NET Core?** Absolut; samma API fungerar på .NET Framework, .NET Core och .NET 5+.

## Vad är “capture console output C#” i samband med Aspose.TeX?
Att fånga konsolutdata innebär att ta allt som normalt skulle visas i terminalfönstret (loggmeddelanden, varningar, kompileringsdetaljer) och skriva det till en beständig fil. Detta är särskilt användbart för felsökning av stora TeX‑projekt eller för att integrera TeX‑bearbetning i automatiserade arbetsflöden.

## Varför åsidosätta jobbnamnet och skriva terminalutdata till en fil?
Att åsidosätta jobbnamnet låter dig bestämma basnamnet för alla genererade artefakter, medan skrivning av terminalloggen ger dig en komplett revisionsspår. Förutsägbara filnamn gör efterbearbetningsskript enkla, och en sparad logg gör det möjligt att granska kompileringssteg i efterhand, vilket är kritiskt för automatiska byggen och parallella exekveringsmiljöer.

## Kvantifierade fördelar med att använda Aspose.TeX
Aspose.TeX stöder **30+ in- och utdataformat**—inklusive `.tex`, `.pdf`, `.xps`, `.html` och vanliga bildtyper—och kan bearbeta **dokument upp till 500 sidor** utan att ladda hela filen i minnet. Bibliotekets streaming‑arkitektur minskar maxminnesanvändningen med upp till **70 %** jämfört med naiva fil‑baserade metoder.

## Importera namnrymder

Lägg till de nödvändiga `using`‑direktiven högst upp i din C#‑fil så kompilatorn kan hitta Aspose.TeX‑typer.

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## Steg 1: Skapa konverteringsalternativ

`TeXOptions`‑klassen kapslar in alla konfigurationsinställningar för ett TeX‑konverteringsjobb. Den låter dig ange konsol‑appbeteende, indatakataloger och utmatningsenheter.

`ConsoleAppOptions` konfigurerar konsol‑applikationsbeteende för Aspose.TeX.  
`TeXConfig` definierar typen av TeX‑bearbetningskonfiguration.

Skapa en `TeXOptions`‑instans med `ConsoleAppOptions`, och ange `ObjectTeX` som `TeXConfig`. Detta talar om för Aspose.TeX att du kör i ett konsol‑applikationsscenario.

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## Steg 2: Ange jobbnamn (åsidosätt standard)

`TeXJob`‑klassen använder jobbnamnet som bas för varje genererad fil. Att åsidosätta det förhindrar biblioteket från att generera slumpmässiga namn som `job_12345`.

```csharp
options.JobName = "overridden-job-name";
```

Sätt `options.JobName = "MyCustomJob"` (eller någon identifierare du föredrar) innan du skapar jobb‑instansen.

## Steg 3: Ange TeX‑indatamapp

`InputWorkingDirectory`‑egenskapen talar om för Aspose.TeX var dina käll `.tex`‑filer finns. Detta är steget **ange tex‑indatamapp**.

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

Ange den absoluta eller relativa sökvägen till mappen som innehåller dina TeX‑källor, t.ex. `@"C:\\TeX\\Sources"`.

## Steg 4: Ange utdatakatalog

Definiera mappen där bearbetade filer och den fångade konsolloggen ska sparas. Aspose.TeX kommer att skapa underkataloger vid behov, men rotmappen måste finnas och vara skrivbar.

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

Tilldela `options.OutputWorkingDirectory = @"C:\\TeX\\Results"` (se till att katalogen har skrivbehörighet).

## Steg 5: Skriv terminalutdata till fil

`OutputFileTerminal` är den konkreta implementationen som skriver varje konsolmeddelande till en fil. Genom att tilldela den till `options.TerminalOut` ersätter du standard‑strömmen i minnet.

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

Exempel: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## Steg 6: Kör jobbet

`TeXJob`‑klassen orkestrerar konverteringen. Du anger det åsidosatta jobbnamnet, utmatningsenheten (`XpsDevice` i detta exempel) och det fullt konfigurerade alternativ‑objektet. `XpsDevice` genererar XPS‑utdata från TeX‑källan.

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

Efter att ha anropat `job.Run();` genererar Aspose.TeX XPS‑filen och skriver hela terminalloggen till `console.log` i utdatakatalogen.

## Vanliga problem & felsökning

| Symtom | Trolig orsak | Åtgärd |
|--------|--------------|--------|
| Ingen utdatafil skapad | Sökvägen till utdatakatalogen är felaktig eller saknar skrivbehörighet | Verifiera att `options.OutputWorkingDirectory` pekar på en giltig mapp och att processen har skrivbehörighet. |
| Terminallogg är tom | `TerminalOut` inte satt eller överskrivet senare | Säkerställ att `options.TerminalOut = new OutputFileTerminal(...);` körs innan `job.Run();`. |
| Jobbet misslyckas med “fil ej funnen” | Indatakatalogen är inte korrekt angiven | Dubbelkolla sökvägen som skickas till `options.InputWorkingDirectory` och att `.tex`‑filerna finns där. |
| Oväntade tecken i logg | Kodningsmismatch mellan TeX‑motor och filsystem | Ställ in `options.Encoding = Encoding.UTF8` (eller lämplig kodning) innan jobbet körs. |
