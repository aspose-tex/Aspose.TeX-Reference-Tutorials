# कंसोल आउटपुट कैप्चर कैसे करें C# – जॉब नाम ओवरराइड करें और डिस्क पर लिखें

इस चरण‑दर‑चरण ट्यूटोरियल में आप Aspose.TeX for .NET का उपयोग करते समय **C# में कंसोल आउटपुट कैसे कैप्चर करें** की खोज करेंगे। जॉब नाम को ओवरराइड करके और टर्मिनल स्ट्रीम को फ़ाइल में रूट करके, आपको निर्धारित फ़ाइल नामकरण, आसान लॉग एग्रीगेशन, और CI/CD पाइपलाइन या बैच‑प्रोसेसिंग परिदृश्यों के लिए एक ठोस आधार मिलता है।

## त्वरित उत्तर
- **“C# में कंसोल आउटपुट कैप्चर” का क्या अर्थ है?** यह Aspose.TeX द्वारा उत्पन्न मानक टर्मिनल स्ट्रीम को आप द्वारा निर्दिष्ट फ़ाइल में रीडायरेक्ट करता है, जिससे हर लॉग, चेतावनी और स्थिति संदेश संरक्षित रहता है।  
- **जॉब नाम को ओवरराइड क्यों करें?** ओवरराइड करने से पूर्वानुमेय फ़ाइल नाम सुनिश्चित होते हैं, समानांतर जॉब्स में नाम टकराव समाप्त होते हैं, और डाउनस्ट्रीम स्क्रिप्ट्स सरल हो जाती हैं।  
- **कौन सा Aspose.TeX क्लास आउटपुट लिखता है?** `OutputFileTerminal`, जिसे `options.TerminalOut` के माध्यम से असाइन किया जाता है। *OutputFileTerminal वह क्लास है जो टर्मिनल संदेशों को फ़ाइल में लिखता है।*  
- **क्या मैं कस्टम TeX इनपुट फ़ोल्डर सेट कर सकता हूँ?** हाँ—`options.InputWorkingDirectory` का उपयोग करके अपने स्रोत `.tex` फ़ाइलों की ओर इशारा करें।  
- **क्या यह तरीका .NET Core के साथ संगत है?** बिल्कुल; वही API .NET Framework, .NET Core, और .NET 5+ पर काम करता है।

## Aspose.TeX के संदर्भ में “C# में कंसोल आउटपुट कैप्चर” क्या है?
कंसोल आउटपुट कैप्चर करने का मतलब है टर्मिनल विंडो में सामान्यतः दिखाई देने वाली सभी चीज़ें (लॉग संदेश, चेतावनियाँ, संकलन विवरण) को एक स्थायी फ़ाइल में लिखना। यह बड़े TeX प्रोजेक्ट्स को डिबग करने या TeX प्रोसेसिंग को स्वचालित कार्यप्रवाहों में एकीकृत करने के लिए विशेष रूप से उपयोगी है।

## जॉब नाम को ओवरराइड करने और टर्मिनल आउटपुट को फ़ाइल में लिखने का कारण क्या है?
जॉब नाम को ओवरराइड करने से आप सभी उत्पन्न आर्टिफैक्ट्स के बेस नाम को तय कर सकते हैं, जबकि टर्मिनल लॉग को लिखने से आपको एक पूर्ण ऑडिट ट्रेल मिलता है। पूर्वानुमेय फ़ाइल नाम पोस्ट‑प्रोसेसिंग स्क्रिप्ट्स को सरल बनाते हैं, और सहेजा गया लॉग आपको बाद में संकलन चरणों की समीक्षा करने की सुविधा देता है, जो स्वचालित बिल्ड्स और समानांतर निष्पादन वातावरण के लिए महत्वपूर्ण है।

## Aspose.TeX के उपयोग के मात्रात्मक लाभ

## नेमस्पेसेस इम्पोर्ट करें

अपने C# फ़ाइल के शीर्ष पर आवश्यक `using` निर्देश जोड़ें ताकि कंपाइलर Aspose.TeX टाइप्स को ढूंढ सके।

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## चरण 1: कन्वर्ज़न विकल्प बनाएं

`TeXOptions` क्लास TeX कन्वर्ज़न जॉब के सभी कॉन्फ़िगरेशन सेटिंग्स को समेटे हुए है। यह आपको कंसोल‑ऐप व्यवहार, इनपुट डायरेक्टरीज़, और आउटपुट डिवाइस निर्दिष्ट करने की अनुमति देता है।

`ConsoleAppOptions` Aspose.TeX के लिए कंसोल‑ऐप्लिकेशन व्यवहार को कॉन्फ़िगर करता है।  
`TeXConfig` TeX प्रोसेसिंग कॉन्फ़िगरेशन के प्रकार को परिभाषित करता है।

`ConsoleAppOptions` के साथ `TeXOptions` का एक इंस्टेंस बनाएं, और `TeXConfig` के रूप में `ObjectTeX` निर्दिष्ट करें। यह Aspose.TeX को बताता है कि आप एक कंसोल‑ऐप्लिकेशन परिदृश्य में चल रहे हैं।

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## चरण 2: जॉब नाम निर्दिष्ट करें (डिफ़ॉल्ट ओवरराइड करें)

`TeXJob` क्लास जॉब नाम को प्रत्येक उत्पन्न फ़ाइल के बेस के रूप में उपयोग करता है। इसे ओवरराइड करने से लाइब्रेरी को `job_12345` जैसी रैंडम नाम बनाने से रोका जाता है।

```csharp
options.JobName = "overridden-job-name";
```

जॉब इंस्टेंस बनाने से पहले `options.JobName = "MyCustomJob"` (या आपका कोई भी पहचानकर्ता) सेट करें।

## चरण 3: TeX इनपुट डायरेक्टरी सेट करें

`InputWorkingDirectory` प्रॉपर्टी Aspose.TeX को बताती है कि आपके स्रोत `.tex` फ़ाइलें कहाँ स्थित हैं। यह **TeX इनपुट डायरेक्टरी सेट** करने का चरण है।

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

उस फ़ोल्डर का पूर्ण या रिलेटिव पाथ प्रदान करें जिसमें आपके TeX स्रोत हैं, उदाहरण के लिए `@"C:\TeX\Sources"`।

## चरण 4: आउटपुट वर्किंग डायरेक्टरी निर्दिष्ट करें

उस फ़ोल्डर को परिभाषित करें जहाँ प्रोसेस्ड फ़ाइलें और कैप्चर किया गया कंसोल लॉग सहेजा जाएगा। Aspose.TeX आवश्यकता अनुसार सब‑फ़ोल्डर बनाएगा, लेकिन रूट फ़ोल्डर मौजूद और लिखने योग्य होना चाहिए।

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

`options.OutputWorkingDirectory = @"C:\TeX\Results"` असाइन करें (सुनिश्चित करें कि डायरेक्टरी में लिखने की अनुमति है)।

## चरण 5: टर्मिनल आउटपुट को फ़ाइल में लिखें

`OutputFileTerminal` वह ठोस इम्प्लीमेंटेशन है जो हर कंसोल संदेश को फ़ाइल में लिखता है। इसे `options.TerminalOut` को असाइन करके, आप डिफ़ॉल्ट इन‑मेमोरी कंसोल स्ट्रीम को बदल देते हैं।

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

उदाहरण: `options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## चरण 6: जॉब चलाएँ

`TeXJob` क्लास कन्वर्ज़न को ऑर्केस्ट्रेट करता है। आप ओवरराइड किया हुआ जॉब नाम, आउटपुट डिवाइस (`XpsDevice` इस उदाहरण में), और पूरी तरह कॉन्फ़िगर किया हुआ options ऑब्जेक्ट प्रदान करते हैं। `XpsDevice` TeX स्रोत से XPS आउटपुट बनाता है।

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

`job.Run();` कॉल करने के बाद, Aspose.TeX XPS फ़ाइल बनाता है और पूरे टर्मिनल लॉग को आउटपुट फ़ोल्डर में `console.log` में लिखता है।
