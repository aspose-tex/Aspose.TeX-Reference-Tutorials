# कंसोल आउटपुट को कैप्चर करना C# – जॉब नाम को ओवरराइड करना और टर्मिनल आउटपुट को डिस्क पर लिखना (C#)

## परिचय

## त्वरित उत्तर
- **“capture console output C#” का क्या अर्थ है?** यह Aspose.TeX द्वारा उत्पन्न मानक टर्मिनल स्ट्रीम को आपके द्वारा निर्दिष्ट फ़ाइल में पुनः निर्देशित करता है।  
- **जॉब नाम को ओवरराइड क्यों करें?** ओवरराइड करने से फ़ाइल नाम पूर्वानुमेय होते हैं और कई जॉब्स चलाते समय टकराव से बचा जा सकता है।  
- **कौन सा Aspose.TeX क्लास आउटपुट लिखता है?** `OutputFileTerminal` (`options.TerminalOut` के माध्यम से उपयोग किया जाता है)।  
- **क्या मैं कस्टम TeX इनपुट फ़ोल्डर सेट कर सकता हूँ?** हाँ—`options.InputWorkingDirectory` का उपयोग करके **TeX इनपुट डायरेक्टरी सेट** करें।  
- **क्या यह तरीका .NET Core के साथ संगत है?** बिल्कुल; वही API .NET Framework और .NET Core दोनों पर काम करती है।

## Aspose.TeX के संदर्भ में “capture console output C#” क्या है?
कंसोल आउटपुट को कैप्चर करना मतलब है कि टर्मिनल विंडो में सामान्यतः दिखने वाले सभी संदेशों (लॉग, चेतावनियाँ, कम्पाइलेशन विवरण) को एक स्थायी फ़ाइल में लिखना। यह बड़े TeX प्रोजेक्ट्स को डिबग करने या TeX प्रोसेसिंग को स्वचालित वर्कफ़्लोज़ में एकीकृत करने के लिए विशेष रूप से उपयोगी है।

## जॉब नाम को ओवरराइड करना और टर्मिनल आउटपुट को फ़ाइल में लिखना क्यों आवश्यक है?
- **पूर्वानुमेय फ़ाइलनाम:** जॉब नाम को ओवरराइड करने से आप उत्पन्न फ़ाइलों के बेस नाम को तय कर सकते हैं, जिससे पोस्ट‑प्रोसेसिंग स्क्रिप्ट लिखना आसान हो जाता है।  
- **ऑडिटेबिलिटी:** टर्मिनल लॉग को संग्रहीत करने से आपको TeX कम्पाइलेशन प्रक्रिया का पूर्ण ऑडिट ट्रेल मिलता है।  
- **समांतर निष्पादन:** जब कई जॉब्स एक साथ चलाए जाते हैं, तो अद्वितीय जॉब नाम फ़ाइल टकराव से बचाते हैं।  

## पूर्वापेक्षाएँ

- Aspose.TeX for .NET लाइब्रेरी: सुनिश्चित करें कि आपके पास Aspose.TeX for .NET लाइब्रेरी स्थापित है। आप इसे [Aspose.TeX वेबसाइट](https://releases.aspose.com/tex/net/) से डाउनलोड कर सकते हैं।  
- .NET विकास पर्यावरण: एक कार्यशील .NET विकास पर्यावरण रखें, जिसमें Visual Studio जैसे एकीकृत विकास पर्यावरण (IDE) शामिल हो।  
- बेसिक C# ज्ञान: C# प्रोग्रामिंग भाषा की मूल बातें से परिचित हों।  
- इनपुट और आउटपुट डायरेक्टरीज़: अपने TeX फ़ाइलों के लिए इनपुट और आउटपुट डायरेक्टरी तैयार करें।  

## नेमस्पेस आयात करें

अपने C# कोड में, Aspose.TeX कार्यक्षमताओं तक पहुँचने के लिए आवश्यक नेमस्पेस शामिल करना सुनिश्चित करें:

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## चरण 1: रूपांतरण विकल्प बनाएं

पहले, हम एक `TeXOptions` इंस्टेंस बनाते हैं जो Aspose.TeX को बताता है कि हम एक कंसोल‑एप्लिकेशन परिदृश्य में चल रहे हैं।

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

`ConsoleAppOptions` के साथ `TeXOptions` बनाएं, और `ObjectTeX` को `TeXConfig` के रूप में निर्दिष्ट करें।

## चरण 2: जॉब नाम निर्दिष्ट करें (डिफ़ॉल्ट ओवरराइड करें)

जॉब नाम को ओवरराइड करने से हमें सभी उत्पन्न आर्टिफैक्ट्स के बेस नाम पर नियंत्रण मिलता है।

```csharp
options.JobName = "overridden-job-name";
```

डिफ़ॉल्ट नाम को ओवरराइड करने के लिए जॉब नाम निर्दिष्ट करें।

## चरण 3: TeX इनपुट डायरेक्टरी सेट करें

Aspose.TeX को बताएं कि आपके स्रोत `.tex` फ़ाइलें कहाँ स्थित हैं। यह **set tex input directory** चरण है।

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

अपने TeX फ़ाइलों के लिए इनपुट वर्किंग डायरेक्टरी सेट करें।

## चरण 4: आउटपुट वर्किंग डायरेक्टरी निर्दिष्ट करें

परिवर्तित फ़ाइलों और कैप्चर किए गए कंसोल लॉग को सहेजने के स्थान को परिभाषित करें।

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

परिवर्तित TeX फ़ाइलों को सहेजने के लिए आउटपुट वर्किंग डायरेक्टरी निर्धारित करें।

## चरण 5: टर्मिनल आउटपुट को फ़ाइल में लिखें

अब हम कंसोल स्ट्रीम को आउटपुट डायरेक्टरी में एक भौतिक फ़ाइल की ओर निर्देशित करते हैं। यह **write terminal output to file** आवश्यकता को लागू करता है।

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

आउटपुट डायरेक्टरी में फ़ाइल में टर्मिनल आउटपुट लिखने के लिए कॉन्फ़िगर करें।

## चरण 6: जॉब चलाएँ

अंत में, हम ओवरराइड किए गए जॉब नाम, XPS आउटपुट डिवाइस, और कॉन्फ़िगर किए गए विकल्पों के साथ एक `TeXJob` बनाते हैं। जॉब चलाने से XPS फ़ाइल उत्पन्न होगी और कंसोल लॉग कैप्चर होगा।

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

जॉब नाम, आउटपुट डिवाइस (`XpsDevice`), और विकल्प निर्दिष्ट करके `TeXJob` बनाएं। अंत में, जॉब चलाएँ।

## सामान्य समस्याएँ और ट्रबलशूटिंग

| लक्षण | संभावित कारण | समाधान |
|---------|--------------|-----|
| कोई आउटपुट फ़ाइल नहीं बनाई गई | आउटपुट डायरेक्टरी पाथ गलत है या लिखने की अनुमति नहीं है | `options.OutputWorkingDirectory` को एक वैध फ़ोल्डर की ओर इंगित करता है और प्रक्रिया के पास लिखने की अनुमति है, इसे सत्यापित करें। |
| टर्मिनल लॉग खाली है | `TerminalOut` सेट नहीं है या बाद में ओवरराइट हो गया है | `options.TerminalOut = new OutputFileTerminal(...);` को `job.Run();` से पहले निष्पादित किया गया है, यह सुनिश्चित करें। |
| जॉब “फ़ाइल नहीं मिली” त्रुटि के साथ विफल हो रहा है | इनपुट डायरेक्टरी सही ढंग से सेट नहीं है | `InputFileSystemDirectory` को पास किया गया पाथ दोबारा जांचें और सुनिश्चित करें कि वहाँ `.tex` फ़ाइलें मौजूद हैं। |
