# Aspose.TeX for .NET के साथ कस्टम LaTeX फ़ॉर्मेट बनाएं

## परिचय

LaTeX उच्च‑गुणवत्ता टाइपसेटिंग का मानक है, और कई .NET डेवलपर्स चाहते हैं कि वे प्रोग्रामेटिक रूप से **कस्टम LaTeX फ़ॉर्मेट** फ़ाइलें बनाएं जो उनके प्रोजेक्ट की ब्रांडिंग या विशेष लेआउट आवश्यकताओं से मेल खाती हों। Aspose.TeX for .NET आपको एक साफ़ API देता है जिससे आप ये फ़ॉर्मेट सीधे C# या VB.NET से जनरेट कर सकते हैं, बिना बाहरी टूल्स के झंझट के। इस ट्यूटोरियल में आप देखेंगे कि इंजन को कैसे सेट‑अप करें, उसे अपने फ़ोल्डरों की ओर कैसे इंगित करें, और एक पुन: उपयोग योग्य कस्टम फ़ॉर्मेट कैसे बनाएं जिसे आप विभिन्न प्रोजेक्ट्स में दोबारा उपयोग कर सकें।

## त्वरित उत्तर
- **“कस्टम LaTeX फ़ॉर्मेट बनाना” का क्या अर्थ है?** यह एक व्यक्तिगत TeX इंजन कॉन्फ़िगरेशन (*.fmt* फ़ाइल) जेनरेट करने को कहा जाता है जिसे बाद में तेज़ कंपाइलेशन के लिए लोड किया जा सकता है।  
- **क्या इसे आज़माने के लिए लाइसेंस चाहिए?** एक फ्री ट्रायल उपलब्ध है; प्रोडक्शन उपयोग के लिए लाइसेंस आवश्यक है।  
- **कौन‑से .NET संस्करण समर्थित हैं?** सभी आधुनिक .NET Framework, .NET Core, और .NET 5/6 संस्करण।  
- **सेट‑अप में कितना समय लगेगा?** आमतौर पर Aspose.TeX इंस्टॉल होने के बाद 10 मिनट से कम।  
- **क्या मैं फ़ॉर्मेट को अन्य एप्लिकेशन में पुन: उपयोग कर सकता हूँ?** हाँ – *.fmt* फ़ाइल को किसी भी TeX इंजन द्वारा लोड किया जा सकता है जो ObjectTeX एक्सटेंशन को समझता हो।

## “कस्टम LaTeX फ़ॉर्मेट बनाना” क्या है?
कस्टम LaTeX फ़ॉर्मेट बनाना का मतलब है TeX मैक्रो, पैकेज और इंजन विकल्पों के सेट को एकल बाइनरी फ़ॉर्मेट फ़ाइल में संकलित करना। यह प्री‑कम्पाइल्ड फ़ाइल बाद में दस्तावेज़ प्रोसेसिंग को तेज़ बनाती है क्योंकि इंजन प्रारंभिक पार्सिंग चरण को छोड़ देता है।

## Aspose.TeX for .NET क्यों उपयोग करें?
- **सीधे .NET इंटीग्रेशन** – अपने C# कोड से सीधे TeX फ़ंक्शनैलिटी को कॉल करें।  
- **कोई बाहरी बाइनरी नहीं** – लाइब्रेरी में सभी आवश्यक चीज़ें शामिल हैं।  
- **I/O पर पूर्ण नियंत्रण** – इनपुट और आउटपुट डायरेक्टरी को प्रोग्रामेटिक रूप से निर्दिष्ट करें।  
- **प्रोफेशनल सपोर्ट** – Aspose फ़ोरम और लाइसेंसिंग विकल्पों तक पहुंच प्राप्त करें।

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित हैं:

### 1. Aspose.TeX for .NET स्थापित करें
नवीनतम संस्करण प्राप्त करने के लिए [download link](https://releases.aspose.com/tex/net/) पर जाएँ। लाइब्रेरी को अपने प्रोजेक्ट में सेट‑अप करने के लिए दस्तावेज़ में दी गई इंस्टॉलेशन निर्देशों का पालन करें।

### 2. आवश्यक नेमस्पेस इम्पोर्ट करें
अपने .NET प्रोजेक्ट में Aspose.TeX फ़ंक्शनैलिटी को उपलब्ध कराने के लिए आवश्यक नेमस्पेस इम्पोर्ट करें। निम्नलिखित `using` निर्देश जोड़ें:

```csharp
using Aspose.TeX.IO;
```

अब हम कोड को चरण‑दर‑चरण देखते हैं।

## कस्टम LaTeX फ़ॉर्मेट कैसे बनाएं

### चरण 1: TeX इंजन विकल्प बनाएं
पहले, हम इंजन को एक कंसोल‑स्टाइल एप्लिकेशन के लिए कॉन्फ़िगर करते हैं और उसे ObjectTeX एक्सटेंशन उपयोग करने के लिए कहते हैं।

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectIniTeX);
```

> **प्रो टिप:** `ConsoleAppOptions` का उपयोग करने से इंजन GUI निर्भरताओं के बिना चलता है, जो सर्वर‑साइड ऑटोमेशन के लिए आदर्श है।

### चरण 2: इनपुट और आउटपुट डायरेक्टरी निर्दिष्ट करें
अब उन फ़ोल्डरों को सेट करें जहाँ आपके स्रोत *.tex* फ़ाइलें स्थित हैं और जहाँ जेनरेटेड फ़ॉर्मेट लिखा जाएगा।

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

> यह चरण **कस्टम LaTeX फ़ॉर्मेट बनाना** वर्कफ़्लो के लिए महत्वपूर्ण है क्योंकि इंजन को उन मैक्रो फ़ाइलों को खोजने की आवश्यकता होती है जिन्हें आप प्री‑कम्पाइल करना चाहते हैं।

### चरण 3: फ़ॉर्मेट निर्माण चलाएँ
अब वह जॉब कॉल करें जो वास्तव में फ़ॉर्मेट को कम्पाइल करता है। आप फ़ॉर्मेट को कोई भी नाम दे सकते हैं; यहाँ हम `"customtex"` उपयोग कर रहे हैं।

```csharp
TeXJob.CreateFormat("customtex", options);
```

इस कॉल के समाप्त होने के बाद, आप आउटपुट डायरेक्टरी में `customtex.fmt` फ़ाइल पाएँगे, जिसे आप पुनः उपयोग के लिए रख सकते हैं।

### चरण 4: साफ़ आउटपुट सुनिश्चित करें
कंसोल आउटपुट को साफ़ रखने के लिए (विशेषकर CI पाइपलाइन में), टर्मिनल में एक खाली लाइन लिखें।

```csharp
options.TerminalOut.Writer.WriteLine();
```

### अतिरिक्त प्रश्न‑उत्तर

**प्रश्न: क्या मैं जेनरेटेड फ़ॉर्मेट को किसी अलग मशीन पर पुनः उपयोग कर सकता हूँ?**  
उत्तर: बिल्कुल। `.fmt` फ़ाइल पोर्टेबल है; इसे लक्ष्य मशीन पर कॉपी करें और इंजन को उस फ़ाइल की ओर इंगित करें।

**प्रश्न: क्या फ़ॉर्मेट में मेरे कस्टम मैक्रो शामिल होते हैं?**  
उत्तर: हाँ, इनपुट डायरेक्टरी में रखी गई सभी `.sty` या `.tex` फ़ाइलें फ़ॉर्मेट में कम्पाइल हो जाती हैं।
