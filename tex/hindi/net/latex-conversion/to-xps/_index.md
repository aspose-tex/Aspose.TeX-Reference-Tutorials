# .NET में LaTeX से XPS - Aspose.TeX के साथ आसान रूपांतरण

## परिचय

यदि आप अपने .NET अनुप्रयोगों में **latex को XPS फ़ॉर्मेट में कैसे बदलें** के बारे में सोच रहे हैं, तो आप सही जगह पर हैं। Aspose.TeX for .NET एक शक्तिशाली, सरल समाधान प्रदान करता है जो आपके लिए सभी जटिल कार्य संभालता है। इस गाइड में हम प्रत्येक चरण को समझेंगे, यह बताएँगे कि प्रत्येक सेटिंग क्यों महत्वपूर्ण है, और दिखाएँगे कि केवल कुछ कोड लाइनों से उच्च‑गुणवत्ता, अनुकूलन योग्य XPS आउटपुट कैसे प्राप्त करें।

## त्वरित उत्तर
- **रूपांतरण को संभालने वाली लाइब्रेरी कौन सी है?** Aspose.TeX for .NET  
- **समर्थित आउटपुट फ़ॉर्मेट?** XPS (also PDF, PNG, etc.)  
- **सामान्य कार्यान्वयन समय?** 10–15 मिनट एक बुनियादी रूपांतरण के लिए  
- **क्या मुझे लाइसेंस चाहिए?** उत्पादन के लिए एक अस्थायी लाइसेंस आवश्यक है; एक मुफ्त ट्रायल उपलब्ध है।  
- **क्या मैं रूपांतरण को मेमोरी से चला सकता हूँ?** हाँ, जैसा कि बाद में दिखाया गया है, `MemoryStream` का उपयोग करके।

## .NET में LaTeX को XPS में कैसे बदलें

नीचे एक संक्षिप्त, चरण‑दर‑चरण मार्गदर्शिका है जो आपको आवश्यक सभी चीज़ों को कवर करती है—पूर्वापेक्षाओं से लेकर वैकल्पिक समायोजनों तक—ताकि आप अपने अनुप्रयोग की व्यावसायिक लॉजिक पर ध्यान केंद्रित कर सकें।

## पूर्वापेक्षाएँ

ट्यूटोरियल में डुबकी लगाने से पहले, सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ मौजूद हैं:

- C# और .NET विकास का कार्यशील ज्ञान।  
- Aspose.TeX for .NET लाइब्रेरी स्थापित है। आप इसे **[यहाँ](https://releases.aspose.com/tex/net/)** से डाउनलोड कर सकते हैं।  
- LaTeX सिंटैक्स और संरचना की समझ।

## नेमस्पेस आयात करें

आइए हमारे .NET अनुप्रयोग के लिए आवश्यक नेमस्पेस आयात करके शुरू करें। ये नेमस्पेस Aspose.TeX कार्यात्मकताओं के साथ इंटरैक्ट करने के लिए महत्वपूर्ण हैं।

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
using System.IO;
using System.Text;
```

## चरण 1: रूपांतरण विकल्प सेट करें

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectLaTeX);
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

यहाँ, हम रूपांतरण विकल्प प्रारंभ करते हैं और इंजन को उस फ़ोल्डर की ओर इंगित करते हैं जिसमें आपकी `.ltx` स्रोत फ़ाइलें हैं।

## चरण 2: इंटरैक्शन मोड सेट करें

```csharp
options.Interaction = Interaction.NonstopMode;
```

नॉन‑स्टॉप मोड इंजन को छोटे चेतावनियों का सामना करने पर भी प्रोसेसिंग जारी रखने के लिए कहता है, जो स्वचालित पाइपलाइन के लिए आदर्श है।

## चरण 3: जॉब नाम सेट करें (वैकल्पिक)

```csharp
// options.JobName = "my-job-name";
```

आप कई दस्तावेज़ों को प्रोसेस करते समय लॉग की पहचान में मदद के लिए एक कस्टम जॉब नाम असाइन कर सकते हैं।

## चरण 4: शीर्षक में तिथि सेट करें (वैकल्पिक)

```csharp
// options.DateTime = new System.DateTime(2022, 12, 18);
```

जनरेटेड टाइटल पेज में एक विशिष्ट तिथि दिखाने के लिए मजबूर करें, जो पुनरुत्पादनीय रिपोर्टों के लिए उपयोगी है।

## चरण 5: लापता पैकेजों को अनदेखा करें

```csharp
options.IgnoreMissingPackages = true;
```

जब इसे `true` पर सेट किया जाता है, तो इंजन लापता LaTeX पैकेजों को त्रुटि फेंके बिना स्किप कर देता है, जिससे बैच रूपांतरण तेज़ हो सकते हैं।

## चरण 6: लिगेचर निष्क्रिय करें

```csharp
options.NoLigatures = true;
```

लिगेचर को निष्क्रिय करने से यह सुनिश्चित होता है कि अक्षर संयोजन ठीक उसी तरह रेंडर हों जैसा टाइप किया गया है, जो कुछ तकनीकी दस्तावेज़ों के लिए आवश्यक है।

## चरण 7: जॉब दोहराएँ (वैकल्पिक)

```csharp
// options.Repeat = true;
```

इस फ़्लैग को सक्षम करने से इंजन को वही जॉब फिर से चलाने के लिए कहा जाता है—इटरेटिव डिबगिंग के लिए उपयोगी।

## चरण 8: आउटपुट वर्किंग डायरेक्टरी निर्दिष्ट करें

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

परिभाषित करें कि जनरेटेड XPS फ़ाइलें कहाँ लिखी जाएँगी।

## चरण 9: XPS के लिए सेव ऑप्शन प्रारंभ करें

```csharp
options.SaveOptions = new XpsSaveOptions(); // Default value. Arbitrary assignment.
```

`XpsSaveOptions` का एक इंस्टेंस बनाकर XPS आउटपुट को बारीकी से ट्यून करें।

## चरण 10: फ़ॉर्मूले रास्टराइज़ करें (वैकल्पिक)

```csharp
options.SaveOptions.RasterizeFormulas = true;
```

जब `true` हो, तो गणितीय फ़ॉर्मूले रास्टर इमेज के रूप में रेंडर होते हैं, जो पुराने XPS व्यूअर्स के साथ संगतता को सुधार सकता है।

## चरण 11: सम्मिलित ग्राफ़िक्स रास्टराइज़ करें (वैकल्पिक)

```csharp
options.SaveOptions.RasterizeIncludedGraphics = true;
```

LaTeX स्रोत में एम्बेडेड वेक्टर ग्राफ़िक्स को रास्टर इमेज में बदलें ताकि विभिन्न प्लेटफ़ॉर्म पर सुसंगत रेंडरिंग हो सके।

## चरण 12: फ़ॉन्ट सबसेट करें

```csharp
options.SaveOptions.SubsetFonts = true;
```

सबसेटिंग केवल उन ग्लिफ़ को एम्बेड करती है जो दस्तावेज़ में वास्तव में उपयोग हुए हैं, जिससे फ़ाइल आकार घटता है।

## चरण 13: LaTeX से XPS रूपांतरण चलाएँ

```csharp
new TeXJob(Path.Combine("Your Input Directory", "sample.ltx"), new XpsDevice(), options).Run();
```

यह एकल पंक्ति रूपांतरण प्रक्रिया को शुरू करती है, `sample.ltx` को पढ़ती है और आउटपुट डायरेक्टरी में एक XPS फ़ाइल बनाती है।

## चरण 14: MemoryStream के साथ LaTeX से XPS रूपांतरण चलाएँ (वैकल्पिक)

```csharp
// new TeXJob(new MemoryStream(Encoding.ASCII.GetBytes(@"\documentclass{article} \begin{document} Hello, World! \end{document}")),
//     new XpsDevice(), options).Run();
```

यदि आप LaTeX स्रोत को सीधे मेमोरी से फीड करना चाहते हैं—शायद ऑन‑द‑फ्लाई जेनरेट किया गया—तो दिखाए अनुसार `MemoryStream` का उपयोग करें।

## चरण 15: मुख्य इनपुट टर्मिनल के साथ LaTeX से XPS रूपांतरण चलाएँ (वैकल्पिक)

```csharp
// new TeXJob(new XpsDevice(), options).Run();
```

यह ओवरलोड आपको डिफ़ॉल्ट इनपुट टर्मिनल से रूपांतरण शुरू करने देता है, जो कमांड‑लाइन परिदृश्यों के लिए उपयोगी है।

## सामान्य समस्याएँ और टिप्स

- **लापता पैकेज:** भले ही `IgnoreMissingPackages` को `true` पर सेट किया हो, कुछ पैकेज आवश्यक होते हैं। सुनिश्चित करें कि कोर पैकेज (जैसे `amsmath`) आपके TeX वितरण में उपलब्ध हैं।  
- **फ़ॉन्ट सबसेटिंग त्रुटियाँ:** यदि आउटपुट XPS गड़बड़ दिखता है, तो पूर्ण फ़ॉन्ट एम्बेड करने के लिए `SubsetFonts` को निष्क्रिय करने का प्रयास करें।  
- **बड़े दस्तावेज़:** बहुत बड़े LaTeX प्रोजेक्ट्स के लिए, JVM हीप साइज बढ़ाएँ (यदि अंतर्निहित TeX इंजन का उपयोग कर रहे हैं) या दस्तावेज़ को छोटे हिस्सों में प्रोसेस करें।
