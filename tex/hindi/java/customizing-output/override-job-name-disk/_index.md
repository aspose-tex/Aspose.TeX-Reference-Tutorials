# टर्मिनल आउटपुट को फ़ाइल में लिखें और जावा में जॉब नाम को ओवरराइड करें

## परिचय

इस ट्यूटोरियल में, आप **कंसोल आउटपुट को कैप्चर करने** के बारे में जानेंगे जबकि आप Aspose.TeX for Java के साथ TeX फ़ाइलों को प्रोसेस कर रहे हैं। हम टर्मिनल आउटपुट को फ़ाइल में लिखने, डिफ़ॉल्ट जॉब नाम को ओवरराइड करने, और कंसोल आउटपुट को रीडायरेक्ट करने के चरणों से गुजरेंगे ताकि लॉग्स को आसानी से ढूँढ़ा और विश्लेषण किया जा सके। ये तकनीकें बैच रूपांतरण या स्वचालित दस्तावेज़ पाइपलाइन के लिए विश्वसनीय लॉगिंग की आवश्यकता होने पर आवश्यक हैं।

## त्वरित उत्तर
- **क्या मैं जॉब नाम बदल सकता हूँ?** हाँ, जॉब चलाने से पहले `options.setJobName(...)` का उपयोग करें।  
- **टर्मिनल आउटपुट कहाँ जाता है?** यह आउटपुट वर्किंग डायरेक्टरी में `<job_name>.trm` के रूप में सहेजा जाता है।  
- **क्या इस सुविधा के लिए लाइसेंस चाहिए?** यह कार्यक्षमता किसी भी वैध Aspose.TeX लाइसेंस के साथ काम करती है; एक मुफ्त ट्रायल भी उपलब्ध है।  
- **आउटपुट फ़ाइल का फ़ॉर्मेट क्या है?** साधारण‑टेक्स्ट टर्मिनल लॉग जो कंसोल आउटपुट को प्रतिबिंबित करता है।  
- **क्या यह अन्य आउटपुट डिवाइसों के साथ संगत है?** बिल्कुल—एक बार लॉग लिख दिया जाए तो आप इसे किसी भी टूल से प्रोसेस कर सकते हैं जो टेक्स्ट फ़ाइलें पढ़ता है।

## Aspose.TeX के संदर्भ में **कंसोल को कैप्चर करने** का क्या अर्थ है?

कंसोल आउटपुट को कैप्चर करना मतलब है सभी चीज़ों को जो सामान्यतः स्टैंडर्ड आउटपुट स्ट्रीम (टर्मिनल) पर दिखाई देती हैं, उन्हें डिस्क पर एक फ़ाइल में रीडायरेक्ट करना। Aspose.TeX के साथ आप इसे आसानी से `OutputFileTerminal` को कॉन्फ़िगर करके और इसे कन्वर्ज़न विकल्पों में असाइन करके कर सकते हैं।

## जॉब नाम को ओवरराइड क्यों करें?

जॉब नाम को ओवरराइड करने से प्रत्येक कन्वर्ज़न रन को एक अनूठा पहचानकर्ता मिलता है। इससे उत्पन्न लॉग फ़ाइलें (`*.trm`) और अन्य आर्टिफैक्ट्स को ट्रैक करना आसान हो जाता है, विशेष रूप से जब आप समानांतर में कई जॉब चलाते हैं या बैच प्रोसेस शेड्यूल करते हैं।

## पूर्वापेक्षाएँ

- जावा प्रोग्रामिंग में बुनियादी दक्षता।  
- Aspose.TeX for Java स्थापित (आधिकारिक [Aspose.TeX Java documentation](https://reference.aspose.com/tex/java/) से डाउनलोड करें)।  
- एक जावा IDE या बिल्ड टूल (Maven/Gradle) जो सैंपल को कंपाइल और रन करने के लिए तैयार हो।

## पैकेज इम्पोर्ट करें

शुरू करने के लिए, आवश्यक पैकेजों को अपने जावा प्रोजेक्ट में इम्पोर्ट करें। अपने जावा फ़ाइल में, निम्नलिखित इम्पोर्ट्स शामिल करें:

```java
package com.aspose.tex.OverridenJobNameAndTerminalOutputWrittenToDisk;

import java.io.IOException;

import com.aspose.tex.InputFileSystemDirectory;
import com.aspose.tex.OutputFileSystemDirectory;
import com.aspose.tex.OutputFileTerminal;
import com.aspose.tex.TeXConfig;
import com.aspose.tex.TeXJob;
import com.aspose.tex.TeXOptions;
import com.aspose.tex.rendering.XpsDevice;

import util.Utils;
```

> **प्रो टिप:** `util.Utils` इम्पोर्ट केवल तभी रखें जब आपको Aspose सैंपल यूटिलिटीज़ से हेल्पर मेथड्स की आवश्यकता हो; अन्यथा आप इसे हटाकर कोड को साफ़ रख सकते हैं।

## जावा में कंसोल आउटपुट को कैसे कैप्चर करें

नीचे एक चरण‑दर‑चरण गाइड है जो दिखाता है कि कैसे कन्वर्ज़न विकल्पों को कॉन्फ़िगर करें, जॉब नाम को ओवरराइड करें, और टर्मिनल आउटपुट को डिस्क पर फ़ाइल में निर्देशित करें।

### चरण 1: कन्वर्ज़न विकल्प बनाएं

सबसे पहले, एक `TeXOptions` इंस्टेंस बनाएं जो डिफ़ॉल्ट ObjectTeX कॉन्फ़िगरेशन का उपयोग करता है। यह ऑब्जेक्ट कन्वर्ज़न प्रक्रिया की सभी सेटिंग्स को रखेगा।

```java
// ExStart:OverrideJobName-WriteTerminalOutputToFileSystem
TeXOptions options = TeXOptions.consoleAppOptions(TeXConfig.objectTeX());
// ExEnd:OverrideJobName-WriteTerminalOutputToFileSystem
```

### चरण 2: जॉब नाम और वर्किंग डायरेक्टरी निर्दिष्ट करें

अगला, एक कस्टम जॉब नाम सेट करें और परिभाषित करें कि इनपुट और आउटपुट फ़ाइलें कहाँ स्थित हैं। यदि आप जॉब नाम सेट नहीं करते हैं, तो `TeXJob` कंस्ट्रक्टर का पहला आर्ग्यूमेंट स्वचालित रूप से जॉब नाम बन जाता है।

```java
options.setJobName("overridden-job-name");
options.setInputWorkingDirectory(new InputFileSystemDirectory("Your Input Directory"));
options.setOutputWorkingDirectory(new OutputFileSystemDirectory("Your Output Directory"));
```

> **जॉब नाम को ओवरराइड क्यों करें?**  
> जॉब नाम को ओवरराइड करने से लॉग फ़ाइलें और उत्पन्न आर्टिफैक्ट्स को पहचानना आसान हो जाता है, विशेष रूप से जब आप समानांतर में कई जॉब चलाते हैं या बैच प्रोसेस को स्वचालित करते हैं।

### चरण 3: टर्मिनल आउटपुट को फ़ाइल सिस्टम में लिखें

Aspose.TeX को बताएं कि वह कंसोल पर सामान्यतः दिखाई देने वाली सभी चीज़ों को कैप्चर करे और उसे फ़ाइल में लिखे। फ़ाइल का नाम `<job_name>.trm` होगा और यह ऊपर परिभाषित आउटपुट वर्किंग डायरेक्टरी में रखी जाएगी।

```java
options.setTerminalOut(new OutputFileTerminal(options.getOutputWorkingDirectory()));
```

### चरण 4: जॉब चलाएँ

अंत में, इच्छित इनपुट फ़ाइल (यहाँ हम एक सरल “hello‑world” उदाहरण का उपयोग करते हैं) और XPS रेंडरिंग डिवाइस के साथ एक `TeXJob` बनाएं। फिर `run()` को कॉल करके कन्वर्ज़न को निष्पादित करें।

```java
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.run();
```

जब जॉब समाप्त हो जाएगा, तो आपको **Your Output Directory** के अंदर `overridden-job-name.trm` नाम की फ़ाइल मिलेगी जिसमें पूर्ण टर्मिनल लॉग होगा।

## Common Pitfalls & Troubleshooting

| समस्या | कारण | समाधान |
|-------|-------|-----|
| **`.trm` फ़ाइल नहीं बन रही** | `setTerminalOut` नहीं बुलाया गया या आउटपुट डायरेक्टरी गायब | सुनिश्चित करें कि आउटपुट डायरेक्टरी मौजूद है और `options.setTerminalOut(...)` को `job.run()` से पहले चलाया गया है। |
| **फ़ाइल नाम ओवरराइड किए गए नाम जैसा नहीं है** | जॉब नाम सही से सेट नहीं हुआ | सुनिश्चित करें कि `options.setJobName("your‑desired‑name")` को `TeXJob` बनाने **से पहले** बुलाया गया है। |
| **खाली लॉग फ़ाइल** | लॉगिंग शुरू होने से पहले एक्सेप्शन फेंके गए | `job.run()` को try‑catch ब्लॉक में रखें और मिसिंग फ़ॉन्ट्स या खराब TeX स्रोत के लिए एक्सेप्शन स्टैक ट्रेस देखें। |
