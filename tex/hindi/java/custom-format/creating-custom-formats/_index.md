# जावा में सुसंगत टाइपसेटिंग के लिए TeX फ़ॉर्मैट कैसे बनाएं

कई दस्तावेज़ों में सुसंगत टाइपसेटिंग एक सिरदर्द बन सकती है—विशेष रूप से जब आपको बार‑बार वही लेआउट नियम चाहिए होते हैं। **इस ट्यूटोरियल में आप सीखेंगे कि Aspose.TeX for Java के साथ TeX फ़ॉर्मैट कैसे बनाएं**, और आप ठीक‑ठीक देखेंगे कि **TeX इनपुट और आउटपुट डायरेक्टरी कैसे सेट करें** ताकि इंजन को पता हो कि स्रोत फ़ाइलें कहाँ पढ़नी हैं और उत्पन्न परिणाम कहाँ लिखने हैं। अंत तक, आप एक कस्टम TeX फ़ॉर्मैट जेनरेट कर पाएंगे जो आपके सभी जावा‑आधारित दस्तावेज़ पाइपलाइन के लिए समान शैली सुनिश्चित करता है।

## त्वरित उत्तर
- **“create custom TeX format” का क्या मतलब है?** यह Aspose.TeX इंजन को मैक्रोज़, फ़ॉन्ट्स और लेआउट नियमों का पुन: उपयोग योग्य सेट कम्पाइल करने के लिए बताता है।
- **क्या मुझे लाइसेंस चाहिए?** विकास के लिए एक फ्री ट्रायल काम करता है; उत्पादन के लिए एक कमर्शियल लाइसेंस आवश्यक है।
- **कौन सा JDK संस्करण आवश्यक है?** Java 8 या उससे ऊपर।
- **क्या मैं रन‑टाइम पर इनपुट फ़ोल्डर बदल सकता हूँ?** हाँ—`setInputWorkingDirectory` का उपयोग करें।
- **क्या आउटपुट फ़ोल्डर कॉन्फ़िगर किया जा सकता है?** बिल्कुल—`setOutputWorkingDirectory` का उपयोग करें।

## कस्टम TeX फ़ॉर्मैट क्या है?
एक कस्टम TeX फ़ॉर्मैट TeX मैक्रोज़, पैकेजेज़ और कॉन्फ़िगरेशन सेटिंग्स का पूर्व‑कम्पाइल्ड संग्रह है जिसे इंजन रन‑टाइम पर लोड करता है। हर दस्तावेज़ के लिए समान स्टाइल फ़ाइलों को पार्स करने के बजाय, आप उन्हें एक बार फ़ॉर्मैट (जैसे, `customtex.fmt`) में कम्पाइल करते हैं और पुनः उपयोग करते हैं, जिससे प्रदर्शन में नाटकीय सुधार होता है और समान रेंडरिंग सुनिश्चित होती है।

## TeX इनपुट और आउटपुट डायरेक्टरी क्यों सेट करें?
**TeX इनपुट डायरेक्टरी** सेट करने से इंजन को पता चलता है कि आपके स्रोत `.tex` फ़ाइलें, फ़ॉन्ट्स और सहायक संसाधन कहाँ स्थित हैं। **TeX आउटपुट डायरेक्टरी** यह निर्धारित करती है कि कम्पाइल्ड PDFs, लॉग्स और सहायक फ़ाइलें कहाँ लिखी जाएँगी। इन पाथ्स को सही ढंग से कॉन्फ़िगर करने से “फ़ाइल नहीं मिली” त्रुटियों से बचा जा सकता है और आपके प्रोजेक्ट फ़ोल्डर को व्यवस्थित रखा जा सकता है।

## पूर्वापेक्षाएँ
- **Aspose.TeX for Java** – [Aspose.TeX डाउनलोड पेज](https://releases.aspose.com/tex/java/) से डाउनलोड करें।
- **वर्किंग डायरेक्टरीज़** – एक *इनपुट* फ़ोल्डर (जहाँ आपकी `.tex` फ़ाइलें हों) और एक *आउटपुट* फ़ोल्डर (जहाँ जेनरेटेड PDFs सेव हों) तय करें। स्निपेट्स में `"Your Input Directory"` और `"Your Output Directory"` को अपने वास्तविक पाथ्स से बदलें।
- **Java Development Kit (JDK)** – संस्करण 8 या उससे नया स्थापित और आपके IDE या बिल्ड सिस्टम में कॉन्फ़िगर किया हुआ।

## पैकेज इम्पोर्ट करें
पहले, उन क्लासेज़ को इम्पोर्ट करें जिनकी हमें आवश्यकता होगी। इस ब्लॉक को जैसा है वैसा ही रखें; यह कोर Aspose.TeX API और सैंपल प्रोजेक्ट में उपयोग की गई एक यूटिलिटी क्लास को लाता है।

```java
package com.aspose.tex.CustomTeXFormatFileCreation;

import java.io.IOException;

import com.aspose.tex.InputFileSystemDirectory;
import com.aspose.tex.OutputFileSystemDirectory;
import com.aspose.tex.TeXConfig;
import com.aspose.tex.TeXJob;
import com.aspose.tex.TeXOptions;

import util.Utils;
```

## कस्टम TeX फ़ॉर्मैट बनाने के लिए चरण‑दर‑चरण गाइड

### चरण 1: TeX Options इनिशियलाइज़ करें (एक “no‑format” इंजन बनाएं)
हम एक `TeXOptions` ऑब्जेक्ट बनाकर शुरू करते हैं जो इंजन को बताता है कि अभी हम कोई पूर्व‑मौजूद फ़ॉर्मैट लोड नहीं करना चाहते। यह **कस्टम TeX फ़ॉर्मैट बनाने** की नींव है।

```java
// Create TeX engine options for no format upon ObjectTeX engine extension.
TeXOptions options = TeXOptions.consoleAppOptions(TeXConfig.objectIniTeX());
```

### चरण 2: TeX इनपुट डायरेक्टरी सेट करें
Aspose.TeX को बताएं कि स्रोत फ़ाइलें, स्टाइल पैकेज और कोई भी सहायक संसाधन कहाँ मिलेंगे। `"Your Input Directory"` को आपके प्रोजेक्ट के इनपुट फ़ोल्डर के एब्सॉल्यूट या रिलेटिव पाथ से बदलें।

```java
// Specify a file system working directory for the input.
options.setInputWorkingDirectory(new InputFileSystemDirectory("Your Input Directory"));
```

> **प्रो टिप:** विकास के दौरान भ्रम से बचने के लिए एब्सॉल्यूट पाथ्स का उपयोग करें, क्योंकि IDE की वर्किंग डायरेक्टरी अलग हो सकती है।

### चरण 3: TeX आउटपुट डायरेक्टरी सेट करें
अब हम यह निर्धारित करते हैं कि कम्पाइल्ड PDFs और लॉग फ़ाइलें कहाँ लिखी जाएँगी। यह **set tex output directory** चरण है।

```java
// Specify a file system working directory for the output.
options.setOutputWorkingDirectory(new OutputFileSystemDirectory("Your Output Directory"));
```

### चरण 4: फ़ॉर्मैट निर्माण कमांड चलाएँ
विकल्पों को कॉन्फ़िगर करने के बाद, हम Aspose.TeX से फ़ॉर्मैट को कम्पाइल करने को कहते हैं। पहला आर्ग्यूमेंट नए फ़ॉर्मैट का नाम है (`"customtex"`), और दूसरा आर्ग्यूमेंट उन विकल्पों को पास करता है जो हमने अभी तैयार किए हैं।

```java
// Run format creation.
TeXJob.createFormat("customtex", options);
```

इस कॉल के समाप्त होने के बाद, आपको अपने आउटपुट डायरेक्टरी में `customtex.fmt` (या इसी तरह का बाइनरी) नाम की फ़ाइल मिलेगी। इस फ़ाइल को भविष्य में लोड करके प्रोसेसिंग को तेज़ किया जा सकता है।

### चरण 5: टर्मिनल आउटपुट को साफ़ करें (वैकल्पिक)
अंतिम स्निपेट कंसोल में एक नई लाइन जोड़ता है ताकि जॉब समाप्त होने के बाद टर्मिनल डिस्प्ले साफ़ दिखे।

```java
// For further output to look fine.
options.getTerminalOut().getWriter().newLine();
// ExEnd:CreateCustomTeXFormatFile
```

## सामान्य समस्याएँ और समाधान

| Issue | Cause | Fix |
|-------|-------|-----|
| **“.tex स्रोत के लिए “फ़ाइल नहीं मिली”** | गलत इनपुट डायरेक्टरी पाथ | `setInputWorkingDirectory` को पास किया गया पाथ आपके `.tex` फ़ाइलों वाले फ़ोल्डर से मेल खाता है, यह सत्यापित करें। |
| **आउटपुट फ़ोल्डर पर अनुमति अस्वीकृत** | लिखने की अनुमति नहीं है | सुनिश्चित करें कि Java प्रोसेस को `setOutputWorkingDirectory` द्वारा सेट किए गए डायरेक्टरी में लिखने की अनुमति है। |
| **फ़ॉर्मैट निर्माण अटक रहा है** | लोड हो रहे पैकेजों की बड़ी संख्या | केवल आवश्यक पैकेजों को प्री‑कम्पाइल करें; यदि अनावश्यक हो तो पूरी TeX डिस्ट्रिब्यूशन लोड करने से बचें। |

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं एक ही कस्टम फ़ॉर्मैट को कई जावा एप्लिकेशन में पुनः उपयोग कर सकता हूँ?**  
**उत्तर:** हाँ। जेनरेटेड `.fmt` फ़ाइल प्लेटफ़ॉर्म‑इंडिपेंडेंट है और किसी भी Aspose.TeX इंजन इंस्टेंस द्वारा लोड की जा सकती है।

**प्रश्न: नया मैक्रो जोड़ने के बाद क्या मुझे फ़ॉर्मैट को फिर से जनरेट करना होगा?**  
**उत्तर:** आपको `TeXJob.createFormat` को फिर से चलाना होगा जब भी आप मैक्रो डिफ़िनिशन या पैकेज सूची बदलते हैं, जिस पर फ़ॉर्मैट निर्भर करता है।

**प्रश्न: क्या रन‑टाइम पर प्रोग्रामेटिकली इनपुट और आउटपुट डायरेक्टरी सेट करना संभव है?**  
**उत्तर:** बिल्कुल—`TeXJob.createFormat` या `TeXJob.process` को कॉल करने से पहले `options.setInputWorkingDirectory(...)` और `options.setOutputWorkingDirectory(...)` को कॉल करें।

**प्रश्न: यह डिफ़ॉल्ट `plain` फ़ॉर्मैट उपयोग करने से कैसे अलग है?**  
**उत्तर:** डिफ़ॉल्ट फ़ॉर्मैट हर बार एक सामान्य मैक्रो सेट लोड करता है, जिससे ओवरहेड बढ़ता है। कस्टम फ़ॉर्मैट प्री‑कम्पाइल्ड, तेज़ और आपके द्वारा परिभाषित सटीक लेआउट को सुनिश्चित करता है।

**प्रश्न: क्या यह गैर‑विंडोज़ ऑपरेटिंग सिस्टम पर काम करेगा?**  
**उत्तर:** हाँ। Aspose.TeX for Java क्रॉस‑प्लेटफ़ॉर्म है; केवल यह सुनिश्चित करें कि फ़ाइल पाथ्स आपके OS के अनुसार सही सेपरेटर का उपयोग करें।

## निष्कर्ष
अब आपके पास Aspose.TeX for Java के साथ **कस्टम TeX फ़ॉर्मैट बनाने** के लिए एक पूर्ण, प्रोडक्शन‑रेडी रेसिपी है। **TeX इनपुट डायरेक्टरी सेट करके** और **TeX आउटपुट डायरेक्टरी सेट करके**, आप यह पूरी तरह नियंत्रित कर सकते हैं कि स्रोत फ़ाइलें कहाँ पढ़ी जाएँ और परिणाम कहाँ लिखे जाएँ, जिससे आपके सभी जावा प्रोजेक्ट्स में विश्वसनीय, दोहराने योग्य टाइपसेटिंग प्राप्त होती है।
