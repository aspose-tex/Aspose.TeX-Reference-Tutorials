# स्टेप बाय स्टेप कन्वर्ज़न – जावा प्रिंटेबल इनवॉइस बनाना: LaTeX से XPS फ़ॉर्मेट

## परिचय

यदि आपको जावा एप्लिकेशन के भीतर LaTeX स्रोतों से **java create printable invoices** बनाने की आवश्यकता है, तो आप सही ट्यूटोरियल पर आए हैं। **Aspose.TeX for Java** के साथ आप LaTeX फ़ाइलों को रेंडर कर सकते हैं, जटिल फ़ॉर्मूले संभाल सकते हैं, और उच्च‑गुणवत्ता वाले XPS फ़ाइलें आउटपुट कर सकते हैं जो बिल्कुल उसी तरह प्रिंट होती हैं जैसा आप अपेक्षा करते हैं। अगले कुछ मिनटों में हम पूरे वर्कफ़्लो को समझेंगे, बताएँगे कि यह विधि इनवॉइस जनरेशन के लिए क्यों आदर्श है, और दिखाएँगे कि अपने रिपोर्टिंग पाइपलाइन के लिए कन्वर्ज़न को कैसे कस्टमाइज़ करें। यह गाइड एक पूर्ण **step by step conversion** दर्शाता है जिसे आप किसी भी जावा प्रोजेक्ट में कॉपी‑पेस्ट कर सकते हैं।

## LaTeX से XPS कन्वर्ज़न के साथ जावा प्रिंटेबल इनवॉइस कैसे बनाएं?

अपना LaTeX स्रोत लोड करें, रेंडरिंग विकल्प कॉन्फ़िगर करें, और XPS डिवाइस को कॉल करें – यही पाँच संक्षिप्त चरणों में पूरी कन्वर्ज़न है। Aspose.TeX फ़ॉन्ट एम्बेडिंग, फ़ॉर्मूला रास्टराइज़ेशन, और ग्राफ़िक्स कन्वर्ज़न को स्वचालित रूप से संभालता है, इसलिए आप सामान्य 2‑पेज दस्तावेज़ों के लिए एक सेकंड से कम समय में तैयार‑प्रिंट XPS इनवॉइस प्राप्त कर लेते हैं।

## स्टेप बाय स्टेप कन्वर्ज़न क्या है?

स्टेप बाय स्टेप कन्वर्ज़न बड़े LaTeX‑से‑XPS कार्य को छोटे, दोहराने योग्य कार्यों में विभाजित करता है, जिससे आप प्रत्येक चरण को स्वतंत्र रूप से वैलिडेट कर सकते हैं। फ़ाइल लोडिंग, विकल्प कॉन्फ़िगरेशन, डिवाइस इनिशियलाइज़ेशन, और अंतिम रन को अलग करके, आप जल्दी से त्रुटियों को ढूँढ़ सकते हैं, प्रदर्शन सेटिंग्स को समायोजित कर सकते हैं, और कई इनवॉइस की बैच प्रोसेसिंग के लिए वही पाइपलाइन पुनः उपयोग कर सकते हैं।

## जावा LaTeX रेंडरिंग के लिए Aspose.TeX का उपयोग क्यों करें?

## पैकेज आयात करें

`import` ब्लॉक आवश्यक क्लासों को स्कोप में लाता है। इस सूची को बिल्कुल वैसा ही रखने से कोड बिना संशोधन के कंपाइल हो जाता है।

```java
package com.aspose.tex.LaTeXXpsConversionSimplest;

import java.io.ByteArrayInputStream;
import java.io.IOException;
import java.util.Calendar;
import java.util.GregorianCalendar;

import com.aspose.tex.InputFileSystemDirectory;
import com.aspose.tex.Interaction;
import com.aspose.tex.OutputFileSystemDirectory;
import com.aspose.tex.TeXConfig;
import com.aspose.tex.TeXJob;
import com.aspose.tex.TeXOptions;
import com.aspose.tex.rendering.XpsDevice;
import com.aspose.tex.rendering.XpsSaveOptions;

import util.Utils;
```

अब चलिए कन्वर्ज़न चरणों को देखते हैं, प्रत्येक कोड फ्रैगमेंट के उद्देश्य को समझाते हुए।

## चरण 1: इनपुट और आउटपुट डायरेक्टरी सेट करें

आपको Aspose.TeX को बताना होगा कि स्रोत `.ltx` फ़ाइल कहाँ है और परिणामी XPS फ़ाइल कहाँ लिखनी है।

```java
options.setInputWorkingDirectory(new InputFileSystemDirectory("Your Input Directory"));
options.setOutputWorkingDirectory(new OutputFileSystemDirectory("Your Output Directory"));
```

*टिप:* `"Your Input Directory"` और `"Your Output Directory"` को अपने मशीन पर मौजूद पूर्ण या सापेक्ष पाथ से बदलें।

## चरण 2: TeX विकल्प कॉन्फ़िगर करें

ये विकल्प नियंत्रित करते हैं कि कन्वर्ज़न के दौरान LaTeX इंजन कैसे व्यवहार करता है। इन्हें अपने दस्तावेज़ की आवश्यकताओं के अनुसार समायोजित करें।  
TeXOptions वह क्लास है जो इन कॉन्फ़िगरेशन सेटिंग्स को रखती है।

```java
options.setInteraction(Interaction.NonstopMode);
options.setDateTime(new GregorianCalendar(2022, Calendar.DECEMBER, 18).getTime());
options.ignoreMissingPackages(true);
options.noLigatures(true);
options.repeat(true);
```

- **इंटरैक्शन मोड** – `NonstopMode` त्रुटियों का सामना करने पर भी प्रोसेसिंग जारी रखता है।  
- **DateTime** – दस्तावेज़ शीर्षक पृष्ठ पर दिखाई देने वाली तिथि सेट करता है।  
- **ignoreMissingPackages** – जब कोई पैकेज नहीं मिलता तो जॉब को फेल होने से रोकता है।  
- **noLigatures** – यदि आप साधारण अक्षर पसंद करते हैं तो टाइपोग्राफिक लिगेचर को निष्क्रिय करता है।  
- **repeat** – क्रॉस‑रेफ़रेंसेज़ के लिए इंजन को पुनः चलाने की अनुमति देता है।  

## चरण 3: XPS सेव ऑप्शन इनिशियलाइज़ करें

XpsSaveOptions XPS आउटपुट पैरामीटर जैसे कंप्रेशन और पेज साइज को कॉन्फ़िगर करता है।

```java
options.setSaveOptions(new XpsSaveOptions());
```

## चरण 4: XPS सेव ऑप्शन को कस्टमाइज़ करें

XPS आउटपुट में फ़ॉर्मूले, ग्राफ़िक्स, और फ़ॉन्ट्स को कैसे संभालना है, इसे सूक्ष्म रूप से ट्यून करें।  
`rasterizeFormulas` निर्धारित करता है कि फ़ॉर्मूले इमेज के रूप में रेंडर किए जाएँ या नहीं।

```java
options.getSaveOptions().rasterizeFormulas(true);
options.getSaveOptions().rasterizeIncludedGraphics(true);
options.getSaveOptions().subsetFonts(true);
```

- **rasterizeFormulas** – गणितीय अभिव्यक्तियों को इमेज में बदलता है, जिससे वे किसी भी XPS व्यूअर पर सही ढंग से प्रदर्शित हों।  
- **rasterizeIncludedGraphics** – एम्बेडेड ग्राफ़िक्स को रास्टराइज़ करने के लिए मजबूर करता है, जिससे संगतता में सुधार हो सकता है।  
- **subsetFonts** – दस्तावेज़ में उपयोग किए गए केवल ग्लिफ़ को एम्बेड करता है, जिससे सामान्य इनवॉइस के लिए फ़ाइल आकार में 40 % तक कमी आती है।  

## चरण 5: LaTeX से XPS कन्वर्ज़न चलाएँ

XpsDevice रेंडर किए गए कंटेंट को XPS फ़ाइल में लिखता है।  
`TeXJob` इनपुट फ़ाइल, आउटपुट डिवाइस (`XpsDevice`), और सभी कॉन्फ़िगर किए गए विकल्पों को जोड़ता है।

```java
new TeXJob("Your Input Directory" + "sample.ltx", new XpsDevice(), options).run();
```

`run()` कॉल समाप्त होने के बाद, आप `sample.xps` को पहले निर्दिष्ट आउटपुट डायरेक्टरी में पाएँगे। यह XPS फ़ाइल सीधे प्रिंटर को भेजी जा सकती है या PDF इनवॉइस बंडल में एम्बेड की जा सकती है।

## अतिरिक्त उदाहरण

### इनपुटस्ट्रीम का उपयोग (सीधे LaTeX स्ट्रिंग को कन्वर्ट करें)

यदि आप फ़ाइल के बजाय मेमोरी से LaTeX स्रोत फीड करना पसंद करते हैं, तो स्रोत को `ByteArrayInputStream` में रैप करें।

```java
new TeXJob(new ByteArrayInputStream(
    "\\documentclass{article} \\begin{document} Hello, World! \\end{document}".getBytes("ASCII")),
    new XpsDevice(), options).run();
```

### मुख्य इनपुट टर्मिनल का उपयोग (Aspose.TeX को फ़ाइल स्वचालित रूप से खोजने दें)

जब आपको फ़ाइल पाथ स्पष्ट रूप से निर्दिष्ट करने की आवश्यकता नहीं होती, तो आप डिफ़ॉल्ट इनपुट टर्मिनल पर भरोसा कर सकते हैं।

```java
new TeXJob(new XpsDevice(), options).run();
```
