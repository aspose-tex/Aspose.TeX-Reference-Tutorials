# 如何捕获控制台输出 C# – 覆盖作业名称并写入磁盘

在本分步教程中，您将了解在使用 Aspose.TeX for .NET 时 **如何捕获控制台输出 C#**。通过覆盖作业名称并将终端流路由到文件，您可以获得确定性的文件命名、轻松的日志聚合，以及为 CI/CD 流水线或批处理场景提供坚实的基础。

## 快速答案
- **What does “capture console output C#” mean?** 它将 Aspose.TeX 生成的标准终端流重定向到您指定的文件中，保留所有日志、警告和状态信息。  
- **Why override the job name?** 覆盖可确保文件名可预测，消除并行作业中的命名冲突，并简化下游脚本。  
- **Which Aspose.TeX class writes the output?** `OutputFileTerminal`，通过 `options.TerminalOut` 分配。*OutputFileTerminal 是一个将终端消息写入文件的类。*  
- **Can I set a custom TeX input folder?** 是的——使用 `options.InputWorkingDirectory` 指向您的源 `.tex` 文件。  
- **Is this approach compatible with .NET Core?** 绝对兼容；相同的 API 在 .NET Framework、.NET Core 和 .NET 5+ 上均可使用。

## 在 Aspose.TeX 上下文中，“capture console output C#” 是什么？
捕获控制台输出是指将通常出现在终端窗口中的所有内容（日志信息、警告、编译细节）写入持久文件。这对于调试大型 TeX 项目或将 TeX 处理集成到自动化工作流中尤为有用。

## 为什么覆盖作业名称并将终端输出写入文件？
覆盖作业名称可让您决定所有生成工件的基础名称，而将终端日志写入文件则提供完整的审计轨迹。可预测的文件名使后处理脚本变得简单，保存的日志使您能够事后审查编译步骤，这对自动化构建和并行执行环境至关重要。

## 使用 Aspose.TeX 的量化收益
Aspose.TeX 支持 **30 多种输入和输出格式**——包括 `.tex`、`.pdf`、`.xps`、`.html` 和常见图像类型——并且能够在不将整个文件加载到内存中的情况下处理 **最多 500 页的文档**。与朴素的基于文件的方法相比，库的流式架构可将峰值内存使用量降低最多 **70 %**。

## 前提条件

- **Aspose.TeX for .NET** – 从 [Aspose.TeX website](https://releases.aspose.com/tex/net/) 下载最新版本。

- **Basic C# knowledge** – 您应熟悉命名空间、对象和文件系统路径。  
- **Input and output directories** – 在磁盘上创建两个文件夹：一个用于源 `.tex` 文件，另一个用于生成的工件和日志。

## 导入命名空间

在 C# 文件的顶部添加所需的 `using` 指令，以便编译器能够定位 Aspose.TeX 类型。

```csharp
using Aspose.TeX.IO;
using Aspose.TeX.Presentation.Xps;
```

## 步骤 1：创建转换选项

`TeXOptions` 类封装了 TeX 转换作业的所有配置设置。它允许您指定控制台应用行为、输入目录和输出设备。

`ConsoleAppOptions` 配置 Aspose.TeX 的控制台应用行为。  
`TeXConfig` 定义 TeX 处理配置的类型。

创建一个带有 `ConsoleAppOptions` 的 `TeXOptions` 实例，并将 `ObjectTeX` 指定为 `TeXConfig`。这告诉 Aspose.TeX 您正在运行在控制台应用场景中。

```csharp
TeXOptions options = TeXOptions.ConsoleAppOptions(TeXConfig.ObjectTeX());
```

## 步骤 2：指定作业名称（覆盖默认）

`TeXJob` 类使用作业名称作为每个生成文件的基础。覆盖它可防止库生成类似 `job_12345` 的随机名称。

```csharp
options.JobName = "overridden-job-name";
```

在创建作业实例之前，设置 `options.JobName = "MyCustomJob"`（或任何您喜欢的标识符）。

## 步骤 3：设置 TeX 输入目录

`InputWorkingDirectory` 属性告诉 Aspose.TeX 在哪里定位您的源 `.tex` 文件。这就是 **设置 tex 输入目录** 的步骤。

```csharp
options.InputWorkingDirectory = new InputFileSystemDirectory("Your Input Directory");
```

提供包含 TeX 源文件的文件夹的绝对或相对路径，例如 `@"C:\\TeX\\Sources"`。

## 步骤 4：指定输出工作目录

定义保存处理后文件和捕获的控制台日志的文件夹。Aspose.TeX 将根据需要创建子文件夹，但根文件夹必须存在且可写。

```csharp
options.OutputWorkingDirectory = new OutputFileSystemDirectory("Your Output Directory");
```

设置 `options.OutputWorkingDirectory = @"C:\\TeX\\Results"`（确保该目录具有写入权限）。

## 步骤 5：将终端输出写入文件

`OutputFileTerminal` 是将每条控制台消息写入文件的具体实现。通过将其分配给 `options.TerminalOut`，您替换了默认的内存中控制台流。

```csharp
options.TerminalOut = new OutputFileTerminal(options.OutputWorkingDirectory);
```

示例：`options.TerminalOut = new OutputFileTerminal(Path.Combine(options.OutputWorkingDirectory, "console.log"));`

## 步骤 6：运行作业

`TeXJob` 类协调转换。您提供覆盖后的作业名称、输出设备（本例中为 `XpsDevice`）以及完整配置的 options 对象。`XpsDevice` 从 TeX 源生成 XPS 输出。

```csharp
TeXJob job = new TeXJob("hello-world", new XpsDevice(), options);
job.Run();
```

调用 `job.Run();` 后，Aspose.TeX 生成 XPS 文件并将整个终端日志写入输出文件夹中的 `console.log`。

## 常见问题与故障排除

| 症状 | 可能原因 | 解决办法 |
|---------|--------------|-----|
| 未创建输出文件 | 输出目录路径不正确或缺少写入权限 | 验证 `options.OutputWorkingDirectory` 指向有效文件夹且进程具有写入权限。 |
| 终端日志为空 | `TerminalOut` 未设置或随后被覆盖 | 确保在 `job.Run();` 之前执行 `options.TerminalOut = new OutputFileTerminal(...);`。 |
| 作业因“未找到文件”而失败 | 输入目录未正确设置 | 再次检查传递给 `options.InputWorkingDirectory` 的路径，并确保其中存在 `.tex` 文件。 |
| 日志中出现意外字符 | TeX 引擎与文件系统之间的编码不匹配 | 在运行作业前设置 `options.Encoding = Encoding.UTF8`（或相应的编码）。 |
